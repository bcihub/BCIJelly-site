bcijelly.model.BaseModel.test

def test(self, *args: Any, **kwargs: Any) -> dict[str, Any]

概述​

model.test 是模型实例上统一的测试入口。

在委托给后端 test(...) 之前，适配器会在可用时恢复由 model.train(...) 捕获的训练状态。

参数​

大多数后端通用的参数​

test_dataset​

  • 类型：tuple，取决于任务。
  • 作用：内存中的测试数据。
  • 必填：是，除非提供了 test_datapath。

verbose​

  • 类型：bool。
  • 作用：打印测试时的详细信息。

test_datapath​

  • 类型：str | None。
  • 作用：基于路径的测试输入。

模型特定的测试参数​

LFADS 分类器​

  • cleanup_temp_ckpt: bool = False
  • 作用：是否在评估后删除临时检查点文件。

LFADS 回归器​

  • reduce: str = "center"
  • 作用：与 LFADS 回归器流程对齐的降维模式。

CycleGAN 分类器/回归器​

  • calib_ratio: float = 0.1
  • min_calib_trials: int = 5
  • max_calib_trials: int | None = 30
  • 作用：在最终评估前从测试数据中划分出校准子集。

Stabilization 分类器/回归器​

  • calib_ratio: float = 0.2
  • min_calib_trials: int = 1
  • max_calib_trials: int | None = None
  • 作用：稳定化适配路径的校准/评估划分。

任务特定的数据集格式​

  • 分类/回归后端通常期望：

    • test_dataset: tuple[np.ndarray, np.ndarray]

  • 语音后端期望：

    • test_dataset: tuple[np.ndarray, np.ndarray, np.ndarray, np.ndarray, np.ndarray]

状态与执行约束​

  • 后端依赖进程内的训练状态；需在同一运行时中先调用 model.train(...) 再调用 model.test(...)。
  • 适配器级别的状态恢复在后端测试调用前自动执行。

返回值​

  • 类型：dict[str, Any]
  • 典型的指标键取决于任务/后端：
    • 分类：通常为 acc
    • 回归：通常为 r2
    • 某些后端也可能输出 loss、mse 或 mae

使用示例​

test_result = model.test(
    test_dataset=loaded_data.test_dataset,
    verbose=False,
)

对于跨天工作流，此方法在合并后的测试样本上运行一次全局测试。你也可以逐天评估：

day_metric_list = []
for one_day_test_dataset in loaded_data.test_day_datasets:
    one_day_result = model.test(test_dataset=one_day_test_dataset, verbose=False)
    day_metric_list.append(float(one_day_result["acc"]))
