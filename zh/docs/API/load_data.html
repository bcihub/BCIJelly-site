bcijelly.load_data

def load_data(
    dataset_name: str,
    *,
    protocol: str = "single_day",
    task: str | None = None,
    split_ratio: str = "7:1:2",
    subject_index: int = 0,
    day_index: int = 0,
    seed: int | None = 42,
    data_dir: Path | None = None,
    verbose: bool = False,
    auto_download_missing: bool = True,
    fallback_to_first_subject: bool = False,
    repo_id: str = DEFAULT_HF_REPO_ID,
    repo_type: str = DEFAULT_HF_REPO_TYPE,
    max_days_per_dataset: int | None = None,
) -> DataLoadResult

概述​

load_data 是 BCIJelly 中的统一数据加载 API。它自动处理以下事项：

  • 数据集发现与可用性检查（本地或自动下载）
  • 协议/任务兼容性验证
  • 单天或跨天任务构建
  • NPZ 读取及任务感知的标签键选择
  • 训练/验证/测试集划分与特征形状对齐
  • 统一输出为 DataLoadResult

参数​

dataset_name​

  • 默认值：无（必填）。
  • 作用：选择要加载的数据集目录。
  • 影响：决定所有下游的发现、验证、任务构建和文件加载分支。

protocol​

  • 默认值：single_day。
  • 支持的值：single_day、cross_day、train-test、train-val-test。
  • 作用：定义加载范式和划分路径。
  • 关键行为：
    • train-test 和 train-val-test 以单天执行语义运行。
    • cross_day 构建以天为分块的训练/验证/测试划分。

task​

  • 默认值：None，此时回退为 classification。
  • 支持的值：classification、regression、speech。
  • 作用：控制标签键选择和标签形状验证。
  • 关键行为：
    • 分类任务期望 (N,) 标签。
    • 回归任务期望 (N,T,2) 标签，且 T 与神经特征长度对齐。
    • 语音任务当前接受 (N,) 或 (N,L)。

split_ratio​

  • 默认值：7:1:2。
  • 作用：控制训练/验证/测试比例。
  • 规则：
    • a:b 表示训练:测试。
    • a:b:c 表示训练:验证:测试。
    • 训练和测试比例均不能为零。
  • 注意事项：
    • 跨天模式下，分配按天计数而非随机按样本划分。
    • 在 train-val-test 下，数据集格式/声明可能要求显式的验证文件。

subject_index​

  • 默认值：0。
  • 作用：在多被试数据集中选择被试子目录。
  • 注意事项：
    • 对于非多被试结构，仅 0 有效。

day_index​

  • 默认值：0。
  • 作用：在单天加载中选择哪个天/会话任务。
  • 注意事项：
    • 对于 split-files 格式，仅 day_index=0 有效。
    • 跨天模式下通常只有一个任务，因此 day_index 通常保持 0。

data_dir​

  • 默认值：None（内部回退为 data）。
  • 作用：覆盖数据集根目录位置。

auto_download_missing​

  • 默认值：True。
  • 作用：自动从 Hugging Face 下载缺失的数据集。

fallback_to_first_subject​

  • 默认值：False。
  • 作用：当 subject_index 超出范围时，回退到第一个被试并给出警告，而非报错。

seed​

  • 默认值：42。
  • 作用：控制随机划分的可复现性。
  • 关键说明：
    • 可选参数；省略时使用默认值 42。

verbose​

  • 默认值：False。
  • 作用：打印数据集/任务摘要用于诊断。

repo_id、repo_type​

  • 作用：控制下载源（默认为官方 HF 数据集仓库）。
  • 典型用途：私有镜像/测试仓库。

max_days_per_dataset​

  • 作用：限制发现/构建的最大天数。
  • 典型用途：快速调试和小规模实验。

使用示例​

import bcijelly as bj

# 单天分类
loaded = bj.load_data(
    dataset_name="1_FALCONM1-A",
    protocol="single_day", task="classification",
    subject_index=0, day_index=0,
    split_ratio="7:1:2",
)

# 跨天分类
loaded = bj.load_data(
    dataset_name="1_FALCONM1-A",
    protocol="cross_day", task="classification",
    subject_index=0,
    split_ratio="7:1:2",
)

# 单天回归
loaded = bj.load_data(
    dataset_name="5_Jango_force",
    protocol="single_day", task="regression",
    subject_index=0, day_index=0,
    split_ratio="7:1:2",
)

# 跨天回归
loaded = bj.load_data(
    dataset_name="5_Jango_force",
    protocol="cross_day", task="regression",
    subject_index=0,
    split_ratio="7:1:2",
)

执行逻辑​

src/bcijelly/dataset/_loading.py 中的主要流程：

  1. 参数与能力检查
  • 根据能力声明验证 protocol。
  • 验证 task（或回退为 classification）。
  • 解析 split_ratio。
  1. 数据集结构发现
  • discover_dataset_structure 识别格式：
    • single_file
    • split_files
    • day_files
    • multi_subject
    • virtual_sessions（用于 VisualGratingTask）
  1. 数据集声明兼容性验证
  • 如果存在 dataset_compatibility.json，验证请求的 protocol/task。
  1. 被试解析
  • 通过 subject_index 解析被试分支。
  • 超出范围时的行为取决于 fallback_to_first_subject。
  1. 分支 A：split_files 直接加载
  • 固定的训练/测试（及可选验证）文件路径流程。
  • train-val-test 可能要求显式的验证文件。
  1. 分支 B：构建任务后选择其一
  • 通过 build_jobs 构建 DatasetJob 列表。
  • 对于 single_day，按 day_index 选择。
  • 对于 cross_day，通常只有一个构建好的任务。
  1. 任务级加载（load_job_data）
  • single_day：
    • 预划分的单文件键 → 直接训练/测试加载。
    • 原始单文件 → 使用 split_ratio + seed 划分。
  • cross_day：
    • 加载训练/验证/测试天分块。
    • 对齐到共享的最小 (time, channels) 形状。
    • 返回聚合的 test_dataset 和逐天的 test_day_datasets。
  1. 返回结果
  • 统一的 DataLoadResult 字段：
    • train_dataset
    • val_dataset（可选）
    • test_dataset
    • test_day_datasets（跨天模式常见）
    • aligned_shape
    • job
    • requested_task
