1_LSTM

一、模型简介​

LSTM（长短期记忆网络）是一种特殊的循环神经网络（RNN）架构，最早由Hochreiter和Schmidhuber于1997年提出。其核心目的是解决传统RNN在处理长序列数据时，梯度消失和梯度爆炸的问题。LSTM通过引入三个门控机制（输入门、遗忘门和输出门），允许网络在学习过程中保持或忘记信息，从而在长时间依赖的任务中发挥优势。

从代码实现与网络拓扑来看，本 LSTM 架构由以下核心模块构成：

1.时序序列编码器 (Sequential Feature Encoder)

原始输入通常为高维多通道神经或生理信号。模型底层依托于高度封装的RNNEncoder组件进行序列摄入。通过设定网络层数与双向标志位（Bidirectional），模型能够构建深层的双向感受野，在当前时间步同时捕获历史上下文与未来演变趋势。

2.核心门控与状态流转机制 (Gating Mechanism & State Transition)

与 Transformer 依赖显式位置编码不同，LSTM 凭借其递归拓扑天然契合强时序依赖任务。通过遗忘门、输入门与输出门的精密协作，网络在漫长的时间步推演中，能够自主决定神经电位特征的“保留”或“遗忘”。这一核心门控机制有效攻克了传统 RNN 的梯度弥散瓶颈。

3.双轨任务解码头 (Dual-Track Task Head)

为完美适配 BCI 等多场景范式，模型在输出端设计了高度灵活的分支解码逻辑

  • 全局池化分类解码 (Pooled Classification Decoding)：

针对离散的动作意图识别或整段信号定性分析，模型提供了灵活的池化策略（pooling）。无论是对时间维度进行全局均值池化，还是精准提取序列末步状态，特征随后都会流入包含 LayerNorm 与 Dropout 的级联全连接层，输出最终的分类 Logits。

  • 时间分布式回归解码 (Time-Distributed Regression Decoding)：

当面临连续运动学坐标解析等 Seq2Seq 回归需求时，模型采用专用的 TimeDistributedLinear 模块。该模块绕过时间池化，直接在长达序列长度的每一个时间步上，独立且共享权重地应用带 LayerNorm 的线性映射，实现对高频动态系统的逐帧高保真追踪。

二、数据预处理与特征工程​

为了保障模型在复杂序列数据（如脑电信号、时序传感器信号）上的训练稳定性，框架对数据工程设定了严格的规范标准：

  • 张量维度约束： 所有输入的特征矩阵必须强制转换为标准的三维数组格式，分别代表样本数（Trials/Samples）、时间步长（Time steps）与通道/特征数（Channels/Features）。

  • 目标标签对齐：

    • 分类任务：要求标签为一维数组。框架内置了完备的方法，可自动将文本、字节或非连续离散数值形式的标签，精准映射为连续的整数索引。
    • 回归任务：严格要求标签矩阵为三维形状，且特征的时间步与标签的时间步必须完全一致。

  • 特征归一化与严格的数据隔离（Z-score Normalization & Data Isolation）：

    为了消除特征量纲差异对梯度更新的不利影响，框架采取了严苛的工程实践规范以杜绝数据泄露（Data Leakage）。系统仅在训练集上独立拟合统计量，并将生成的标准化统计量单向、冻结地映射应用至验证集与测试集，确保模型性能评估的绝对公平性与真实泛化能力。

  • 特征选择与工程：

    对于复杂的时序数据，可以通过特征选择、PCA降维或时域与频域的特征工程，进一步提升LSTM模型的性能。

四、调用示例​


import numpy as np
import bcijelly

def test_single_day() -> None:
    '''
        单天任务示例。
        完整的使用流程包括：
        1. 引用模型模块，例如 bcijelly.model.lstm.classifier。模块的选择决定了使用哪个模型，以及是进行分类任务还是回归任务（目前不支持回归任务）。
        2. 调用 bcijelly.load_data 加载数据，得到 loaded_data。
        3. 调用模型模块的 train 函数进行训练，得到 train_result。
        4. 调用模型模块的 test 函数进行测试，得到 test_result。
        5. 调用 bcijelly.summary 对结果进行总结和展示。
    '''

    import bcijelly.model.lstm.classifier as module      # 选择模型，设置分类/回归任务（暂不支持回归任务）

    loaded_data = bcijelly.load_data(
        dataset_name="1_FALCONM1-A",    # 选择数据集
        protocol="single_day",                  # "single_day" | "cross_day"，此处演示单天解码
        subject_index=0,                        # 范围: [0, n_subjects - 1]
        day_index=0,                            # 范围: [0, n_days - 1]
        seed=42,                                # int，单天解码中用于设置划分测试集和验证集的随机种子
        single_day_test_size=0.2,               # 范围: (0, 1)，表示测试集占单日数据的比例
        single_day_val_ratio=0.125,             # 范围: [0, 1)，仅当 val_dataset 不是 None 时有效，表示 val 占 train+val 的比例
        # auto_download_missing=True,           # 设置本地缺数据时自动从Hugging Face下载，默认为True
        verbose=False,
    ）

    train_result = module.train(
        train_dataset=loaded_data.train_dataset,
        val_dataset=loaded_data.val_dataset,    # loaded_data.val_dataset 或 None
        seed=42,  # int
        device="auto",
        run_name="demo_single_day_lstm", # 用于结果目录命名
        epochs=1,
        plot=False,
        verbose=False,
    )

    test_result = module.test(
        test_dataset=loaded_data.test_dataset,
        verbose=False,
    )

    bcijelly.summary(
        loaded_data=loaded_data,                # bj.load_data 的返回结果
        Module=module,
        train_result=train_result,              # module.train 返回结果
        test_result=test_result,                # module.test 返回结果
        mode="full",                            # "compact" | "full"，控制 summary 展示的详细程度
    )

def test_cross_day() -> None:
    '''
        跨天任务示例。
        在测试代码部分，我们演示了两种计算测试结果的方式：
        1. 将所有测试日的数据合在一起进行测试，得到一个整体的测试结果。
        2. 对每一个测试日的数据单独进行测试，得到每个测试日的测试结果，然后计算这些测试结果的平均值作为整体的测试结果。
        用户根据自己的需求选择其中一种方式。
    '''

    import bcijelly.model.lstm.classifier as module      # 选择模型，设置分类/回归任务（暂不支持回归任务）

    loaded_data = bcijelly.load_data(
        dataset_name="1_FALCONM1-A",
        protocol="cross_day",                   # "single_day" | "cross_day"，此处演示跨天解码
        subject_index=0,                        # 范围: [0, n_subjects - 1]
        # auto_download_missing=True,           # 设置本地缺数据时自动从Hugging Face下载，默认为True
        verbose=False,
    )

    train_result = module.train(
        train_dataset=loaded_data.train_dataset,
        val_dataset=loaded_data.val_dataset,    # loaded_data.val_dataset 或 None
        seed=42,  # int
        device="auto",
        run_name="demo_cross_day_lstm",  # 用于结果目录命名
        epochs=1,
        plot=False,
        verbose=False,
    )

    # 所有测试日共同测试
    test_result = module.test(
         test_dataset=loaded_data.test_dataset,
         verbose=False,
    )

    # 每一个测试日单独测试
    day_acc_list: list[float] = []
    for one_day_test_dataset in loaded_data.test_day_datasets:
        one_day_result = module.test(
            test_dataset=one_day_test_dataset,
            verbose=False,
        )
        day_acc_list.append(float(one_day_result["acc"]))

    test_result = {
        "acc": float(np.mean(day_acc_list)),
        "per_day_acc": day_acc_list,
    }

    bcijelly.summary(
        loaded_data=loaded_data,                # bj.load_data 的返回结果
        Module=module,
        train_result=train_result,              # module.train 返回结果
        test_result=test_result,                # module.test 返回结果
        mode="full",                            # "compact" | "full"
    )

if __name__ == "__main__":
    test_single_day()
    test_cross_day()

五、模型调试与评估​

LSTM模型的调试与评估通常涉及以下几个方面：

  • 评估指标：
    • 分类任务：框架不仅实时计算整体准确率，还会自动生成完整的混淆矩阵，帮助直观诊断类别不平衡或错判偏好。
    • 回归任务：在原始物理量纲空间内（自动逆归一化后），框架会严格计算均方误差、平均绝对误差以及决定系数。决定系数能更精确地衡量高保真轨迹追踪任务中的拟合优度。
  • 调试技巧：

    • 强制梯度裁剪（Mandatory Gradient Clipping）：为彻底规避传统RNN/LSTM架构在长序列训练中固有的梯度爆炸问题，底层函数在每一次优化器步进前，都会强制执行梯度裁剪。这作为框架的内置保护防线，大幅提升了模型在低信噪比数据下的训练鲁棒性。

    • 动态早停与最优权重快照（Dynamic Early Stopping & Best Checkpoint）：为有效抵御模型过拟合，框架在训练过程中深度集成了结合验证集表现的动态早停机制。模型会自动监控并保存验证集表现最优的权重状态，若在容忍周期内无提升则提前终止训练，从而避免过拟合。

    • 学习率调整：学习率过大会导致训练不稳定，过小则可能导致训练速度过慢。可以通过逐步衰减学习率或使用自适应学习率优化器来优化训练过程。

六、应用说明​

该 LSTM 实现框架在设计上表现出了极强的扩展性，能够灵活应对不同领域的核心痛点：

  • 运动意图解码（Motor Imagery, MI）： 运动想象任务要求受试者在不产生肌肉肌电活动的前提下，在意识中模拟特定动作。大脑头皮脑电（EEG）在此期间会产生非平稳的事件相关去同步/同步（ERD/ERS）现象。LSTM 能够灵敏捕获此类微弱且随时间弥散的动态相变特征，从而实现极高精度的意图分类，为神经假肢与康复外骨骼提供可靠的控制源。

  • 情感状态识别（Emotion Recognition）： 神经生理信号提供了剥离主观伪装的最真实的底层情绪映射。通过提取多频带特征序列，双向 LSTM 能够在时间维度上精准追踪人类情绪的动态时序演变，进而输出高分辨率的情感状态评估。

  • 默读语音接口（Silent Speech Interfaces）： 针对丧失发音能力的患者，基于脑机接口的“内心语言”解码成为前沿难点。患者在脑海中默念指令词汇时，皮层网络会产生具有强烈时序构架的神经脉冲序列。这恰是 LSTM 展现其长程语义记忆与时序依赖挖掘能力的最佳理论着力点。

  • 极低延迟的在线闭环控制潜力 (Ultra-low Latency Online Closed-loop Control)： 在 BCI 的临床落地中，计算延迟是决定假肢控制或光标点击成功率的生死线。当配置为单向（Unidirectional，传入 --no_bidirectional）时，LSTM 架构采用严格的因果（Causal）时间步长递进处理，在推理阶段无需等待未来的神经数据窗，从而具备极致的低延迟特性，非常适合无缝部署于实时闭环系统中。 而在离线的高精度解码任务中，则可保持默认的**双向（Bidirectional）**模式以获取完整的全局上下文。
