11_POYO

一、模型简介​

POYO（Perceiver-based One-shot Yielding Optimizer）是一种基于Perceiver架构的神经事件序列解码模型。与传统将神经脉冲序列化为固定时间网格的方法不同，POYO的核心创新在于：将每个脉冲事件视为一个独立的token（包含神经元标识与精确时间戳），并通过一组可学习的时序潜查询向量（Latent Queries）在连续时间轴上自适应地提取信息。这一设计使模型能够以亚毫秒级时间分辨率处理异步脉冲流，天然适配事件驱动的神经形态数据。

从代码实现与网络拓扑来看，本 POYO 架构由以下核心模块构成：

  1. 事件化Token编码器（build_poyo_input）

原始的密集脉冲矩阵（time × channels）首先被解包为事件序列：

  • 每个非零脉冲计数被展开为一个或多个事件token，每个token携带其所属神经元的单元索引与精确时间戳。
  • 为每个神经元生成起始与终止标记token（token_type=1,2），标识其有效记录的时间窗口。
  • 可选地以二值模式（value>0 → 1事件）处理连续值信号，避免token数量爆炸。
  • 所有token按时间排序，形成变长事件序列。
  1. Perceiver交叉注意力主干

POYO的核心架构遵循Perceiver的设计范式：

  • 潜查询向量：在时间轴上以固定步长均匀采样若干时间点，每个时间点分配若干可学习的潜查询向量（默认32个/步）。
  • 交叉注意力：潜查询向量作为Query，事件token序列作为Key和Value，通过多层交叉注意力机制从异步脉冲流中提取信息。
  • 自注意力：潜查询向量之间通过自注意力层进行信息交换与整合。
  • 该架构将变长事件序列压缩为固定维度的潜表征序列，计算复杂度与事件数量线性相关。
  1. 神经元与会话嵌入

每个事件token的表示由以下分量相加构成：

  • 单元嵌入：可学习的神经元标识嵌入，区分不同记录通道。
  • 会话嵌入：可学习的实验会话标识嵌入，用于跨天对齐。
  • 时间嵌入：基于精确时间戳的位置编码。
  1. 模态特异性读出

模型通过注册表机制支持不同的输出模态：

  • 光标速度二维读出：适用于连续运动解码任务，输出二维运动学坐标。
  • 支持扩展至其他输出模态（如肌电信号、语音参数等）。
  1. 预训练与微调

POYO支持在大规模多会话数据上预训练后，通过以下步骤适配新会话：

  • 重新初始化单元与会话嵌入层的词汇表。
  • 加载预训练的Perceiver主干权重。
  • 以较小学习率微调全模型。

二、调用示例​

分类任务​


import bcijelly as bj

loaded_data = bj.load_data(
    dataset_name="5_Jango",
    protocol="cross_day", task="classification",
    subject_index=0, split_ratio="7:1:2",
)
model = bcijelly.model.poyo(task="classification")
train_result = model.train(loaded_data.train_dataset, loaded_data.val_dataset)
test_result = model.test(loaded_data.test_dataset)

bj.summary(loaded_data, model, train_result, test_result)

回归任务​

import bcijelly as bj

loaded_data = bj.load_data(
    dataset_name="5_Jango",
    protocol="cross_day", task="regression",
    subject_index=0, split_ratio="7:1:2",
)
model = bcijelly.model.poyo(task="regression")
train_result = model.train(loaded_data.train_dataset, loaded_data.val_dataset)
test_result = model.test(loaded_data.test_dataset)

bj.summary(loaded_data, model, train_result, test_result)

默认情况下不加载预训练权重。如需加载预训练权重，请使用：

train_result = model.train(
    loaded_data.train_dataset, loaded_data.val_dataset,
    pretrained_path="auto",
)

四、模型调试与评估​

评估指标

  • 回归任务：在自动逆归一化后的原始物理空间内，计算均方误差（MSE）、平均绝对误差（MAE）及决定系数。

核心调试技巧

  • 潜查询密度：num_latents_per_step 参数决定每个时间步的潜查询向量数量。更多的潜查询提供更高的信息提取容量，但增加计算开销。
  • 时间采样步长：latent_step 参数（默认1/8秒）决定了潜查询在时间轴上的采样密度。较小的步长适合需要高时间分辨率的任务（如精细运动轨迹）。
  • Perceiver深度：depth 参数（默认24层）控制交叉注意力与自注意力的堆叠层数。深层网络具有更强的表征能力，但训练数据需求与计算成本也相应增加。
  • 事件化模式：对于密集脉冲数据，使用二值事件模式（continuous_event_mode=True）可将每个非零值仅展开为1个token，避免计数展开导致的token数量爆炸。
  • 预训练权重：使用 pretrained_path="auto" 可自动下载并加载官方预训练权重，在小样本场景下显著提升性能。

五、应用说明​

POYO 架构在以下场景中展现出独特优势：

(1) 异步事件驱动解码：与基于固定时间窗的模型不同，POYO直接处理异步脉冲事件流，保留了每个脉冲的精确时间信息。这一特性使其在高时间分辨率解码任务（如实时光标控制）中具有天然优势，无需人为选择时间bin大小。

(2) 多会话统一模型：通过可学习的单元与会话嵌入，POYO能够在单一模型中同时处理多个实验会话、多个被试甚至多个脑区的神经数据，实现真正的"一个模型适配所有人"的BCI范式。
