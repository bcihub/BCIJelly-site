7_Language Transformer

一、模型简介​

Language Transformer 是一种专为高频神经电生理信号解析与连续语音解码（Neural Speech Decoding）构建的端到端深度架构。在脑机接口（BCI）领域，尤其是面对复杂序列到序列（seq2seq）的映射任务时，传统的递归神经网络（如 LSTM）往往受制于时间步的串行计算瓶颈与极长序列下的梯度衰减。

本架构彻底摒弃了马尔可夫假设下的时序递归范式，引入了纯粹的 全局多头自注意力机制（Multi-Head Self-Attention）。结合 11_Speech-style 的联结主义时间分类（CTC）推断框架，模型能够在处理单次语音试次（Single-trial）时，瞬间捕获整段神经序列中任意两个时间步之间的非线性长程依赖，实现从高维时空动力学流形到离散语义符号的并行解码。

从理论机制与网络拓扑来看，本架构由以下核心模块构成：

  1. 前端过滤与局部流形展平 (Frontend Filtration & Manifold Unfolding)

    神经群体活动往往伴随高频的瞬时泊松噪声。模型底层的 SpeechFeatureFrontend 并非简单透传信号，而是执行了一套严密的物理学预处理：

    • 高频噪声过滤：利用 GaussianSmoothing 进行一维卷积平滑，在时域上滤除高频伪影。
    • 跨天域偏移补偿：内置 day_weights 与 day_bias 参数矩阵，通过仿射变换在输入端执行跨天特征空间的线性校准。
    • 局部感受野重构：应用 Softsign 激活后，利用 nn.Unfold 将一维时序信号切分为重叠的二维窗口特征。这一操作极大地弥补了 Transformer 捕捉局部特征能力较弱的短板。

  2. 高维投影与绝对时序注入 (High-dimensional Projection & Positional Encoding)

    由于 Transformer 天生缺乏处理序列顺序的归纳偏置（Inductive Bias），前端输出的特征会首先通过线性投影对齐至统一的隐藏流形维度（d_model）。随后，模型引入基于正弦与余弦函数的位置编码 (Positional Encoding)，将绝对的时间节律信息无缝注入到神经特征中，赋予了模型解析脉冲序列时间演化规律的能力。

  3. 深层全局注意力编码器 (Deep Global Attention Encoder)

    模型的核心计算引擎由多层 TransformerEncoderLayer 堆叠而成。

    • 多头注意力 (Multi-Head Attention)：允许模型在不同的特征子空间中，同时关注发音准备、肌肉激活与声学反馈等多个生理维度的时序依赖。
    • 前置归一化架构 (Norm-First Architecture)：显式设置 norm_first=True。在极深的拓扑结构中，这种设计被证明能极大提升残差连接（Residual Connections）中梯度的反向传播效率，保障训练的绝对稳定。

  4. 异步序列解码与边缘化推断 (Asynchronous CTC Decoding)

    模型输出端映射至词表空间后，引入 nn.CTCLoss。该机制通过概率边缘化（Marginalization），允许在未对齐的输入脉冲序列与目标文本序列之间进行端到端训练，彻底打破了传统解码器对帧级别强制对齐的依赖。

二、数据预处理与特征工程​

Language Transformer 的训练稳定性格外依赖于底层数据流的纯净度。框架对时序神经张量的摄入制定了极其严苛的工程学规范：

  • 神经时序张量化严格约束 (Rigorous Neural Temporal Tensorization)：

    底层引擎强制要求特征矩阵 X 为标准三维张量 (N, T, C)（即试次、时间步、通道），而语义标签 y 为二维序列 (N, S)。

  • 动态边界对齐 (Dynamic Boundary Alignment)：

    鉴于语音意图长短不一，框架要求所有样本必须伴随真实的物理边界记录（X_len 与 y_len）。Transformer 的注意力掩码（Attention Mask）将严格依据这些边界进行动态截断，彻底杜绝了无效填充数据（Padding Tokens）对注意力权重的污染。

  • 统计量单向冻结与防泄露机制 (Unidirectional Statistic Freezing)：

    系统在拟合特征分布（Z-score 统计量）时采取了绝对隔离的防泄露策略。均值与标准差仅在训练域内独立拟合，并以冻结状态单向映射至验证与测试域，确保了泛化评估的绝对真实性。

  • 低信噪比环境下的随机扰动注入 (Stochastic Perturbation Injection)：

    在训练前向传播中，框架支持动态注入高斯白噪声（white_noise_sd）与通道恒定偏移噪声（constant_offset_sd），强迫全局注意力机制学习对电极阻抗变化更具鲁棒性的不变特征表征。

四、调用示例​


import numpy as np
import bcijelly

def test_single_day() -> None:
    '''
        单天任务示例。
        完整的使用流程包括：
        1. 引用模型模块，例如 bcijelly.model.language_transformer.classifier。模块的选择决定了使用哪个模型，以及是进行分类任务还是回归任务（目前不支持回归任务）。
        2. 调用 bcijelly.load_data 加载数据，得到 loaded_data。
        3. 调用模型模块的 train 函数进行训练，得到 train_result。
        4. 调用模型模块的 test 函数进行测试，得到 test_result。
        5. 调用 bcijelly.summary 对结果进行总结和展示。
    '''

    import bcijelly.model.language_transformer.classifier as module      # 选择模型，设置分类/回归任务（暂不支持回归任务）

    loaded_data = bcijelly.load_data(
        dataset_name="1_FALCONM1-A",    # 选择数据集
        protocol="single_day",                  # "single_day" | "cross_day"，此处演示单天解码
        subject_index=0,                        # 范围: [0, n_subjects - 1]
        day_index=0,                            # 范围: [0, n_days - 1]
        seed=42,                                # int，单天解码中用于设置划分测试集和验证集的随机种子
        single_day_test_size=0.2,               # 范围: (0, 1)，表示测试集占单日数据的比例
        single_day_val_ratio=0.125,             # 范围: [0, 1)，仅当 val_dataset 不是 None 时有效，表示 val 占 train+val 的比例
        # auto_download_missing=True,           # 设置本地缺数据时自动从Hugging Face下载，默认为True
        verbose=False,
    ）

    train_result = module.train(
        train_dataset=loaded_data.train_dataset,
        val_dataset=loaded_data.val_dataset,    # loaded_data.val_dataset 或 None
        seed=42,  # int
        device="auto",
        run_name="demo_single_day_language_transformer", # 用于结果目录命名
        epochs=1,
        plot=False,
        verbose=False,
    )

    test_result = module.test(
        test_dataset=loaded_data.test_dataset,
        verbose=False,
    )

    bcijelly.summary(
        loaded_data=loaded_data,                # bj.load_data 的返回结果
        Module=module,
        train_result=train_result,              # module.train 返回结果
        test_result=test_result,                # module.test 返回结果
        mode="full",                            # "compact" | "full"，控制 summary 展示的详细程度
    )

def test_cross_day() -> None:
    '''
        跨天任务示例。
        在测试代码部分，我们演示了两种计算测试结果的方式：
        1. 将所有测试日的数据合在一起进行测试，得到一个整体的测试结果。
        2. 对每一个测试日的数据单独进行测试，得到每个测试日的测试结果，然后计算这些测试结果的平均值作为整体的测试结果。
        用户根据自己的需求选择其中一种方式。
    '''

    import bcijelly.model.language_transformer.classifier as module      # 选择模型，设置分类/回归任务（暂不支持回归任务）

    loaded_data = bcijelly.load_data(
        dataset_name="1_FALCONM1-A",
        protocol="cross_day",                   # "single_day" | "cross_day"，此处演示跨天解码
        subject_index=0,                        # 范围: [0, n_subjects - 1]
        # auto_download_missing=True,           # 设置本地缺数据时自动从Hugging Face下载，默认为True
        verbose=False,
    )

    train_result = module.train(
        train_dataset=loaded_data.train_dataset,
        val_dataset=loaded_data.val_dataset,    # loaded_data.val_dataset 或 None
        seed=42,  # int
        device="auto",
        run_name="demo_cross_day_language_transformer",  # 用于结果目录命名
        epochs=1,
        plot=False,
        verbose=False,
    )

    # 所有测试日共同测试
    test_result = module.test(
         test_dataset=loaded_data.test_dataset,
         verbose=False,
    )

    # 每一个测试日单独测试
    day_acc_list: list[float] = []
    for one_day_test_dataset in loaded_data.test_day_datasets:
        one_day_result = module.test(
            test_dataset=one_day_test_dataset,
            verbose=False,
        )
        day_acc_list.append(float(one_day_result["acc"]))

    test_result = {
        "acc": float(np.mean(day_acc_list)),
        "per_day_acc": day_acc_list,
    }

    bcijelly.summary(
        loaded_data=loaded_data,                # bj.load_data 的返回结果
        Module=module,
        train_result=train_result,              # module.train 返回结果
        test_result=test_result,                # module.test 返回结果
        mode="full",                            # "compact" | "full"
    )

if __name__ == "__main__":
    test_single_day()
    test_cross_day()

五、模型调试与评估​

在连续语言解码领域，模型的评估必须跳脱出简单的逐帧准确率，聚焦于全局序列拓扑的保真度：

  • 序列编辑距离评估 (Levenshtein Sequence Evaluation)：

    框架摒弃了生硬的交叉熵指标，采用字符错误率（CER）作为唯一指导北极星。通过计算模型贪心解码序列与真实语义符号之间的最小插入、删除与替换次数（即 Levenshtein 距离），CER 能够最直观地反映 BCI 系统在真实沟通场景下的可用度。

  • 强制动力学稳定 (Mandatory Dynamical Stabilization)：

    自注意力机制在深层网络中容易出现梯度分布异常。为规避此类风险，底层优化引擎被硬编码为在每次前向/反向步进间执行严格的 clip_grad_norm_ （阈值 1.0）裁剪，辅以 Norm-First 架构，充当了模型训练的双重物理安全防线。

  • 自适应退火与泛化边界防御 (Adaptive Annealing & Generalization Defense)：

    系统深度集成了 ReduceLROnPlateau 调度器。当模型在错误率曲面上陷入局部鞍点时，调度器会自动进行学习率减半退火（factor=0.5）。配合严密的验证集状态监控与最优权重快照（best_model.pt）回滚机制，完美阻断了模型由于过度拟合噪声数据而导致的泛化能力坍塌。

六、应用说明​

Language Transformer 架构凭借其全局注意力机制，在时序神经解码领域展现出了降维打击般的应用潜力：

  • 打破时序对齐桎梏的自然语言生成 (Alignment-Free Natural Language Generation)

    传统的解码系统要求数据具备精确的“帧-音素”级标签，而本模型利用 CTC 的边缘化能力与 Transformer 的全局视野，允许直接使用整句文本作为目标标签。这一特性极大降低了神经数据标注的成本，使得“从大脑意图直接生成自然语言”的实时语音 BCI 成为现实。

  • 高保真特征的全局溯源 (Global Traceability of High-Fidelity Features)

    相较于 LSTM 的隐性记忆流转，Transformer 的多头注意力权重（Attention Weights）具有极高的可解释性。它能够自动过滤掉神经元群体发放中的瞬时噪声（赋予极低权重），并在极长的时间跨度内将高权重的注意力聚焦于具有真实语义关联的发放簇上，实现高频电生理信号到文字的精准映射。

  • 应对非平稳域偏移的深层鲁棒性 (Deep Robustness against Non-stationary Domain Shifts)

    脑机接口系统面临的最大梦魇是信号分布的长期漂移。通过灵活调配参数（如跨天场景下显式关闭 use_day_idx 并调整 kernel_len），研究者可以利用 Transformer 极强的全局抽象能力，强迫网络学习真正的“神经语义不变量”，从而实现跨天（Cross-day）长时程解码性能的稳定维持。
