3_LFADS

一、模型简介​

LFADS 是一种专为神经科学数据设计的变分序列自编码器（Variational Sequential Autoencoder）。在脑机接口（BCI）与计算神经科学领域，单次试验（Single-trial）的神经脉冲数据往往伴随极高的泊松噪声。LFADS 突破了传统高斯平滑或试次平均（Trial-averaging）方法的局限，其核心假设在于：复杂的神经群体活动是由一个低维的、非线性的内在动力学系统所驱动的。

从理论机制来看，LFADS 的核心架构由以下五个关键部分构成：

  1. 变分序列自编码架构（Variational Sequential Autoencoder）

    LFADS 将循环神经网络（RNN）强大的时序建模能力与变分推断（Variational Inference）的概率框架进行深度融合。它不再输出单一的确定性轨迹，而是学习隐变量的后验概率分布，从而在极高噪声的单脉冲数据中实现了前所未有的鲁棒性与泛化能力。

  2. 初始条件推断（Initial Condition Inference）

    在每次运动或认知任务的起点，大脑的动力学系统均处于特定的初始状态。LFADS 利用一个反向运行的 RNN 编码器（IC Encoder），通过读取整段神经序列的历史信息，精准推断出驱动当前试次动力学演化的初始条件（Initial Condition）及其分布概率。

  3. 自主动力学生成器（Autonomous Dynamical Generator）

    作为 LFADS 的核心状态转移模块，在接收到初始条件后，生成器（通常为多层 RNN）会根据自身学习到的非线性动力学规则在时间尺度上自主向前演化。这一过程有效刻画了运动皮层在执行动作时所表现出的内在低维流形（Manifold）特性。

  4. 外部扰动推断与控制器（Inferred Inputs & Controller）

    为应对大脑活动中不可预见的外部刺激或未被记录脑区的反馈输入，LFADS 创新性地引入了控制器输入编码器（CI Encoder）与 RNN 控制器（Controller）。这二者在每个时间步动态推断外部扰动，并将其作为时变输入（Time-varying inputs）实时注入生成器，从而显著提升了模型对非自主性突发神经活动的拟合能力。

  5. 泊松发射与发放率重构（Poisson Emission & Firing Rate Reconstruction）

    LFADS 的最终输出并非直接拟合离散的脉冲序列，而是将生成器的高维状态映射为低维的潜因子（Latent Factors），随后通过指数映射（以确保非负性）还原出底层真实且平滑的神经元发放率（Firing Rates）。模型采用泊松负对数似然作为重构目标，严格契合了神经脉冲发放的统计物理规律。

二、数据预处理与特征工程​

LFADS 作为一种基于无监督学习的生成式模型，其输入数据的预处理需严格遵循神经物理学规范：

  • 脉冲计数张量化（Spike Binning & Tensorization）：

    输入数据必须为离散的脉冲计数（Spike Counts），而非经过平滑处理的连续信号。通常需将神经元发放的时间序列划分至固定的时间窗口内，从而构建出包含试次、时间步与神经元通道数的三维张量。

  • 协调丢弃（Coordinated Dropout）：

    此为 LFADS 保障泛化能力的核心特征工程机制。在训练阶段，模型会随机遮挡输入张量中部分神经元的脉冲数据，强制生成器仅依赖其余神经元的群体动力学（Population Dynamics）来推断被遮挡神经元的发放率。此举有效避免了自编码器退化为缺乏实际意义的恒等映射（Identity Mapping）。

  • 试次对齐（Trial Alignment）：

    为确保“初始条件推断”能够有效捕获跨试次的变异性，通常需要将连续的脑电或脉冲信号依据特定的行为事件（如运动起始、视觉刺激呈现）进行严格的时间尺度对齐与切分。

四、调用示例​


import numpy as np
import bcijelly

def test_single_day() -> None:
    '''
        单天任务示例。
        完整的使用流程包括：
        1. 引用模型模块，例如 bcijelly.model.lfads.classifier。模块的选择决定了使用哪个模型，以及是进行分类任务还是回归任务（目前不支持回归任务）。
        2. 调用 bcijelly.load_data 加载数据，得到 loaded_data。
        3. 调用模型模块的 train 函数进行训练，得到 train_result。
        4. 调用模型模块的 test 函数进行测试，得到 test_result。
        5. 调用 bcijelly.summary 对结果进行总结和展示。
    '''

    import bcijelly.model.lfads.classifier as module      # 选择模型，设置分类/回归任务（暂不支持回归任务）

    loaded_data = bcijelly.load_data(
        dataset_name="1_FALCONM1-A",    # 选择数据集
        protocol="single_day",                  # "single_day" | "cross_day"，此处演示单天解码
        subject_index=0,                        # 范围: [0, n_subjects - 1]
        day_index=0,                            # 范围: [0, n_days - 1]
        seed=42,                                # int，单天解码中用于设置划分测试集和验证集的随机种子
        single_day_test_size=0.2,               # 范围: (0, 1)，表示测试集占单日数据的比例
        single_day_val_ratio=0.125,             # 范围: [0, 1)，仅当 val_dataset 不是 None 时有效，表示 val 占 train+val 的比例
        # auto_download_missing=True,           # 设置本地缺数据时自动从Hugging Face下载，默认为True
        verbose=False,
    ）

    train_result = module.train(
        train_dataset=loaded_data.train_dataset,
        val_dataset=loaded_data.val_dataset,    # loaded_data.val_dataset 或 None
        seed=42,  # int
        device="auto",
        run_name="demo_single_day_lfads", # 用于结果目录命名
        epochs=1,
        plot=False,
        verbose=False,
    )

    test_result = module.test(
        test_dataset=loaded_data.test_dataset,
        verbose=False,
    )

    bcijelly.summary(
        loaded_data=loaded_data,                # bj.load_data 的返回结果
        Module=module,
        train_result=train_result,              # module.train 返回结果
        test_result=test_result,                # module.test 返回结果
        mode="full",                            # "compact" | "full"，控制 summary 展示的详细程度
    )

def test_cross_day() -> None:
    '''
        跨天任务示例。
        在测试代码部分，我们演示了两种计算测试结果的方式：
        1. 将所有测试日的数据合在一起进行测试，得到一个整体的测试结果。
        2. 对每一个测试日的数据单独进行测试，得到每个测试日的测试结果，然后计算这些测试结果的平均值作为整体的测试结果。
        用户根据自己的需求选择其中一种方式。
    '''

    import bcijelly.model.lfads.classifier as module      # 选择模型，设置分类/回归任务（暂不支持回归任务）

    loaded_data = bcijelly.load_data(
        dataset_name="1_FALCONM1-A",
        protocol="cross_day",                   # "single_day" | "cross_day"，此处演示跨天解码
        subject_index=0,                        # 范围: [0, n_subjects - 1]
        # auto_download_missing=True,           # 设置本地缺数据时自动从Hugging Face下载，默认为True
        verbose=False,
    )

    train_result = module.train(
        train_dataset=loaded_data.train_dataset,
        val_dataset=loaded_data.val_dataset,    # loaded_data.val_dataset 或 None
        seed=42,  # int
        device="auto",
        run_name="demo_cross_day_lfads",  # 用于结果目录命名
        epochs=1,
        plot=False,
        verbose=False,
    )

    # 所有测试日共同测试
    test_result = module.test(
         test_dataset=loaded_data.test_dataset,
         verbose=False,
     )

    # 每一个测试日单独测试
    day_acc_list: list[float] = []
    for one_day_test_dataset in loaded_data.test_day_datasets:
        one_day_result = module.test(
            test_dataset=one_day_test_dataset,
            verbose=False,
        )
        day_acc_list.append(float(one_day_result["acc"]))

    test_result = {
        "acc": float(np.mean(day_acc_list)),
        "per_day_acc": day_acc_list,
    }

    bcijelly.summary(
        loaded_data=loaded_data,                # bj.load_data 的返回结果
        Module=module,
        train_result=train_result,              # module.train 返回结果
        test_result=test_result,                # module.test 返回结果
        mode="full",                            # "compact" | "full"
    )

if __name__ == "__main__":
    test_single_day()
    test_cross_day()

五、模型调试与评估​

在神经科学领域，LFADS 的性能评估无法单纯依赖传统的均方误差或交叉熵，而必须紧密结合其泊松发射特性：

  • 单脉冲比特数（Bits Per Spike, BPS）：

    这是评估生成模型在脉冲序列处理上表现的黄金标准。基于泊松负对数似然，BPS 衡量了 LFADS 重构的瞬时发放率相较于基线模型所能提供的额外信息量增益。BPS 值越高，表明模型捕获的神经动力学细节越为丰富。

  • PSTH 重构保真度（PSTH Reconstruction Fidelity）：

    在验证模型去噪有效性时，通常将模型推断的单次试验发放率进行平均处理，并与经验上的峰值放电时间直方图（Peri-Stimulus Time Histogram, PSTH）计算决定系数。优秀的 LFADS 模型能够在保持单试次变异性的前提下，高保真地还原 PSTH。

  • 后验方差监控（Posterior Variance Monitoring）：

    在模型调试过程中，必须密切监控初始条件和控制器输出的后验方差。若方差坍缩至人为设定的下限，则表明模型未能有效利用变分机制学习数据的不确定性，此时需及时调整 KL 退火策略。

六、应用说明​

依托上述五大核心机制，LFADS 成功破局了现代 BCI 领域内数个棘手的理论与工程难题：

  • 实现高保真“单次试验（Single-trial）”闭环控制

    现实环境下的 BCI 必须基于单次大脑意图做出即时响应。LFADS 利用“自主动力学生成器”与“控制器”的协同作用，在完全不依赖历史试次平均的条件下，从单次高噪脉冲中精准提取平滑发放率，从而大幅提升了实时意图解码的信噪比。

  • 流形对齐与 BCI 长期稳定化（Stabilizing BCIs）

    植入式微电极阵列记录的神经信号会随时间推移发生严重的非平稳漂移（如电极微动、胶质细胞包裹等）。鉴于 LFADS 提取的“潜因子（Latent Factors）”真实表征了神经群体固有的低维动力学流形（Manifold），研究人员可以通过对齐跨天的潜因子空间，确保下游 BCI 解码器在长达数月的时间跨度内维持极高且稳定的性能，彻底免除了频繁且昂贵的模型重新校准成本。

  • 解译未被记录的脑区干扰

    运动皮层在运作时常受视觉反馈、触觉反馈或其他脑区自发活动的干扰。LFADS 的“外部扰动推断与控制器”机制不仅有效提升了信号重构精度，更为神经科学家提供了一种逆向分析并推断这些未被直接观测到的外部输入信号的方法，为探究全脑网络连通性提供了极具价值的分析工具。

  • 离线动力学溯源与微延迟块解码 (Offline Dynamics Tracing & Delayed Block Decoding)

    从计算范式来看，LFADS 的“初始条件推断”强依赖于双向循环神经网络对整段历史与未来序列的通读，以此实现极高保真度的神经流形平滑。这种非因果（Non-causal）特性决定了它目前最核心的战场在于离线（Offline）的单次试验神经动力学分析与科学发现。若需将其部署于实时 BCI 闭环系统中，则必须做出工程妥协，采用带有特定计算延迟的块解码（Block Decoding）策略，在平滑保真度与控制延迟之间寻找最佳的平衡。
