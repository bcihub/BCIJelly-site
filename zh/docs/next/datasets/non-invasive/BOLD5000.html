1_BOLD5000

数据集链接

BOLD5000

一、项目背景

本项目旨在对 BOLD5000 数据集进行标准化预处理，提取 fMRI 特征并进行维度转换，为后续视觉认知相关的模型训练（如图像分类、脑活动预测等）提供高质量输入特征及对应标签。

二、数据处理

1. fMRI 时间序列提取​

直接使用数据集release 2.0 版本（BOLD5000 Release 2.0）中提供的经过 GLM 分析后的 ROIs 所有体素的时间序列数据作为 fMRI 特征（文件夹BOLD5000_GLMsingle_ROI_betas）

该数据已实现如下过程：【见文章 BOLD5000 A public fMRI dataset of 5000 images】

  1. 预处理（Preprocessing）

    所有原始 fMRI 数据已使用 fMRIPREP 1.1.4 完成标准化流程，包括：

    • 运动校正、切片时间校正、几何畸变校正
    • 功能-结构配准、MNI152NLin2009cAsym 空间标准化
    • 9 个扰动变量（6 个头动参数 + CSF/WM/全局信号）回归
    • 128 s 高通滤波

  2. 功能定位器（Functional Localizer）分析

    • 在 8（或 6）个 localizer run 上，使用 SPM12 建立 GLM 并运行以下对比：
      • Scene > Object & Scrambled → PPA、RSC、OPA
      • Object > Scrambled → LOC
      • Scrambled > Baseline → EarlyVis
    • 对显著体素进行 FWE 校正（p < 0.0001, k ≥ 30），获得个体空间的二值化 ROI 掩膜，随后重采样到 MNI 体素网格。
    • 时间窗取平均（基于ROI信号激活峰值情况）
      • CSI1-3：TR3 + TR4（4–8 s）平均
      • CSI4：仅 TR3（4–6 s）
    • 信号提取：每个 ROI 内所有体素的时间序列，去均值化（demeaned）。

  3. 主实验单 trial GLM 建模

    • 在已预处理的主实验数据上，为 每张刺激图片 逐一建立 GLM-single-trial（LS-S）模型：
      • 每幅刺激图片对应一个回归量（stick function，1 s ON、9 s OFF）
      • 再次包含相同的 9 个扰动回归量、run 回归量和 128 s 高通滤波
    • 对每个 ROI 掩膜内的体素，求解得到该图片在该体素上的 β 系数（事件相关幅值）。

2. 标签（基于events.tsv）​

标签直接从数据集中的event.csv文件读取即可。

每个 fMRI run 都有对应的 events.tsv 文件，字段包括：

  • onset: 图像出现的时间点
  • duration: 通常为 1s（图像呈现时间）
  • trial_type: 固定为 stimulus
  • stimulus: 图像文件名
  • ImgID: 图像唯一 ID（用于配对图像特征）
  • ImgType: 图像类型（ImageNet、coco、scenes）

标签处理方法如下：

  1. 从event.csv文件中提取图像类型标签（ImgType）并进行标准化处理，标准化指将rep类（重复出现）和非rep归为一类。
  2. 将字符串格式的标签转换为机器学习模型可接受的数值格式

3. 1D 时间序列转 2D 特征（基于 GAF 方法）​

整体流程（参考文献 [1]）：​

  1. 时间序列提取 从每个 fMRI 轮次（run）和每个活跃体素中提取完整的时间序列，该序列代表 BOLD 信号强度随时间的分布。由于每个轮次包含 37 个视觉刺激（图像），时间序列长度为 37。

  2. 时间序列分割 根据试次中呈现的图像所属数据集（COCO、ImageNet、SUN），将完整时间序列分割为三部分，分别对应观看三类图像时的 BOLD 信号。

  3. 预处理 对分割后的体素特异性时间序列进行线性插值处理，将其统一调整为相同的时间长度，以满足后续分析的需求。

  4. 1D 时间序列到 2D 空间特征的转换

    通过 Gramian Angular Field（GAF）将 1D 时间序列转换为 2D 矩阵，以捕获时间序列中的空间关联特征。

GAF核心原理​

Gramian Angular Field（GAF）：通过极坐标转换，将归一化后的时间序列映射为 2D 矩阵，生成 Gramian Angular Summation Field（GASF）和 Gramian Angular Difference Field（GADF），保留时间依赖性且无信息丢失。

具体步骤​

（1）数据归一化​

将 1D 时间序列（长度为 T）归一化至 [-1, 1]范围，公式：

python运行

x_norm = 2 * (x - min(x)) / (max(x) - min(x)) - 1

其中 x为原始时间序列，x_norm为归一化结果。

（2）极坐标转换​

将归一化序列映射为极坐标 (r, θ)：

  • 角度 θ：由 x_norm通过 θ = arccos(x_norm)计算（因 x_norm ∈ [-1, 1]，θ ∈ [0, π]）；
  • 半径 r：由时间步长归一化得到，r = t / T（t为时间索引，t ∈ [0, T-1]）。
（3）GASF 与 GADF 计算​
  • GASF：GASF[i, j] = cos(θ_i + θ_j)
  • GADF：GADF[i, j] = sin(θ_i - θ_j)

其中 i, j为矩阵索引，θ_i、θ_j分别为第 i和第 j个时间步的角度。

三、主要函数说明

mark_img函数的主要功能是将时序数据转换为适用于卷积神经网络（CNN）输入的图像数据，具体说明如下：

def mark_img(dat):
  gram = GramianAngularField(image_size=16, method='summation')##change method='difference' for GramianAngularField difference
  gram_t = gram.fit_transform(dat.iloc[:,:-1])
  x = gram_t.reshape(gram_t.shape[0], 16, 16,1)
  y=dat['label']
  return x,y

功能解析​

  1. 时序转图像的核心工具使用 pyts.image库中的 GramianAngularField（格拉姆角场）算法，将时序数据转换为图像。这是处理时序数据的一种常用技巧，能保留时序数据的时间相关性和幅值信息。
  2. 参数说明
    • GramianAngularField(image_size=16, method='summation')：生成16x16大小的图像，转换方法为 summation（求和），也可改为 difference（差分）。
    • dat：输入的DataFrame，其中最后一列是标签（label），其余列是时序特征数据。
  • 参数 16为文章指定，基本依据数据集对应的实验设计本身。
  1. 数据处理步骤
    • 提取输入数据中除最后一列（标签列）之外的时序特征（dat.iloc[:,:-1]）。
    • 通过格拉姆角场转换得到图像数据 gram_t，形状为 (样本数, 16, 16)。
    • 将图像数据重塑为CNN要求的输入格式 (样本数, 16, 16, 1)（最后一个维度为通道数，这里是单通道灰度图）。
    • 提取标签列 dat['label']作为目标变量 y。
  2. 返回值
    • x：转换后的图像数据（形状为 (n_samples, 16, 16, 1)）。
    • y：对应的标签数据，用于模型训练的目标值。

该函数实现了从一维时序到二维图像的转换，为后续的深度学习分类任务做准备。

四、使用说明

数据以被试为单位进行存储（支持直接使用单个被试数据进行训练）。

使用方法：

  1. 直接通过 np.load 函数加载 .npz 格式文件
  2. 加载后可通过 fmri 和 label 两个关键字获取对应数据：
    • fmri：fMRI 二维图像数据，形状为 (trials, height, weight)，其中 height 和 weight 均为 16
    • label：标签数据，包含 0、1、2 三种取值，分别对应：
      • 0：coco 类别
      • 1：imagenet 类别
      • 2：scenes 类别 三类标签的数量比例为 2:2:1

五、注意事项

当前数据是基于ROIs（感兴趣区域）内所有体素的信号（参考文章[1]）。在实际分析中，除了使用全部体素信号外，也可考虑对每个ROI单独进行降维处理后再开展后续分析工作。

Sinha, "Spatial Encoding of BOLD fMRI Time Series for Categorizing Static Images Across Visual Datasets: A Pilot Study on Human Vision," TENCON 2023 - 2023 IEEE Region 10 Conference (TENCON), Chiang Mai, Thailand, 2023, pp. 1117-1122, doi: 10.1109/TENCON58879.2023.10322476.
