2_GOD

数据集链接

Generic Object Decoding (fMRI on ImageNet) - OpenNeuro

一、数据集的简单说明

基于分类任务的需求，我们仅使用 perceptionTraining 子集（150 类 × 8 张 = 1 200 张自然图像）。该子集已在原始论文中完成了头动校正、空间配准及全局归一化等标准 fMRI 预处理，可用于后续训练-测试划分。

二、数据处理思路

1. ROI 信号提取（基于已有数据与 mask 文件）​

  1. 输入• 预处理后的 4-D fMRI 时间序列（func.nii.gz），位于文件夹 derivatives下• 预定义的 ROI mask（V1.nii.gz, LOC.nii.gz, FFA.nii.gz, PPA.nii.gz 等，二值掩模）

  2. 流程

    import nibabel as nib, numpy as np
    img   = nib.load('func.nii.gz').get_fdata()      # (X,Y,Z,T)
    mask  = nib.load('V1.nii.gz').get_fdata().astype(bool)
    roi_t = img[mask, :]                             # (N_voxel, T)

  3. 输出 • 每个 ROI 的二维矩阵：roi_t ( 体素数 × TR 数)。

2. 标签匹配（时间对齐）​

  1. 对齐逻辑• fMRI 采样周期 TR = 3 s；HRF 峰值滞后 ≈ 4–6 s。• 将刺激 onset 时刻 向后移动 1 个 TR（3 s） 后取对应 TR 的信号作为该刺激样本。
  2. 标签表 • CSV： onset, category_index • 通过 onset 与 TR 索引建立一一映射。

三、fMRI数据处理完整流程

一、数据预处理流程​

1. 数据加载与准备​

  • 原始数据格式：fMRI数据以 .nii.gz格式存储，事件标记以 .tsv格式存储，ROI掩码以 .nii.gz格式存储
  • 数据组织：按被试(subject)、会话(session)和运行(run)层级组织
  • 关键参数：
    • TR(重复时间)：3.0秒
    • 刺激时长：9.0秒
    • 每个试次的TR数量：3个(9.0/3.0)

2. 文件匹配与索引​

通过正则表达式匹配不同类型的文件：

  • fMRI数据文件：(sub-\d+)_ses-(perceptionTraining\d+)_task-perception_run-(\d+)_bold_preproc\.nii\.gz
  • 事件标记文件：(sub-\d+)_ses-(perceptionTraining\d+)_task-perception_run-(\d+)_events\.tsv
  • ROI掩码文件：(sub-\d+)_mask_(.+)\.nii\.gz

构建数据索引字典，建立被试与对应数据文件的映射关系。

3. ROI与体素处理​

  1. 加载每个被试的ROI掩码，进行二值化处理
  2. 统计每个ROI的体素数量，计算总体现素数
  3. 按固定顺序保存ROI名称及对应体素数量，确保数据一致性

4. 试次数据提取​

  1. 加载fMRI数据(形状为[x, y, z, t])和事件文件
  2. 根据事件文件中的 onset时间确定每个试次对应的TR索引
  3. 提取每个试次中所有ROI的体素信号：
    • 按ROI顺序拼接体素信号
    • 形成形状为[总体素数, TRs_per_trial]的试次数据
  4. 对试次标签进行编码，统一转换为0开始的整数编码

二、特征工程流程​

1. 数据标准化​

  • 使用 StandardScaler对数据进行标准化处理
  • 处理对象：每个时间点(TR)的所有试次数据
  • 标准化目标：均值为0，标准差为1
  • 形状变化：[Trials, 总体素数] → [Trials, 总体素数]

2. PCA降维​

  • 使用 sklearn.decomposition.PCA进行主成分分析
  • 核心参数：n_components(主成分数量，可根据需求调整)
  • 降维策略：对每个时间点(TR)分别处理
  • 形状变化：[Trials, 总体素数] → [Trials, 主成分数]

3. 特征矩阵构建​

  • 整合所有时间点的降维结果
  • 最终特征矩阵形状：(样本数 × 主成分数 × TR数)
  • 保留PCA模型参数(成分矩阵和解释方差比例)

四、使用说明

数据以被试为单位进行存储。

  1. 直接通过 np.load 函数加载 .npz 格式文件。
  2. 加载后可通过以下关键字获取对应数据：
    • fmri：fMRI 数据形状为 (trials, 主成分数, TRs)。
    • label：标签数据，包含 0~149 共计 150 种取值，分别对应不同的类别标签。
    • 额外扩展关键字：
      • label_names：获取标签对应的具体名称信息。
      • rois：获取感兴趣区域（ROIs）的相关信息。
      • roi_voxel_counts：获取每个感兴趣区域内的体素数量信息。

注意事项

  • 可根据实际需求调整 n_components参数，以平衡降维效果与计算资源消耗。本研究上传的数据已降维至1000维，该维度是目前实验中取得最优分类效果的设置。
  • 针对数据增强，可尝试按类别进行处理，但实验结果并未显示提升分类效果，因此最终未纳入流程中。
