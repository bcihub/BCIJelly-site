5_Jongo

iBCI 数据预处理与滑窗数据集构建

数据来源：Using adversarial networks to extend brain computer interface decoding accuracy over time（Dryad 数据集）。包含多只猴不同任务（ISO/CO/RT/PG），Blackrock 96 通道 Utah 阵列阈值过零脉冲，多单元（threshold crossings）。

1. 目标​

将 .mat 会话文件中 spike counts（1 ms 采样） 与 行为学信号（力/速度/位置等）：

  1. 统一按固定 bin 大小（如 20 ms）降采样；
  2. 按 trial 切分、过滤成功试次；
  3. 针对回归/分类任务构建 滑动时间窗样本；
  4. 导出 features, labels 的 .npz 以供下游模型训练。

2. 任务与变量​

  • 任务范式：
    • ISO（等长力控）：屏幕上光标由腕部力控制，8 方向随机外靶；
    • CO（Center-out 伸手）：操纵平面设备到 8 个外靶；
    • RT（Random-target）：随机序列目标；
    • PG（Power/Key grasp）：抓握力控，3 个矩形目标。
  • 通道：M1 96 通道，多单元阈值过零；
  • 脉冲：spike_counts，单位 1ms bin 的脉冲计数（channel × time）；
  • 行为：force/position/EMG 等（本脚本示例使用 force → 2D）。

3. 目录与文件​

  • 压缩包示例：Jango_ISO_2015.7z 解压后得到若干 .mat 文件。
  • 每个 .mat 文件下的关键键值（以 xds 组为例）：
    • trial_result：试次状态码（82 表成功）；
    • time_frame：毫秒时间戳；
    • spike_counts：通道×时间的脉冲计数（1 ms）；
    • force：2×time 力信号（本脚本示例作为行为）；
    • trial_start_time / trial_end_time：试次起止时间；
    • trial_target_dir：试次目标方向（单位度，-45）。

不同任务/猴的行为变量名可能不同，请按 README 映射至需要的行为（如 position、velocity 或 FSR 力）。

4. 处理流程概览​

  1. 枚举 `` 文件 → 用 h5py 读取 xds/* 键值；
  2. 筛选成功试次（trial_result == 82）；
  3. 按试次时间戳切片 spike_counts 与行为信号；
  4. 时间对齐降采样（binning）：
    • 脉冲：bin_spikes_for_regression(spikes, bin_size) → 每通道在每个 bin 的计数；
    • 行为：bin_behavior_for_regression(behavior, bin_size) → 每个 bin 行为均值；
  5. 方向标签规范化：将 {0,45,90,135,180,-135,-90,-45} 映射到 {0..7}；
  6. 滑窗构建样本 create_windowed_dataset(data, label, trial_labels, window, stride)：
    • window（单位=bin 数）与 stride 控制时间窗长度与步幅；
    • 输出：features [N, window, C]，labels [N, window, F]，trial_id/dir；
  7. 保存：每个会话/文件保存为 filename.npz，含 features、labels。

5. 核心函数说明​

5.1 bin_spikes_for_regression(spikes, bin_size)​

  • 输入：spikes 为 [channels, time_steps]（注意脚本内先行按此形状读取），bin_size 为 ms 数（如 20）。
  • 处理：按 expect_num_bins = time_steps // bin_size 取整分箱，舍弃尾部不足一 bin；对每个 bin 内进行 求和 得到脉冲计数；
  • 输出：[expect_num_bins, channels]（时间维在前）。

若输入为 [time, channel]，请在调用前转置为 [channel, time]。

5.2 bin_behavior_for_regression(behavior, bin_size)​

  • 输入：behavior [time_steps, features]；
  • 处理：按相同 bin_size 取均值；
  • 输出：[expect_num_bins, features]。

5.3 create_windowed_dataset(data, label, trial_labels, window_size, stride)​

  • 输入：
    • data：按 trial 的脉冲序列列表（每项 [T, C]）；
    • label：按 trial 的行为序列列表（每项 [T, F]）；
    • trial_labels：每个 trial 的离散标签（如 8 方向中的某一类，整型）；
    • window_size：窗长（单位=bin）；stride：步幅（单位=bin）。
  • 输出：
    • features [N, window, C]
    • labels [N, window, F]
    • trial_meta [N]（与窗一一对应的 trial 标签）
  • 注意：丢弃长度 < window_size 的 trial；采用 包含式滑窗（start=0..T-window step=stride）。

5.4 nprmsce(data_root, bin_size=20, window=30, step=6)​

  • 作用：批处理目录下所有 .mat，完成 trial 切片、binning、滑窗、保存；
  • 默认参数：
    • bin_size=20 ms（即 50 Hz）；
    • window=30 bin（约 600 ms）；
    • step=6 bin（约 120 ms） → 80% overlap；
  • 输出文件：与输入 .mat 同名的 .npz，字段：features, labels。

6. 输出数据格式​

  • features: np.ndarray，形状 [N, W, C]
    • N：滑窗样本数；W：窗内时间步（bin 数）；C：神经通道数；
  • labels: np.ndarray，形状 [N, W, F]
    • F：行为学特征维度（示例为 2：force_x, force_y）。

若用于分类（如 8 方向），可基于 trial_meta/trial_target_dir 在窗级别派生标签（例如用窗中心或多数投票）。当前脚本保存的是回归标签（逐时间步的行为）。

7. 使用示例​

# 1) 解压 Jango_ISO_2015.7z 至 ./Jango_ISO_2015
# 2) 运行脚本（Python 3.9+）
python main.py  # 内含 nprmsce 调用
# 3) 生成 <session>.npz
import numpy as np
Z = np.load('Jango_ISO_2015_sessionX.npz')
X, Y = Z['features'], Z['labels']  # X:[N,W,C], Y:[N,W,F]

8. 关键超参数建议​

  • bin_size：10–50 ms（与行为频带匹配；20 ms 常用）
  • window：300–1000 ms（依据任务动态；CO/ISO 可 500–800 ms）
  • stride：窗口的 10–50%（控制样本相关性与规模）
  • 行为选择：
    • ISO：force（2D）；
    • CO/RT：position/velocity（2D）；
    • PG：FSR 力（1D 或 2D 差分/和）。

9. 常见坑位与对策​

  • 形状对不齐：spike_counts 读出为 [C, T]；行为需转为 [T, F] 后再 bin；
  • 尾部舍弃：binning 取整会丢弃不足一 bin 的尾巴 → 若需保留可改为填充或可变窗；
  • 试次索引：用 time_frame 掩码切片，务必用 indices[0] : indices[-1] + 1；
  • 任务差异：不同任务的行为键不同，需在读取层兼容；
  • 标签映射：方向角度必须完整映射到 {0..7}；异常角度需跳过或重映射；
  • 数据类型：大数组建议 int16/float32，避免 int8 溢出（视计数大小而定）。

10. 扩展与下游​

  • 回归：解码 2D 力/速度/位置（R²、RMSE 指标）；
  • 分类：8 方向离散意图（准确率、F1）；
  • 跨日对齐：可进一步接入 CycleGAN/ADAN/Procrustes 等对齐方法以提升跨时稳健性；
  • 深度模型输入：将 [N, W, C] 转换为 CNN/RNN/Transformer 所需形状（如 [N, C, W]）。

11. 依赖​

  • numpy, h5py, scipy, matplotlib, pickle（可选）

12. 引用与致谢​

  • Dataset: Ma et al., 2023, Dryad. Using adversarial networks to extend brain computer interface decoding accuracy over time.
  • Hardware/Preprocessing：Blackrock Neurotech；阈值设定（-4.75x ~ -6.25x RMS）；50 ms 非重叠窗 + 100 ms 高斯平滑（参考 README）。
