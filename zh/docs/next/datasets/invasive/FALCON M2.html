2_FALCON M2

数据处理流程说明

数据集链接​

https://dandiarchive.org/dandiset/000953?search=000953&pos=1

一、项目背景​

本项目旨在处理 FALCON Benchmark M2: primary motor cortex recordings in primate during finger movement 实验中的神经数据（.nwb 文件），实现从神经元 spike 时间数据到行为轨迹标签的同步数据预处理，为后续机器学习模型训练提供输入特征和标签。

二、核心功能​

  • 读取 .nwb 文件中的神经元 spike 时间点和连续行为数据（手指位置与速度）
  • 构建 trial 级别的脉冲矩阵
  • 以固定时间窗对 spike 数据进行降采样（binning），与行为数据频率对齐
  • 使用滑动窗口切分数据，形成可供模型训练的输入输出样本集
  • 保存处理结果为 .npy 文件
  • 绘制部分 trial 的脉冲 raster 图，辅助数据质量检验

三、主要函数说明​

1. bin_spikes_for_regression(spikes, bin_size)​

  • 功能：将原始 spike 二值矩阵按时间窗 bin_size 降采样，统计每个 bin 内脉冲计数。
  • 输入：
    • spikes：numpy.ndarray，形状 [time_steps, channels]
    • bin_size：int，时间窗口大小（单位为采样点数）
  • 输出：
    • binned_spikes：numpy.ndarray，形状 [expect_num_bins, channels]，每个 bin 内各通道脉冲计数

2. create_windowed_dataset_for_regression(data, label, window_size, stride)​

  • 功能：将 bin 后的脉冲数据及对应行为标签，按滑动窗口切分成训练样本。
  • 输入：
    • data：numpy.ndarray，形状 [num_trials, time, channels]
    • label：numpy.ndarray，形状 [num_trials, time, behaviour_dim]
    • window_size：int，滑动窗口长度
    • stride：int，滑动步长
  • 输出：
    • all_features_np：切分后的特征样本，形状 [num_samples, window_size, channels]
    • all_labels_np：对应标签，形状 [num_samples, window_size, behaviour_dim]

3. falconm2(data_root, save_root, window_size, stride)​

  • 功能：
    • 遍历指定目录下所有 .nwb 文件
    • 读取神经脉冲时间点和行为数据
    • 构建 trial 脉冲矩阵，执行 20ms binning
    • 利用滑动窗口切分数据样本
    • 保存处理好的脉冲和标签数据
    • 绘制部分 trial raster 图，保存为图片
  • 参数：
    • data_root：原始 .nwb 文件目录
    • save_root：处理结果保存目录
    • window_size：滑动窗口长度（单位时间步）
    • stride：滑动步长（单位时间步）

四、使用说明​

  1. 准备数据 将 MonkeyN-held-in-calib 的所有 .nwb 文件放置于 data_root 指定目录下。

  2. 运行脚本 在 Python 环境中执行：

    python falconm2(
        data_root=r'..\data\sub-MonkeyN-held-in-calib',
        save_root=r'..\out\sub-MonkeyN-held-in-calib',
        window_size=20,
        stride=10
    )

  3. 查看结果

    • 脉冲特征保存路径示例： ..\out\sub-MonkeyN-held-in-calib\20201019Run1\spikes_20201019Run1.npy
    • 标签保存路径示例： ..\out\sub-MonkeyN-held-in-calib\20201019Run1\reg_labels_20201019Run1.npy
    • Raster 图保存路径示例： ..\out\sub-MonkeyN-held-in-calib\20201019Run1\trial_0_raster.png

  4. 数据加载示例

    import numpy as np

    features = np.load(r'..\out\sub-MonkeyN-held-in-calib\20201019Run1\spikes_20201019Run1.npy')
    labels = np.load(r'..\out\sub-MonkeyN-held-in-calib\20201019Run1\reg_labels_20201019Run1.npy')

    print(features.shape, labels.shape)

  5. 最终数据集形状说明

    经过完整预处理流程（包括 spike binning 和滑窗切片）后，我们得到两个主要数组：

    1. spikes_*.npy

      • 数据类型：numpy.ndarray

      • 形状：[num_samples, window_size, num_channels]

      • 含义：

        • num_samples：表示被滑动窗口切分出的样本数量
        • window_size：每个样本的时间长度（例如设为 20，对应 400ms）
        • num_channels：神经元通道总数（来自 spike 单元数量，如 64、96 等）

      每一个样本可以看作一个 [window_size, num_channels] 的二维神经活动矩阵，是模型的输入特征。

    2. reg_labels_*.npy

      • 数据类型：numpy.ndarray

      • 形状：[num_samples, window_size, label_dim]

      • 含义：

        • num_samples：与 spikes_*.npy 相同，表示样本数量
        • window_size：与 spikes_*.npy 对齐
        • label_dim：标签维度，通常是 2（代表手指的 (x, y) 位置），或 4（包含位置 + 速度）

      每个标签数组与对应的 spike 数据样本一一对齐，用于训练模型进行轨迹回归。

五、注意事项​

  • 脉冲 binning固定窗口大小为 20ms，可根据需要调整 bin_spikes_for_regression 的 bin_size 参数。
  • 滑动窗口参数 window_size 和 stride 需根据模型输入要求和数据量合理设定。
  • 行为数据采样频率为 50Hz，脉冲时间单位已转换为 ms，确保与行为数据对齐。
  • 部分 trial 长度可能不足滑动窗口长度，代码中会跳过这些 trial。

六、常见问题及支持​

  • 读取 .nwb 文件失败时，请确认文件完整且格式正确。
  • 数据维度不匹配时，请检查行为和脉冲数据是否正确对齐。
  • 若需要增加数据预处理或可视化功能，请联系开发者进行定制。
