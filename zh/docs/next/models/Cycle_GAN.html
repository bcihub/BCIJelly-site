5_Cycle-GAN

一、模型简介​

Cycle-GAN（循环生成对抗网络）在此处被创新性地引入为一种无监督的神经域自适应（Unsupervised Domain Adaptation）对齐器。在脑机接口（BCI）的长期应用中，微电极阵列记录的神经脉冲数据常常因电极微小位移或神经元自然演变而产生特征空间的非平稳漂移。Cycle-GAN 突破了传统必须依赖目标域大量标签重新训练下游解码器的局限，其核心机制在于：通过构建双向映射，将目标日漂移后的神经脉冲特征无缝对齐回源日的特征空间，从而实现预训练解码器的免重训复用。

从理论与代码架构来看，该 Cycle-GAN 对齐系统由以下四个关键模块构成：

  1. 双向神经特征生成器（Bidirectional Feature Generators）

    模型实例化了两个对称的生成器，均采用三层多层感知机架构，并融合了失活与激活机制。生成器负责学习高维神经特征流形之间的非线性映射，将源域特征转换为目标域特征，或将目标域特征还原至源域。

  2. 对抗鉴别器（Adversarial Discriminators）

    系统包含两个独立的鉴别器模块（也是三层结构，输出单节点），分别负责区分真实神经特征与生成器伪造的对齐特征。通过计算对抗损失，鉴别器与生成器在极小极大博弈中驱动对齐特征逼近真实的神经放电分布。

  3. 循环一致性与身份保留约束（Cycle & Identity Constraints）

    为防止生成器在映射高维神经脉冲时丢失核心的运动学或意图信息，模型强制引入了两种关键惩罚机制：

    • 循环一致性损失（Cycle Loss）： 确保特征经过双重映射（从源域到目标域，再回到源域）后，能精确重构原始输入。
    • 身份保留损失（Identity Loss）： 当生成器输入同域特征时，约束其输出保持不变，避免对基底神经信号产生过度扭曲。

  4. 解耦的下游解码器（Decoupled Downstream Decoders）

    系统解除了对齐器与解码器的耦合限制。提供用于拟合连续运动学信号（如力度、肌电图）的回归器（采用平滑损失），以及用于离散动作/状态识别的分类器（采用交叉熵损失，并内嵌梯度裁剪机制以防爆炸）。

二、数据预处理与特征工程​

Cycle-GAN 对齐器对神经物理信号的输入格式有着严格的规范，系统内置了高度定制化的预处理流水线：

  • 张量降维与去零清洗（Tensor Reshaping & Padding Removal）：

    由于下游滤波与解码架构的需求，原始的三维神经脉冲张量（包含试次、时间步与通道）需被展平为二维时序矩阵。同时，系统会自动识别并剔除全零的填充标签，确保对齐器仅学习有效的神经动力学状态。

  • 特征维动态对齐（Feature Dimension Alignment）：

    针对跨天或跨周电极通道数量可能发生变化的情况，系统机制会强制将特征的通道维度补零填充或截断至设定的目标维度，从而保证网络输入的结构一致性。

  • 精密的时间域切分（Temporal Domain Splitting）：

    在构建源域与目标域时，系统执行高度复杂的试次划分。支持将当天特定比例的数据划为训练集、支持聚合多天数据为源域，甚至支持通过精准匹配文件名，提取同一跨度内的时间序列作为跨周训练基准。

三、参数说明​

本文档详细整理了基于 Cycle-GAN 进行跨天/跨周数据对齐，并使用解码网络进行分类与回归的各个模块核心参数。参数提取自项目中的训练接口、宏观调度脚本以及文档说明。

1. 跨天/跨周宏观调度参数 (Global & Scheduling)​

主要用于定义数据集在多天或多周环境下的加载与切分逻辑。

参数名称类型适用脚本详细说明
data_dirstr独立运行脚本数据集的根目录存放位置。
day0_decoder_pathstr独立运行脚本预训练好的源域解码器模型权重路径。
seed_listList[int]独立运行脚本随机种子列表，用于批量跑多次实验消除随机误差。
first_days_of_weekList[int]字典构建脚本记录跨周数据集中“每周第一天”的索引列表，用于自动判定跨周边界。
day_numberint字典构建脚本当前数据的天数索引，用于计算所属的周数。

2. 核心对齐与训练命令行参数 (classifier.py & regressor.py)​

分类器与回归器共享了一套 Cycle-GAN 对齐训练逻辑，但它们的默认超参数针对各自任务进行了不同的适配。

命令行参数 / API 参数类型默认基线值 (回归 / 分类)详细说明与场景覆写规则 (Overrides)
--train_datapathstrNone / None用于训练的数据集文件路径。
--val_datapathstr仅分类特有: None(仅分类) 用于验证的文件路径。
--test_datapathstrNone / None用于测试评估的数据集文件路径。
--enable_alignmentbool-核心对齐开关。注：单天解码 (single-day) 设为 False；跨天解码 (cross-day) 必须设为 True 以激活 CycleGAN。
--batch_sizeint32 / 32训练和验证时的数据批处理大小。单天与跨天场景均维持 32。
--epochsint8 / 400对齐器（生成器和判别器）的训练总轮数。单天与跨天分类基线为 400。注：Jango_force 跨天分类任务覆写为 500。
--decoder_epochsint40 / 100源域数据解码器的训练总轮数。单天与跨天分类基线为 100。注：Jango_force 跨天分类任务覆写为 50。
--decoder_lrfloat0.001 / 0.001解码器的初始学习率。
--g_lrfloat0.0005 / 0.001生成器的学习率。
--d_lrfloat0.0005 / 0.01判别器的学习率。
--source_ratiofloat0.8 / 0.9划分源域训练数据与验证数据的比例。
--identity_weightfloat20.0 / 5.0一致性损失权重。单天与跨天分类基线为 5。注：Jango_force 跨天分类任务增强至 7.5。
--cycle_weightfloat20.0 / 5.0循环一致性损失权重。单天与跨天分类基线为 5。注：Jango_force 跨天分类任务增强至 7.5。
--loss_typestr"L1" / "L1"重建损失的计算方式配置。
--seedint42全局随机种子，默认设为 42 以保证实验可复现性。

3. 校准与早停参数 (Calibration & Early Stopping)​

这些参数主要在分类器中暴露，用于控制测试阶段的对齐校准以及模型早停机制。

命令行参数 / API 参数类型默认基线值详细说明与场景覆写规则 (Overrides)
--val_ratiofloat0.2(仅分类) 未提供验证集路径时，从训练集中切分验证集的比例。单天与跨天均维持 0.2。
--patienceint10(仅分类) 早停耐心值，连续指定轮次无提升则终止。单天与跨天分类均使用 10。
--calib_ratiofloat0.1(仅分类) 测试阶段，划出前一定比例的数据用于校准。基线为 0.1。注：Jango_force 跨天分类覆写为 0.15。
--min_calib_trialsint5(仅分类) 用于校准的最少试次数量。基线为 5。注：Jango_force 跨天分类覆写为 15。
--max_calib_trialsint30(仅分类) 用于校准的最多试次数量，防止校准成本过高。基线为 30。注：Jango_force 跨天分类收紧至 28。

4. 模型架构参数 (Model Architecture)​

用于定义生成器、判别器以及解码器的网络结构和正则化强度。

命令行参数类型默认基线值 (回归 / 分类)详细说明与场景说明
--generator_hiddenint256 / 0生成器的隐藏层维度。分类任务（含单天与跨天解码）默认设为 0，表示自动使用输入特征维度。
--discriminator_hiddenint256 / 0判别器的隐藏层维度。分类任务（含单天与跨天解码）默认设为 0 以自动推断。
--dropout_decoderfloat0.3 / 0.3解码器网络中的失活概率。单天与跨天解码均固定为 0.3。
--dropout_gfloat0.2 / 0.2生成器网络中的失活概率。单天与跨天解码均固定为 0.2。
--dropout_dfloat0.2 / 0.2判别器网络中的失活概率。单天与跨天解码均固定为 0.2。
in_dim / out_dimint-（代码内部参数）分别代表模型的输入特征维度和输出层的维度。

四、调用示例​

五、模型调试与评估​

本系统构建了严密的阶段性验证与指标追踪体系，模型调试的核心聚焦于回归与分类的泛化评价：

  • 回归任务多维评估：

    模型不仅评估传统的平均误差，最关键的核心指标为多输出方差加权决定系数。为了更细腻地剖析二维运动学解码，代码会额外拆分并输出独立坐标轴的逼近度。

  • 分类任务准确率监控：

    评估标准聚焦于严格的分类准确率。对于具有多个离散类别的数据，准确率的提升直接反映了特征流形的有效重合度。

  • 周期性验证与断点持久化：

    对抗训练过程高度不稳，因此系统在每迭代特定轮次后，会将生成器置于验证模式并输入验证集。系统将追踪验证集经过对齐并送入冻结的源域解码器后的表现。表现最顶峰的生成器权重与解码器权重将被保存为独立文件。

  • 全链路日志记录与后处理：

    框架内置日志记录对象，并将逐日的验证结果汇总输出至特定的表格文件中。为适配进一步的数据分析流水线，用户亦可通过脚本，将去除表头的数据矩阵无损映射回张量文件。

六、应用说明​

依托非配对域自适应能力，本 Cycle-GAN 架构精准解决了当前长期植入式 BCI 领域的最痛点问题：

  • 彻底免除逐日昂贵的模型重训成本（Zero-Retraining Reuse）

    受制于大脑物理环境的微观改变，即便最先进的解码器，在模型校准数天后也会出现性能悬崖。本系统巧妙地转变了思路：不修改庞大的下游解码器，而是去修正输入数据本身。通过对齐器将未来的偏移特征拉扯回训练首日的特征空间中，极大延长了单一解码器的生命周期。

  • 破解长期离线数据的标签匮乏困境

    Cycle-GAN 本质属于无监督的生成对抗架构（核心损失的计算无需真实标签的参与）。这意味着，在现实场景下，患者无需每天花费数小时重复进行带标注的校准范式，系统仅依靠患者自由发出的少量无标签脑电序列，即可后台静默完成流形空间的重构与自适应对齐。

  • 泛化适配高度异构的神经任务

    无论是在猕猴数据集上进行的连续二维力度轨迹回归，还是执行的跨周多意图模式分类，该对齐流水线展现出了跨范式的通用性。只需简单替换尾部的回归器或分类器，统一的特征流形对齐器便可无缝驱动各种多元异构下游任务。

  • 离线域自适应与在线解耦运行 (Offline Domain Adaptation & Online Decoupled Execution)

    Cycle-GAN 在系统生态中的占位极为特殊。其极其耗时的生成对抗训练与特征流形对齐过程，通常被严格限制在离线算力集群或跨日盘前校准阶段执行。然而，由于该架构在设计上实现了“对齐生成器”与“下游解码器”的完美物理层面解耦，一旦生成器完成跨日流形映射的权重固化，被实时映射校准的数据流即可零延迟地送入下游网络，从而为复杂的控制任务提供了纯粹、高鲁棒的在线实时闭环运行环境。
