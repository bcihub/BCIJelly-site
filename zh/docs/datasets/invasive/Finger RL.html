3_Finger RL

Finger_RL 数据集包含两名四肢瘫痪的人类参与者在尝试进行单个手指运动时的单神经元记录。

数据集链接：https://dandiarchive.org/dandiset/000252/0.230408.2207​

DOI：https://doi.org/10.48324/dandi.000252/0.230408.2207​

一、实验设计与数据采集​

被试对象及数量​

  • 物种：人类
  • 被试者：2 名四肢瘫痪的参与者

记录范式​

  • 核心脑区：
  • 左侧顶后皮层 (PPC, PC-IP 区)： 负责提取高层级的运动意图、空间映射及认知规划信号。
  • 左侧运动皮层手部区域 (MC hand knob)： 针对性记录控制手部精细动作的初级运动执行信号。
  • 通过植入式电极阵列捕获离散神经元的脉冲 (Spike) 活动。

任务范式​

  • 实验任务：尝试独立手指运动（左右手）

采样率​

  • 30 kHz

二、数据处理​

数据类型​

  • 单神经元脉冲数据、trial 信息、手指标签

数据格式​

  • Neurodata Without Borders（NWB）

数据集结构​

  • 原始数据：../data/sub-P1/*.nwb
  • 输出数据：../out/sub-P1/YYYYMMDD.npz
    • features：[Trial, Time, Channel]，时间维已 binning 为 50
    • labels：每 trial 对应的手指类别

数据处理流程​

本流程旨在将原始的 .nwb 文件解析并重构为适用于运动分类任务的标准化张量数据（.npz 格式）。

1. 核心数据解析 (Core Data Parsing) 遍历所有原始 .nwb 文件，提取解码所需的三大核心数据：

  • 神经脉冲：读取 units['spike_times']（各通道神经元放电时间），并按系统采样率 (FS) 将其转化为采样点索引。
  • 时序边界：读取 intervals['trials'] 提取每个试次 (trial) 的 start_time 与 stop_time。
  • 目标标签：读取 trials['finger'] 获取每个 trial 对应的手指运动类别。

2. 试次截取与矩阵构建 (Trial Segmentation & Matrix Construction)

  • 针对每个单独的 trial，系统根据其起止时间精准截取神经脉冲，并将其转换为形状为 [time_steps, channels] 的二维密集矩阵，记录每个离散时间点上的脉冲数量。

3. 降采样与时间维度对齐 (Downsampling & Temporal Alignment)

  • 为了消除不同 trial 之间执行时间的差异，流程内部调用 bin_spikes_for_classify(spikes, expect_num_bins=50) 函数进行时间窗聚合。该操作将变长的特征矩阵强制划分为固定的 50 个时间窗 (bins)，输出标准的 [50, channels] 矩阵。对于持续时间较短的 trial，系统将自动在序列末尾补零 (Zero-padding) 以确保严格的维度对齐。

4. 张量堆叠与数据导出 (Tensor Assembly & Export)

  • 将所有规范化后的 trial 沿样本维度堆叠，形成全局的三维特征张量。最终将特征矩阵与对应的行为标签打包，导出为高压缩比的 .npz 文件，供深度学习模型直接调用。

三、使用示例​


import bcijelly as bj

loaded_data = bj.load_data(
    dataset_name="3_Finger RL",            # 选择数据集
    protocol="single_day",                  # "single_day" | "cross_day"，此处演示单天解码
    subject_index=0,                        # 范围: [0, n_subjects - 1]
    day_index=0,                            # 范围: [0, n_days - 1]
    seed=42,                                # int，单天解码中用于设置划分测试集和验证集的随机种子
    single_day_test_size=0.2,               # 范围: (0, 1)，表示测试集占单日数据的比例
    single_day_val_ratio=0.125,             # 范围: [0, 1)，仅当 val_dataset 不是 None 时有效，表示 val 占 train+val 的比例
    # auto_download_missing=True,           # 设置本地缺数据时自动从Hugging Face下载，默认为True
    verbose=False,
)

train_dataset, val_dataset, test_dataset = loaded_data.train_dataset, loaded_data.val_dataset, loaded_data.test_dataset

train_neural_signal, train_label = train_dataset
val_neural_signal, val_label = val_dataset
test_neural_signal, test_label = test_dataset

print(train_neural_signal.shape, train_label.shape)
print(val_neural_signal.shape, val_label.shape)
print(test_neural_signal.shape, test_label.shape)
