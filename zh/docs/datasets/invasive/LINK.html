6_LINK

长期皮层内神经活动与运动学

数据集链接：https://github.com/chesteklab/LINK_dataset​

DOI：10.1016/j.neuron.2021.08.009​

  • DANDI ID: 001201
  • 许可协议: CC-BY-4.0
  • 访问方式: Open Access

一、实验设计与数据采集​

被试对象及数量​

  • 物种：恒河猴
  • 被试者数：1（单猴记录，跨 3.5 年多个 session）

记录范式​

  • 目标脑区： 记录位置聚焦于初级运动皮层（M1），该区域是提取运动执行指令与肢体控制信号的核心源。
  • 采集配置： 通过在 M1 区植入的高密度电极阵列，实现对运动意图相关神经活动的高时空分辨率采样。

任务范式​

  • 自主节奏手指运动任务，包括两组手指：食指和中-环-小指
  • 屏幕上显示目标：中心-外展或随机目标
  • 每个 session: 375 个 trial
  • 行为事件：目标出现、运动开始、目标达成

二、数据处理​

数据格式​

  • Neurodata Without Borders（NWB）
  • 总 session 数：312，覆盖 303 天 (~3.5 年)
  • 每个 NWB 文件包含：
    • units：神经元 spike 时间
    • SpikingBandPower：多单位活动
    • ThresholdCrossings：阈值脉冲事件
    • index/mrs_position/velocity：运动学
    • trials：trial 级元数据

数据处理流程​

本流程旨在将原始 .nwb 文件中的神经活动与运动学特征进行提取、对齐，并通过滑窗切片转化为适用于深度学习回归模型（连续轨迹预测）的标准化 .npz 数据集。

1. 数据读取与试次切分 (Data Loading & Trial Segmentation)

  • 利用 pynwb 库加载原始文件，提取底层的神经信号（包含阈值脉冲与脉冲带功率 SBP）以及行为信号（手指连续位置与速度）。系统根据 trials 表精确提取各个试次的起止时间戳，将整段连续的记录按 trial 进行独立切分。

2. 特征提取与维度对齐 (Feature Extraction & Alignment)

针对切分出的每个 trial，分别构建时间严格对齐的二维特征张量：

  • 神经特征矩阵：提取形状为 [time, channels] 的输入矩阵（支持将离散脉冲事件转换为平均发放率或经平滑处理的连续特征）。
  • 运动学标签矩阵：同步提取形状为 [time, 2] 的输出标签（代表手指的二维坐标位置或速度）。

3. 滑动窗口采样 (Sliding Window Sampling)

  • 为了让回归网络充分学习神经活动与短时连续运动之间的映射关系，系统沿时间维度对每个 trial 执行包含式滑动窗口截取。通过设定固定的窗口长度（如 100 个时间点）与滑动步长（如 10 个时间点），在时间轴上同步成对地截取神经特征窗与运动学标签窗。

4. 张量堆叠与数据导出 (Tensor Assembly & Export)

  • 将所有 trial 经滑窗截取生成的短时样本片段拼接堆叠，构建成一个全局统一的三维数据集。最终将处理好的输入特征 (XXX) 与回归标签 (YYY) 打包，保存为高压缩比的 .npz 文件，供下游神经网络直接加载训练。

三、使用示例​

import bcijelly as bj

loaded_data = bj.load_data(
    dataset_name="6_LINK",            # 选择数据集
    protocol="single_day",                  # "single_day" | "cross_day"，此处演示单天解码
    subject_index=0,                        # 范围: [0, n_subjects - 1]
    day_index=0,                            # 范围: [0, n_days - 1]
    seed=42,                                # int，单天解码中用于设置划分测试集和验证集的随机种子
    single_day_test_size=0.2,               # 范围: (0, 1)，表示测试集占单日数据的比例
    single_day_val_ratio=0.125,             # 范围: [0, 1)，仅当 val_dataset 不是 None 时有效，表示 val 占 train+val 的比例
    # auto_download_missing=True,           # 设置本地缺数据时自动从Hugging Face下载，默认为True
    verbose=False,
)

train_dataset, val_dataset, test_dataset = loaded_data.train_dataset, loaded_data.val_dataset, loaded_data.test_dataset

train_neural_signal, train_label = train_dataset
val_neural_signal, val_label = val_dataset
test_neural_signal, test_label = test_dataset

print(train_neural_signal.shape, train_label.shape)
print(val_neural_signal.shape, val_label.shape)
print(test_neural_signal.shape, test_label.shape)
