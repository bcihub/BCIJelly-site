5_Jango

预处理与滑动窗口数据集构建

数据来源：Using adversarial networks to extend brain computer interface decoding accuracy over time（Dryad 数据集）。包含多只猴不同任务（ISO/CO/RT/PG）

数据集链接：https://datadryad.org/dataset/doi:10.5061/dryad.cvdncjt7n​

一、实验设计与数据采集​

被试对象及数量​

  • 物种：恒河猴
  • 被试者数量：多只

记录范式​

  • 记录位置：Blackrock 96 通道 Utah 阵列阈值过零脉冲，多单元（threshold crossings）。

任务范式​

  • ISO（等长力控）：屏幕上光标由腕部力控制，8 方向随机外靶；
  • CO（Center-out 伸手）：操纵平面设备到 8 个外靶；
  • RT（Random-target）：随机序列目标；
  • PG（Power/Key grasp）：抓握力控，3 个矩形目标。

通道配置​

  • M1 96 通道

二、数据处理​

数据格式​

预处理参数​

参数建议值备注
bin_size20 ms10-50ms 范围内波动
window500-800 ms覆盖运动规划的时间跨度
stride10%-50% window平衡样本量与独立性

变量​

  • 脉冲：spike_counts，单位 1ms bin 的脉冲计数（channel × time）；
  • 行为：force/position/EMG 等（本脚本示例使用 force → 2D）。

任务标签 (Labels)​

  • ISO: 使用 force (2D)
  • CO/RT: 使用 position 或 velocity (2D)
  • PG: 使用 FSR 压力信号 (1D/2D)

数据集结构​

  • 压缩包示例：Jango_ISO_2015.7z 解压后得到若干 .mat 文件。
  • 每个 .mat 文件下的关键键值（以 xds 组为例）：
    • trial_result：试次状态码（82 表成功）；
    • time_frame：毫秒时间戳；
    • spike_counts：通道×时间的脉冲计数（1 ms）；
    • force：2×time 力信号（本脚本示例作为行为）；
    • trial_start_time / trial_end_time：试次起止时间；
    • trial_target_dir：试次目标方向（单位度，-45）。

不同任务/猴的行为变量名可能不同，请按 README 映射至需要的行为（如 position、velocity 或 FSR 力）。

数据处理流程​

本流程旨在将 .mat 会话文件中的高频（1 ms 采样）神经脉冲与行为学信号（力/速度等），经过统一降采样与滑窗切片，转化为可直接用于深度学习回归/分类任务的 .npz 标准张量数据集。

1. 批量读取与试次筛选 (Batch Initialization & Filtering)

  • 通过调用主控脚本 nprmsce(data_root, bin_size=20, window=30, step=6) 启动自动化流水线。系统将使用 h5py 遍历读取目录下的 .mat 文件（提取 xds/* 键值），并根据行为学标签严格过滤出成功试次（trial_result == 82），随后按试次的时间戳对神经和行为信号进行精准切片。

2. 降采样与特征对齐 (Binning & Alignment)

由于原始信号频率过高，流程会对截取的试次数据进行固定时间窗（如 bin_size=20ms）的对齐降采样，并将运动方向标签规范化映射至 {0..7}：

  • 神经脉冲：调用 bin_spikes_for_regression(spikes, bin_size)，对每个时间块内的脉冲进行求和 (Sum)，将 [Channels, Time] 的稀疏矩阵转化为 [Bins, Channels] 的稠密放电特征。
  • 行为信号：同步调用 bin_behavior_for_regression(behavior, bin_size)，对每个时间块内的物理量（如力、速度）取均值 (Mean)，输出 [Bins, Features] 矩阵。

3. 滑窗构建训练样本 (Sliding Window Segmentation)

  • 为捕捉时序动态特征，系统将对齐后的数据送入 create_windowed_dataset 函数。利用设定的窗口长度（window=30 bins，约 600 ms）与步幅（stride=6 bins，实现 80% 重叠），在时间轴上执行包含式滑动截取（丢弃短于窗长的 trial）。最终将数据重构为具有时间感受野的三维样本集。

4. 张量导出与格式说明 (Tensor Export & Formatting)

处理完毕后，每个会话文件将被独立保存为同名的 .npz 压缩包。导出的核心数组如下：

  • features (XXX)：形状为 [N, W, C] 的输入特征矩阵（N=样本数，W=时间窗长度，C=神经通道数）。
  • labels (YYY)：形状为 [N, W, F] 的回归目标矩阵（F=行为学特征维度，如 force_x, force_y）。

三、使用示例​

1. 数据集加载与检查​

import bcijelly as bj
loaded_data = bj.load_data(
    dataset_name="5_Jango",            # 选择数据集
    protocol="single_day",                  # "single_day" | "cross_day" | ...
    task="classification",                  # "classification" | "regression" | ...
    subject_index=0,                        # 范围: [0, n_subjects - 1]
    day_index=0,                            # 范围: [0, n_days - 1]
    split_ratio="7:1:2",                    # 表示训练(-验证)-测试集划分比例，格式为"m:n"或"m:v:n"
    verbose=False,
)
train_dataset, val_dataset, test_dataset = loaded_data.train_dataset, loaded_data.val_dataset, loaded_data.test_dataset
train_neural_signal, train_label = train_dataset
val_neural_signal, val_label = val_dataset
test_neural_signal, test_label = test_dataset

print(train_neural_signal.shape, train_label.shape)
print(val_neural_signal.shape, val_label.shape)
print(test_neural_signal.shape, test_label.shape)

2. 全流程训练与评估​

import bcijelly
# 加载模型与数据集
import bcijelly.model.transformer.classifier as module
loaded_data = bcijelly.load_data(
    "5_Jango",
    protocol="single_day", task="classification",
    subject_index=0, day_index=0, split_ratio="7:1:2",
)
# 执行训练、验证与测试，生成结果汇总
train_result = module.train(loaded_data.train_dataset, loaded_data.val_dataset)
test_result = module.test(loaded_data.test_dataset)
bcijelly.summary(
    loaded_data, module, train_result, test_result,
    mode="compact",
)

四、注意事项​

常见坑位与对策​

  • 形状对不齐：spike_counts 读出为 [C, T]；行为需转为 [T, F] 后再 bin；
  • 尾部舍弃：binning 取整会丢弃不足一 bin 的尾巴 → 若需保留可改为填充或可变窗；
  • 试次索引：用 time_frame 掩码切片，务必用 indices[0] : indices[-1] + 1；
  • 任务差异：不同任务的行为键不同，需在读取层兼容；
  • 标签映射：方向角度必须完整映射到 {0..7}；异常角度需跳过或重映射；
  • 数据类型：大数组建议 int16/float32，避免 int8 溢出（视计数大小而定）。
