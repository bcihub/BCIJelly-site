7_Calcium Imaging

数据集链接：https://zenodo.org/doi/10.5281/zenodo.10408334​

DOI：10.5281/zenodo.10408334​

一、实验设计与数据采集​

被试对象及数量​

  • 物种：小鼠
  • 被试者数量：5只

记录范式​

  • 采用介观单光子宽场钙成像（Mesoscopic single-photon / wide-field calcium imaging）技术，记录头部固定（head-fixed）小鼠大脑皮层的荧光信号。图像空间分辨率调整为 128 × 128 像素。

任务范式​

  • 同步获取神经活动与行为表现，用于分析皮层动态与行为之间的关系。

    • 行为定义：根据小鼠的运动速度进行二元标记：运动速度 > 0.5 cm/s 的帧定义为“奔跑（run）”状态，< 0.5 cm/s 的帧定义为“休息（rest）”状态。

    • 实验规模：总共包含 64 个实验阶段（sessions），每个阶段时长 10 分钟。5 只小鼠对应的实验阶段数分别为 11、12、14、15 和 12 个。

采样率​

  • 钙成像图像帧率为 30 帧/秒 (30 fps)。每个 10 分钟的实验阶段约产生 18,000 帧图像数据。

二、数据处理​

数据格式​

  • 成像数据：.tif（原始图像）

  • 行为标签：.mat

  • 映射配置：.csv

  • 统计参数：.npy（均值与标准差）

  • 预处理后数据：.npz（供深度学习模型直接加载读取）

数据处理流程​

本流程旨在将原始的宽场钙成像图像和对应的行为标签转换为可供模型训练的标准时间序列窗口格式，涵盖了从数据加载、时序对齐、特征标准化到序列窗口化与保存的完整流水线。

1. 数据加载与时序对齐 (Data Loading & Temporal Alignment)

  • 读取 .csv 配置文件以获取数据 ID 与小鼠编号的映射关系，并从 .mat 文件中提取对应的行为序列（run/rest）标签。使用 tifffile 库加载 .tif 格式的宽场钙成像原始数据，通过容差匹配算法（允许 ±5 帧误差）处理长度不匹配的情况（截断或填充），以严格对齐图像帧数与行为标签序列。

2. 逐帧标准化与降维 (Frame-wise Standardization & Dimensionality Reduction)

  • 从预先计算好的文件中加载对应实验阶段的逐像素均值（mean）和标准差（std）。对对齐后的每一帧影像进行逐像素（per-pixel）标准化处理。根据设定（如 flatten=True），将标准化后的单帧图像沿空间维度展开并取通道平均值，从而降维成一维的特征向量。

3. 序列滑动窗口化 (Sequence Sliding Windowing)

  • 设定固定的时间窗口长度（T=30，即连续 30 帧作为一个样本），对降维后的连续特征序列进行步长为 1 的滑动窗口切片，从而生成所有可能的连续时间窗样本集。

4. 标签生成与格式化保存 (Label Generation & Formatting Storage)

  • 针对切分出的每个时间窗口，通过对其内部包含的所有单帧标签进行多数投票（Majority voting，即频数最大值）来确定该窗口的全局标签。最终，将窗口化后的特征张量（calcium）与对齐后的标签数组（label）打包，按照 YYYYMMDD_Mouse#.npz 的规范统一命名并保存为 .npz 格式文件，以供模型直接加载训练。

三、使用示例​

import bcijelly as bj
loaded_data = bj.load_data(
    dataset_name="7_Calcium Imaging",            # 选择数据集
    protocol="single_day",                  # "single_day" | "cross_day" | ...
    task="classification",                  # "classification" | "regression" | ...
    subject_index=0,                        # 范围: [0, n_subjects - 1]
    day_index=0,                            # 范围: [0, n_days - 1]
    split_ratio="7:1:2",                    # 表示训练(-验证)-测试集划分比例，格式为"m:n"或"m:v:n"
    verbose=False,
)
train_dataset, val_dataset, test_dataset = loaded_data.train_dataset, loaded_data.val_dataset, loaded_data.test_dataset
train_neural_signal, train_label = train_dataset
val_neural_signal, val_label = val_dataset
test_neural_signal, test_label = test_dataset

print(train_neural_signal.shape, train_label.shape)
print(val_neural_signal.shape, val_label.shape)
print(test_neural_signal.shape, test_label.shape)

2. 全流程训练与评估​

import bcijelly
# 加载模型与数据集
import bcijelly.model.transformer.classifier as module
loaded_data = bcijelly.load_data(
    "7_Calcium Imaging",
    protocol="single_day", task="classification",
    subject_index=0, day_index=0, split_ratio="7:1:2",
)
# 执行训练、验证与测试，生成结果汇总
train_result = module.train(loaded_data.train_dataset, loaded_data.val_dataset)
test_result = module.test(loaded_data.test_dataset)
bcijelly.summary(
    loaded_data, module, train_result, test_result,
    mode="compact",
)

四、注意事项​

  • 模型划分策略：原研究在进行深度学习解码分析时，采用跨小鼠（cross-subject）的交叉验证策略。5 只小鼠按 3:1:1 的比例严格划分为训练集、验证集和测试集，总共生成了 20 个模型配置以验证泛化能力。在利用此数据复现或开展新研究时，需注意防止数据泄露（Data Leakage）。

  • 数据维度：如果 flatten=False，.npz 中保留的 calcium 张量将包含完整的空间维度 [N, T, C, H, W]，数据体积会大幅增加，适用于 3D-CNN 等保留空间拓扑结构的网络模型。
