2_GOD

基于分类任务的需求，我们仅使用 perceptionTraining 子集（150 类 × 8 张 = 1 200 张自然图像）。该子集已在原始论文中完成了头动校正、空间配准及全局归一化等标准 fMRI 预处理，可用于后续训练-测试划分。

数据集链接：Generic Object Decoding (fMRI on ImageNet) - OpenNeuro​

DOI：10.1038/ncomms15037​

一、实验设计与数据采集​

被试对象及数量​

记录范式​

  • fMRI（功能性磁共振成像）。每个扫描会话包含：
    • 功能性图像 (EPI)：覆盖全脑。
    • 解剖图像 (inplane T2 加权)：与 EPI 使用相同切片获取。
    • 高分辨率解剖参考图 (T1 加权)：每个被试仅扫描一次，用于结构参考。

任务范式​

实验分为两大类任务：图像呈现实验（感知）与想象实验（表象）。

  • 图像呈现实验（感知任务）：被试观看来自 ImageNet 的物体图像，并执行 1-back 图像重复任务。

    • 训练集 (perceptionTraining)：包含 150 个物体类别，每类 8 张，共计 1 200 张自然图像。每张图像仅呈现 1 次（共 24 个 run，3-5 个会话）。

    • 测试集 (perceptionTest)：包含 50 个物体类别，每类 1 张，共计 50 张图像。每张图像重复呈现 35 次（共 35 个 run，4-6 个会话）。

  • 想象实验 (imageryTest)：被试被要求在脑海中视觉想象测试集中的 50 个类别之一。

    • 共 20 个 run（3-5 个扫描会话），每个 run 包含 25 个类别，每个类别提供 10 个样本。

    • 被试还需对心理想象的生动程度（vividness）进行自我评价打分。

二、数据处理​

数据格式​

  • MRI 影像数据：所有 DICOM 文件均通过 FreeSurfer 的 mri_convert 转换为 Nifti-1 (.nii.gz) 格式。T1 加权图像已使用 pydeface 进行了去面部特征处理。

  • 任务事件文件：.tsv 格式，包含详细的刺激时间戳和行为学记录。

  • ROI 掩码：二值化的掩码图像（.nii.gz）。

数据集结构​

数据集按被试层级组织，包含以下主要目录/会话类型：

  • ses-perceptionTraining：感知训练集的 fMRI 数据。

  • ses-perceptionTest：感知测试集的 fMRI 数据。

  • ses-imageryTest：想象实验的 fMRI 数据。

  • ses-anatomy：T1 加权解剖参考图像。

  • sourcedata：手动定义的感兴趣区域 (ROI) 掩码文件。

  • derivatives/preproc-spm：使用 SPM 预处理后的 fMRI 数据。

数据处理流程​

本流程旨在将预处理后的 4D fMRI 时间序列数据转换为适用于模型训练的降维特征张量格式，涵盖了从数据读取、提取对齐、特征工程到最终数据结构构建的完整流水线。

1. 数据加载与索引 (Data Loading & Indexing)

  • 从 derivatives 文件夹中获取预处理后的 4D fMRI 数据 (.nii.gz)、预定义的 ROI 掩模文件 (.nii.gz) 以及事件标记文件 (.tsv)。数据严格按被试 (subject)、会话 (session) 和运行 (run) 层级组织，并通过正则表达式匹配构建数据索引字典，以建立被试与对应数据文件的映射关系。

2. 信号提取与时间对齐 (Signal Extraction & Temporal Alignment)

  • 加载每个被试的 ROI 掩模并进行二值化处理，统计并按固定顺序提取各 ROI 的二维体素信号矩阵。基于 fMRI 采样周期 (TR = 3.0 s) 与 HRF 峰值滞后特性，将刺激 onset 时刻向后移动 1 个 TR 进行时间对齐，从而提取每个试次中对应 TR 的体素信号，拼接形成形状为 [总体素数, TRs_per_trial] 的试次数据，并将试次标签统一转换为从 0 开始的整数编码。

3. 特征工程与降维 (Feature Engineering & Dimensionality Reduction)

  • 使用 StandardScaler 对每个时间点 (TR) 的所有试次数据进行标准化处理，使数据均值为 0、标准差为 1。随后，利用 PCA 对每个时间点分别进行降维处理，整合所有时间点的降维结果，构建最终的特征矩阵并保留主成分矩阵和解释方差比例等模型参数。

4. 标签生成与数据存储 (Label Generation & Data Storage)

  • 处理完成的数据以被试为单位存储为 .npz 格式。通过 np.load 加载后，可利用 fmri 关键字提取形状为 (trials, 主成分数, TRs) 的特征矩阵，利用 label 提取 0~149（共 150 种取值）的分类标签。此外，还可通过 label_names、rois 及 roi_voxel_counts 获取具体的标签名称、感兴趣区域信息及其内部体素数量统计。

三、使用示例​

import bcijelly as bj
loaded_data = bj.load_data(
    dataset_name="2_GOD",            # 选择数据集
    protocol="single_day",                  # "single_day" | "cross_day" | ...
    task="classification",                  # "classification" | "regression" | ...
    subject_index=0,                        # 范围: [0, n_subjects - 1]
    day_index=0,                            # 范围: [0, n_days - 1]
    split_ratio="7:1:2",                    # 表示训练(-验证)-测试集划分比例，格式为"m:n"或"m:v:n"
    verbose=False,
)
train_dataset, val_dataset, test_dataset = loaded_data.train_dataset, loaded_data.val_dataset, loaded_data.test_dataset
train_neural_signal, train_label = train_dataset
val_neural_signal, val_label = val_dataset
test_neural_signal, test_label = test_dataset

print(train_neural_signal.shape, train_label.shape)
print(val_neural_signal.shape, val_label.shape)
print(test_neural_signal.shape, test_label.shape)

2. 全流程训练与评估​

import bcijelly
# 加载模型与数据集
import bcijelly.model.transformer.classifier as module
loaded_data = bcijelly.load_data(
    "2_GOD",
    protocol="single_day", task="classification",
    subject_index=0, day_index=0, split_ratio="7:1:2",
)
# 执行训练、验证与测试，生成结果汇总
train_result = module.train(loaded_data.train_dataset, loaded_data.val_dataset)
test_result = module.test(loaded_data.test_dataset)
bcijelly.summary(
    loaded_data, module, train_result, test_result,
    mode="compact",
)

四、注意事项​

  • 可根据实际需求调整 n_components参数，以平衡降维效果与计算资源消耗。本研究上传的数据已降维至1000维，该维度是目前实验中取得最优分类效果的设置。
  • 针对数据增强，可尝试按类别进行处理，但实验结果并未显示提升分类效果，因此最终未纳入流程中。
