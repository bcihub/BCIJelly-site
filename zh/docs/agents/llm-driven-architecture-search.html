大模型驱动的自动化架构搜索

LLM驱动的AAS​

  大模型读取三份结构化文件（Task.md、Module.md 和 History.json），使用预训练后冻结的共享编码器，组装并评估候选解码器。达到验证阈值的解码器进入留出测试并记录在知识进化步骤中，低于阈值的解码器进入诊断与设计循环，提出新模块供重新评估。右侧展示了停止条件（最多 3 个新模块设计、25 次大模型调用、60 分钟或连续 2 轮无提升）。

  五个侵入式 BCI 数据集（Reaching B、Visual Coding、Jango、Indy 和 Finger PPC）的解码性能，涵盖分类与回归任务，记录来自人、猕猴和小鼠。三种大模型后端（DeepSeek-V4 Pro、ChatGPT-5.5 和 Claude Opus-4.7）搜索得到的解码器与三个前沿预训练模型（NDT1、NDT2 和 POYO）以及自研预训练模型（UniBCI34）进行对比。各面板分别展示分类任务、回归任务以及跨物种联合训练设置（人+猕猴、人+小鼠、猕猴+小鼠、三物种混合）。分类任务报告 Accuracy，回归任务报告决定系数（R2，以百分比表示）。柱状图表示各记录日（或记录会话，针对单次记录的数据集）的均值，散点表示各记录日，误差棒表示一个标准差。统计显著性采用配对 t 检验评估。 *P < 0.05, **P < 0.01, ***P < 0.001.

LLM-driven AAS中的shared encoder​

  共享编码器将形状为 [N, T, C] 的神经输入张量（N 为试次数，T 为时间步数，C 为通道数）映射为形状为 [N, T, H] 的共享特征表示（H 为隐藏维度）。输入先经批次归一化（per-batch z-norm），再通过逐任务输入投影（实现为置换不变的 Deepset 模块：共享的逐通道 MLP 接通道均值池化或线性层），产出形状为 [N, T, H] 的表示。一个可学习的全局 token r_task ∈ RH 沿时间维度前插，得到形状为 [N, T+1, H] 的序列，并加上可学习的位置编码。该 token 使所有时间步通过自注意力交换信息，在最终块后被丢弃。序列经过六层 Transformer 块处理，每层包含层归一化、多头自注意力（8 个头）和前馈网络，均带残差连接，H(l) 表示第 l 层后的隐藏状态。最终层归一化后丢弃全局 token，得到形状为 [N, T, H] 的逐时间步共享特征，送入任务特异性解码器。

  大模型驱动闭环 AAS 在五个侵入式 BCI 数据集上的单任务解码性能。每个解码器在五个数据集（Jango、Indy、Finger PPC、Reaching B 和 Visual Coding）上分别独立训练和评估，不做跨数据集或跨物种联合训练，与上方跨物种联合训练结果图中的设置互补。模型、柱状图、散点和误差棒的约定与该图一致。分类数据集（Finger PPC、Reaching B 和 Visual Coding）使用 Accuracy，回归数据集（Jango 和 Indy）使用决定系数（R2，以百分比表示）。即使在单任务设置下，大模型驱动 AAS 搜索得到的解码器仍与前沿预训练模型保持竞争力，在 Jango 上超越基线，在 Finger PPC、Reaching B 和 Visual Coding 上与基线持平，而 NDT1 在 Indy 上保持优势。UniBCI 在 Finger PPC 和 Visual Coding 上表现强劲，取得最高均值。统计显著性采用配对 t 检验评估。 *P < 0.05, **P < 0.01, ***P < 0.001.

LLM-driven AAS​

  前文介绍的架构搜索方法一次仅能处理单一任务，且依靠随机采样遍历模块库。为实现多任务解码，并以引导式设计替代随机采样，我们为 AAS 引入大语言模型，构建可闭环完成规划、故障诊断与解码器迭代优化的全新框架（见上方工作流图）。该智能体读取三份定义搜索上下文的结构化文件：任务说明文件（Task.md，涵盖训练/验证/测试集划分、物种、任务类型，支持单任务与多任务解码）、模块说明文件（Module.md，描述全部可用网络模块）、历史记录文件（History.json，存储过往搜索得到的架构及其参数配置、验证与测试成绩）。依托上述上下文，大模型首先规划搜索流程，并在训练集上预训练共享编码器后冻结其权重；随后进入解码器搜索与筛选阶段，按规划方案组装候选解码器并在验证集上评估。验证指标超过阈值的解码器进入预留测试集完成泛化测试，其架构与成绩写入历史文件，完成知识进化；未达阈值的解码器则进入诊断与设计循环，由大模型分析验证失败原因、设计新模块、校验模块维度与数据匹配后，送回组装阶段重新评估。当满足预设停止条件时循环终止，全部迭代积累的知识统一存入历史文件。

  我们在五个侵入式 BCI 数据集（Reaching B、Visual Coding、Jango、Indy 和 Finger PPC）上评估了这一大模型驱动的搜索框架，这些数据集涵盖分类与回归任务，记录来自人、猕猴和小鼠。为单独对比不同大模型后端的解码器搜索效果，三组实验统一采用相同的共享编码器架构（见上方编码器架构图及补充表 20），以相同的数据划分方案训练并在解码器搜索前冻结。随后分别由 DeepSeek-V4 Pro、ChatGPT-5.5 和 Claude Opus-4.7 独立执行解码器搜索，每个大模型自主规划并搜索各自的解码器。我们将搜索得到的解码器与三个前沿预训练模型（NDT1、NDT2 和 POYO）以及自研预训练模型（UniBCI34）进行对比，评估场景包括单任务设置（见上方单任务结果图）、分类与回归任务，以及跨物种联合训练设置（人+猕猴、人+小鼠、猕猴+小鼠、三物种混合，见上方跨物种结果图及补充文件 2）。统计显著性采用配对 t 检验评估（补充文件 1）。

  在全部测试场景下，大模型驱动 AAS 产出的编码器-解码器组合均与前沿预训练模型性能持平，且在部分跨物种联合训练场景中显著超越（见上方跨物种结果图）。性能差距最突出的场景为跨物种联合训练：在 Jango 和 Reaching B 数据集上，搜索得到的解码器显著优于最优预训练基线（配对 t 检验，P < 0.05）。在其余数据集以及单独拆分的分类与回归任务中，搜索解码器与最优基线无显著差异。在 Indy 数据集的猕猴+小鼠和三物种混合设置中，NDT1 的性能显著优于搜索解码器。三种大模型后端之间无显著差异。

  UniBCI 在 Finger PPC 和 Visual Coding 数据集上表现强劲，与最优预训练基线无显著差异，并在多个设置下取得 Finger PPC 的最高均值。在其余数据集上，UniBCI 的性能低于前沿基线模型。

使用示例​

  大模型驱动 AAS 通过 bcijelly.LLMSearchAgent 对外暴露，封装了上述闭环搜索流程。该 API 使用任务说明、模块描述和搜索历史文件定义搜索上下文，并应用显式停止条件（最大新设计模块数、大模型调用次数、运行时长和无改进轮数）。给定多任务数据划分后，LLMSearchAgent 执行引导式解码器搜索，在指定验证集上评估候选解码器，并将选中的架构与成绩写回历史文件。这为使用固定数据划分、模块描述和搜索历史重新运行大模型驱动 AAS 提供了可复现的入口。

from bcijelly import LLMSearchAgent

llm_agent = LLMSearchAgent(
    max_design_attempts = 3,
    max_llm_calls = 25,
    max_wall_clock_min = 60,
    no_improve_rounds = 2)

multitask_data = llm_agent.split_data(
    data_path = "/data/invasive",
    split_mode = "csv")

llm_agent.search(
    multitask_data,
    task_file = "./docs/task.md",
    module_files = "./docs/modules/",
    history_file = "./docs/history.json")
