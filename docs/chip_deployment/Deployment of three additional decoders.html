Compilation and deployment of decoding models in BCIJelly

Unified deployment pipeline​

PyTorch-trained ANN models are parsed into a hardware-ready computational graph that is then compiled for the target chip. SNN models pass through an additional node conversion step that translates PyTorch operators into spike-based equivalents, after which the converted graph follows the same parsing and compilation steps as ANNs. The choice of ANN or SNN deployment is set by a single argument to the conversion API.

Native operator coverage​

Native operator coverage of three representative chips (left), a general-purpose GPU, the Lynxi neuromorphic chip and TaiBai, an in-house neuromorphic chip. Fully supported (green check), partially supported (half-orange) and not supported (red cross).

Real deployment of MFSNN​

Real deployment of MFSNN on the three platforms (right): decoding accuracy is essentially identical across the GPU, Lynxi and TaiBai, confirming that operator conversion preserves model performance during deployment. Power consumption is reduced by roughly 30-fold on Lynxi (0.57 W) and 50-fold on TaiBai (0.34 W) compared with the GPU (16.48 W，NVIDIA RTX 4060).

Operator conversion and graph-level optimization of MFSNN during compilation​

Left, the original MFSNN architecture, in which long sequences of Conv2D, BN2D and LIF layers are interleaved with matrix multiplication, Hadamard product and matrix addition operations. Middle, after operator-level optimization, each Conv2D-BN2D-LIF sequence is fused into a single spike-conv2D operator that the target chip can execute natively. Right, after graph-level optimization, the computational graph is placed onto the core array of TaiBai with a reduction in core count from 113 to 33.

Architecture of the hardware-deployment cross-day regression decoder identified by SearchAgent​

The decoder takes a neural input tensor of shape [N, T, 96], where N is the number of trials, T the number of time steps and 96 the number of channels, and produces a continuous output of shape [N, T, 2], corresponding to the two-dimensional (x, y) position. The input is first mapped into a hidden representation by a fully connected projection and passed through a non-linear activation. The core of the network is a UFO Attention module identified by the architecture search (dashed box), in which the hidden representation x is projected into query, key and value through learned weights. The key and value branches are each followed by a cross-normalization step (XNorm), and the branches are combined through successive matrix multiplications to produce the attended representation. A further non-linear activation and a fully connected projection then map this representation to the two-dimensional output. This decoder generalizes across recording days, achieving successful cross-day decoding on the Jango dataset.

Deployment of three additional decoders on the Lynxi neuromorphic chip​

Decoding performance and power consumption of three decoders deployed on a GPU and on the Lynxi neuromorphic chip, following the same deployment procedure.

LSTM on Finger PPC (classification)​

LSTM, a decoder collected in BCIJelly, evaluated on a classification task using the Finger PPC dataset. Decoding accuracy is 96.10% on both the GPU and Lynxi, while power consumption is reduced from 40.610 W to 1.033 W (roughly 39-fold).

FENet on Jango (regression)​

FENet, also collected in BCIJelly, evaluated on a regression task using the Jango dataset. The decoding performance (R2) is 0.8845 on the GPU and 0.8836 on Lynxi, while power consumption is reduced from 31.415 W to 0.606 W (roughly 52-fold).

SearchAgent decoder on Jango (regression)​

A decoder searched by SearchAgent, was evaluated on the same regression task. R2 is 0.8499 on both platforms, while power consumption is reduced from 31.714 W to 0.872 W (roughly 36-fold). Across all three decoders, decoding performance on the GPU and Lynxi is essentially identical, confirming that operator conversion preserves model performance during deployment. These results demonstrate that the BCIJelly toChip pipeline generalizes across both collected and automatically generated decoders.

Demo: Decoder Deployment on Lynxi Chip​


import bcijelly as bj

chip_model = bj.toChip(
    model_path="./model.pth",
    input_shape=(1, 1, 128, 128),
    output_dir="./lynxi_out",
    mode="SNN",
    chip_type="Lynxi",
    sim_steps=20,
)

compiled_path = chip_model.compile()
print("Compiled SNN(Lynxi) model:", compiled_path)

Demo: Decoder Deployment on Xylo Chip​


import bcijelly as bj

chip_model = bj.toChip(
    model_path="./xylo_model.pth",
    input_shape=(128, 50),
    output_dir="./xylo_out",
    mode="SNN",
    chip_type="xylo",
    xylo_encoding_path="./xylo_encoding.npz",
    xylo_external_readout="./xylo_readout.npz",
)

compiled_result = chip_model.compile()
print("Compiled Xylo model config:", compiled_result["config"])

# Test on config backend (CPU simulation)
test_result = chip_model.test(
    test_data,
    test_label,
    hardware=False,
    input_type="raw",
)
print("Xylo test result (config):", test_result)
