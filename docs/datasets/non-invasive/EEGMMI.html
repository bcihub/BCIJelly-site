3_EEGMMI

This project provides a way to preprocess the EEG Motor Movement/Imagery dataset by transforming raw EEG signals into signals for model training. Only two subsets of the EEG Motor Movement/Imagery dataset were used in the project. The first subset contains only performed left or right hand motor tasks. The second subset contains only imagined left or right hand motor tasks. In both subsets, a total of 103 out of 109 subjects were used, and subjects 38, 88, 89, 92, 100, and 104 were omitted due to data annotation errors.

Repository Link：https://physionet.org/content/eegmmidb/1.0.0/​

DOI：https://doi.org/10.13026/C28G6P​

I.Experimental Design and Data Acquisition​

Subjects​

  • Species: Human
  • Number of subjects: 109

Recording Specifications​

  • The BCI2000 system was used to record 64-channel EEG signals. The physical electrode layout strictly follows the international 10-10 system (excluding peripheral or specific electrodes such as Nz, F9, F10, FT9, FT10, A1, A2, TP9, TP10, P9, and P10).

Task Paradigm​

The experiment primarily focuses on Motor Execution and Motor Imagery. Participants are required to respond according to target cues displayed on the screen.

  • Each participant must complete 14 experimental runs, which include:
    • Baseline state: 2 runs of 1 minute each (eyes closed resting, eyes open resting).
    • Task state: 12 runs of 2 minutes each (the following 4 tasks are repeated 3 times):
    • Task 1: Execute opening and closing of the left or right fist based on left-right cues displayed on the screen.
    • Task 2: Imagine opening and closing the left or right fist based on left-right cues displayed on the screen.
    • Task 3: Execute opening and closing of both fists (up) or both feet (down) based on up-down cues displayed on the screen.
    • Task 4: Imagine opening and closing both fists (up) or both feet (down) based on up-down cues displayed on the screen.

Sampling Rate​

  • 160 Hz (i.e., 160 data samples are collected per second per channel).

II.Data Processing​

Data Format​

  • Original Data Format: The data is provided in EDF+ format (containing 64 EEG signal channels and 1 event annotation channel), accompanied by an .event annotation file for parsing.
  • Preprocessed Format: After filtering and segmenting, the data is outputted in the commonly used NumPy array format for machine learning, and saved in a compressed .npz file (e.g., EMM_task1_preprocessed.npz) for easy direct use in code.

Data Processing Pipeline​

This pipeline is designed to convert raw EEG recording data into a standardized tensor format suitable for model training, covering the complete workflow from data loading to feature and label construction.

1. Data Loading

  • The core function load_data() is used to directly read and load the 64-channel raw EEG experimental data from 109 volunteers, recorded by the BCI2000 system, from the local folder.

2. Signal Preprocessing & Chunking

  • A notch filter is applied to the EEG signal to eliminate 60 Hz power line noise, followed by band-pass filtering between 2 Hz and 60 Hz. The experimental data is then strictly truncated into 4-second segments (at 160 Hz sampling rate, resulting in 640 samples). Using a sliding window method, the data is further divided into 8 non-overlapping time segments (each window contains 80 samples).

3. Feature Extraction

  • Features are extracted from the filtered and chunked data, and formatted into a NumPy array that can be processed by both CPU and GPU. A feature matrix with the shape (number of samples, number of channels, number of time points) is constructed and saved in .npz format to the output folder.

4. Label Generation & Mapping

  • Each segmented window is precisely mapped to its corresponding experimental condition, receiving the same target labels as the original trials (e.g., representing the type of movement and whether the movement is for the left or right hand). The final supervised label array is generated and aligned with the shape (number of samples, labels).

III. Demo​

1. Data Exploration​

import bcijelly as bj
loaded_data = bj.load_data(
    dataset_name="3_EEGMMI",            # Select the dataset
    protocol="single_day",                  # "single_day" | "cross_day" | ...
    task="classification",                  # "classification" | "regression" | ...
    subject_index=0,                        # Range: [0, n_subjects - 1]
    day_index=0,                            # Range: [0, n_days - 1]
    split_ratio="7:1:2",                    # Split ratio for train(-val)-test sets, format: "m:n" or "m:v:n"
    verbose=False,
)
train_dataset, val_dataset, test_dataset = loaded_data.train_dataset, loaded_data.val_dataset, loaded_data.test_dataset
train_neural_signal, train_label = train_dataset
val_neural_signal, val_label = val_dataset
test_neural_signal, test_label = test_dataset

print(train_neural_signal.shape, train_label.shape)
print(val_neural_signal.shape, val_label.shape)
print(test_neural_signal.shape, test_label.shape)

2. Full Pipeline​

import bcijelly
# Load a model and a dataset
import bcijelly.model.transformer.classifier as module
loaded_data = bcijelly.load_data(
    "3_EEGMMI",
    protocol="single_day", task="classification",
    subject_index=0, day_index=0, split_ratio="7:1:2",
)
# train, validation, and test, with result summary
train_result = module.train(loaded_data.train_dataset, loaded_data.val_dataset)
test_result = module.test(loaded_data.test_dataset)
bcijelly.summary(
    loaded_data, module, train_result, test_result,
    mode="compact",
)
