3_Finger RL

The Finger_RL dataset contains single-neuron recordings from two tetraplegic human participants attempting individual finger movements.

Repository Link：https://dandiarchive.org/dandiset/000252/0.230408.2207​

DOI：https://doi.org/10.48324/dandi.000252/0.230408.2207​

I.Experimental Design and Data Acquisition​

Subjects​

  • Species: Human
  • Subjects: 2 tetraplegic participants

Recording Specifications​

  • Core Brain Regions:

  • Left Posterior Parietal Cortex (PPC, area PC-IP): Responsible for extracting high-level motor intentions, spatial mapping, and cognitive planning signals.

  • Left Motor Cortex hand knob (MC hand knob): Targeted for recording primary motor execution signals specifically associated with fine hand motor control.

  • Neural Sensing: Captures discrete neuronal spike activity (single-unit activity) via chronically implanted intracortical microelectrode arrays.

Task Paradigm​

  • Task: Attempted individual finger movements (both hands)

Sampling Rate​

  • 30 kHz

II.Data Processing​

Data type​

  • Single-neuron spike data, trial information, finger labels

Data Format​

  • Neurodata Without Borders（NWB）

Dataset Structure​

  • Raw data: ../data/sub-P1/*.nwb
  • Processed output: ../out/sub-P1/YYYYMMDD.npz
    • features: [Trials, Time, Channels], with time dimension binned to 50
    • labels: finger class for each trial

Data Processing Pipeline​

This pipeline is designed to parse and reconstruct raw .nwb files into standardized tensor data (.npz format) optimized for motor classification tasks.

1. Core Data Parsing The system iterates through all raw .nwb files to extract the three primary data streams required for decoding:

  • Neural Spikes: Retrieves units['spike_times'] (neuronal firing timestamps across channels) and converts them into sample indices based on the system Sampling Frequency (FS).
  • Temporal Boundaries: Accesses intervals['trials'] to extract the precise start_time and stop_time for each trial.
  • Target Labels: Reads trials['finger'] to obtain the specific finger movement category associated with each trial.

2. Trial Segmentation & Matrix Construction

  • For each individual trial, the system performs precise extraction of neural spikes based on the onset and offset timestamps. These spikes are converted into a 2D dense matrix with the shape [time_steps, channels], recording the exact spike counts at each discrete time point.

3. Downsampling & Temporal Alignment (Binning & Padding)

  • To eliminate variability in execution duration across different trials, the pipeline invokes the bin_spikes_for_classify(spikes, expect_num_bins=50) function.
  • Aggregation: This operation aggregates the variable-length feature matrices into 50 fixed time windows (bins), outputting a standardized [50, channels] matrix.
  • Zero-padding: For trials with shorter durations, the system automatically applies zero-padding at the end of the sequence to ensure strict dimensional alignment across the entire dataset.

4. Tensor Assembly & Export

  • All normalized trials are stacked along the sample dimension to form a global 3D feature tensor. Finally, the feature matrices and their corresponding behavioral labels are packaged and exported as a high-compression .npz file, allowing for seamless integration with deep learning training frameworks.

III.Demo​

1. Data Exploration​

import bcijelly as bj
loaded_data = bj.load_data(
    dataset_name="3_Finger RL",            # Select the dataset
    protocol="single_day",                  # "single_day" | "cross_day" | ...
    task="classification",                  # "classification" | "regression" | ...
    subject_index=0,                        # Range: [0, n_subjects - 1]
    day_index=0,                            # Range: [0, n_days - 1]
    split_ratio="7:1:2",                    # Split ratio for train(-val)-test sets, format: "m:n" or "m:v:n"
    verbose=False,
)
train_dataset, val_dataset, test_dataset = loaded_data.train_dataset, loaded_data.val_dataset, loaded_data.test_dataset
train_neural_signal, train_label = train_dataset
val_neural_signal, val_label = val_dataset
test_neural_signal, test_label = test_dataset

print(train_neural_signal.shape, train_label.shape)
print(val_neural_signal.shape, val_label.shape)
print(test_neural_signal.shape, test_label.shape)

2. Full Pipeline​

import bcijelly
# Load a model and a dataset
import bcijelly.model.transformer.classifier as module
loaded_data = bcijelly.load_data(
    "3_Finger RL",
    protocol="single_day", task="classification",
    subject_index=0, day_index=0, split_ratio="7:1:2",
)
# train, validation, and test, with result summary
train_result = module.train(loaded_data.train_dataset, loaded_data.val_dataset)
test_result = module.test(loaded_data.test_dataset)
bcijelly.summary(
    loaded_data, module, train_result, test_result,
    mode="compact",
)
