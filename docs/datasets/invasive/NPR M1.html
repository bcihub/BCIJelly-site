7_NPR M1

Spike-Cursor Dataset

This dataset is sourced from Zenodo Record 3854034, which contains invasive neural recordings from the primary motor cortex (M1) of non-human primates performing a center-out cursor control task. The neural data were recorded using a 192-channel Utah array, and the behavioral data consist of 2D cursor positions and corresponding target positions. Each .mat file corresponds to one recording session.

Repository Link：https://zenodo.org/records/3854034​

DOI：10.5281/zenodo.3854034​

I.Experimental Design and Data Acquisition​

Subjects​

  • Species: Macaca mulatta (Rhesus monkey)
  • Number of subjects: 3

Recording Specifications​

  • Core Configuration: Most experiments recorded activity from the primary motor cortex (M1) with a total of 96 channels.
  • Extended Configuration: Some experiments implemented dual-area synchronous recordings from both M1 and the primary somatosensory cortex (S1), with a total of 192 channels.
  • Recorded Data: Includes offline sorted spike times (Spike times). Each physical channel can distinguish and extract up to 3 independent sorted units (Sorted Units) based on firing characteristics.
  • Behavioral Signals and Kinematic Data: The experiment synchronously captures the subject's movement features through high-frequency sampling, including:
  • Cursor Position (cursor_pos): Reflects the subject's real-time movement trajectory.
  • Target Position (target_pos): Marks the visual target endpoint of the task, used to decode movement intentions.

Task Paradigm​

  • Behavioral Task: Monkeys voluntarily control their movement velocity to reach for targets arranged in a grid (e.g., an 8×8 grid), with no gaps between grid positions or pre-movement delay.

  • Experimental Task：Center-out cursor reaching task (center-to-target).

Sampling Rate​

  • 250Hz

Channel Configuration​

  • Subject: Non-human primate (NHP) implanted with a 192-channel Utah array.

II.Data Processing​

Dataset Structure​

The original dataset includes:

  • Spikes: Sorted spike times for up to 3 units per channel.
  • Cursor position: Continuous 2D coordinates.
  • Target position: 2D coordinates indicating the current target location.
  • Timestamps: Trial-aligned timing information.

Data Processing Pipeline​

This workflow uses preprocess.py to preprocess raw .mat data, aiming to generate a standardized binning and windowing dataset suitable for regression decoding models.

1. Spike Time Conversion

  • The system first extracts spike times from all channels and units, merging the spike times of different units belonging to the same channel. The discrete spike times are then converted into a continuous spike count matrix, strictly following the sampling frequency of the behavioral signals.

2. Trial Segmentation

  • The trial boundaries are dynamically defined by calculating the per-frame displacement of the target position. When the frame-to-frame displacement exceeds a preset threshold (> 1 unit), the system marks the beginning of a new trial. The continuous neural and behavioral data are then precisely segmented into independent trial fragments at these boundaries.

3. Time Binning & Alignment

  • The segmented trials undergo downsampling and bidirectional alignment.
    • Spike Binning: The total number of spikes within a fixed-length bin (default bin_size is 5 samples) is counted.
    • Behavioral Binning: The mean of the cursor position is calculated within the identical bin window, ensuring strict alignment of neural signals and behavioral labels along the time dimension.

4. Windowing for Regression To construct training samples for the regression task, the system performs a sliding window operation on the aligned data (default window size window_size is 100 bins, and the stride stride is 10 bins). This step generates paired independent windows for each trial:

  • Features: Spike count matrix with shape [window_size, n_channels].
  • Labels: Binned cursor position with shape [window_size, behavior_dim].

5. Result Export

  • After processing, each data session generates and exports standardized tensor files:
    • Input features spikes.npy (shape [n_samples, window_size, n_channels]).
    • Regression labels behaviors.npy (shape [n_samples, window_size, behavior_dim]).

Additionally, the system automatically plots and saves raster plots (.png) for the first 3 trials to assist in data quality checks.

III.Demo​

import bcijelly as bj

loaded_data = bj.load_data(
    dataset_name="7_NPR M1",            # Select the dataset
    protocol="single_day",                  # "single_day" | "cross_day"; demonstrates single-day decoding here
    subject_index=0,                        # Range: [0, n_subjects - 1]
    day_index=0,                            # Range: [0, n_days - 1]
    seed=42,                                # int; random seed for splitting test and validation sets in single-day decoding
    single_day_test_size=0.2,               # Range: (0, 1); proportion of the single-day data allocated to the test set
    single_day_val_ratio=0.125,             # Range: [0, 1); only valid when val_dataset is not None; represents the ratio of val within the train+val split
    # auto_download_missing=True,           # Automatically download from Hugging Face if local data is missing; defaults to True
    verbose=False,
)

train_dataset, val_dataset, test_dataset = loaded_data.train_dataset, loaded_data.val_dataset, loaded_data.test_dataset

train_neural_signal, train_label = train_dataset
val_neural_signal, val_label = val_dataset
test_neural_signal, test_label = test_dataset

print(train_neural_signal.shape, train_label.shape)
print(val_neural_signal.shape, val_label.shape)
print(test_neural_signal.shape, test_label.shape)
