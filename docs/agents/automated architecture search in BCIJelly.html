Modular decoders and automated architecture search in BCIJelly

The algorithmic module library in BCIJelly​

Left, the 80 modules currently included in BCIJelly, grouped into four functional categories (fully connected, backbones, convolutional and attention). Right, schematic showing how the same modules are combined to construct a range of established and recent decoding architectures collected in BCIJelly.

The pipeline of AAS​

Given a neural recording, AAS first scans the module library, then runs the Agent Architecture Search (AAS) algorithm, which iteratively samples candidate architectures, trains each on a training set and evaluates it on a validation set. Candidates whose validation metric exceeds a predefined threshold are retained, the rest are discarded. The loop stops when K valid candidates have been collected or a maximum number of search rounds is reached, after which an ensemble selection step combines the retained candidates into a single model whose performance is reported on a held-out test set.

Performance of AAS across five independent searches​

Each violin shows the distribution of test-day performance for one search, with markers indicating individual test days. Adjacent matrices report the number of overlapping modules between pairs of searches, showing that different runs converge to distinct module combinations. The best run reached a median accuracy of 0.71 on the classification task (11 test days spanning 35 days) and a median R² of 0.80 on the regression task (13 test days spanning 88 days).

Architectures of the best decoders identified by AAS​

Classification decoder (search index = 5)​

This model corresponds to the best classification search result (search index = 5), which is an ensemble of four sub-networks.

Each sub-network shares the same overall structure, starting from an InputProjection module and ending in a ClassificationHead, but differs in its intermediate feature-extraction modules, which combine components such as TripletAttention, CondConv, RepBlock, Involution, SKAttention and a ResNet18 backbone. The four sub-networks are combined into the final classification decoder.

Regression decoder (search index = 5)​

This model corresponds to the best regression search result (search index = 5), consisting of a FlattenInput module followed by three fully connected layers (FlattenFC with 256, 128 and 128 units) and a RegressionHead.

InputProjection module​

The InputProjection module, which maps the input neural tensor of shape (C, T, N), where C is the number of channels, T the number of time steps and N the number of trials, into a hidden representation through a learned linear transformation with weight W and bias b.

FlattenInput module​

The FlattenInput module, which reshapes the neural input for the fully connected regression network.

The ClassificationHead, which applies average pooling over the temporal dimension followed by a linear projection (weight W of shape hidden_dim × num_class) to produce class scores.

RegressionHead​

The RegressionHead, which applies a linear projection (weight W of shape hidden_dim × C2, where C2 is the dimensionality of the continuous output) to produce the regression output.
