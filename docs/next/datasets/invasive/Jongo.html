5_Jongo

Preprocessing and Sliding Window Dataset Construction

Dataset LINK

https://datadryad.org/dataset/doi:10.5061/dryad.cvdncjt7n

1. Objective​

From .mat session files, extract spike counts (1 ms sampling) and behavioral signals (force/velocity/position, etc.):

  1. Downsample both to a fixed bin size (e.g., 20 ms);
  2. Segment data by trial and filter successful trials;
  3. Construct sliding window samples for regression/classification tasks;
  4. Export features and labels as .npz files for downstream model training.

2. Tasks and Variables​

  • Task paradigms:
    • ISO (Isometric Force Control): Cursor on screen controlled by wrist force, 8 random outer targets;
    • CO (Center-Out Reaching): Move planar manipulandum to 8 outer targets;
    • RT (Random-target): Sequence of randomly presented targets;
    • PG (Power/Key Grasp): Grasp force control task with 3 rectangular targets.
  • Channels: 96 M1 channels, multi-unit threshold crossings;
  • Spikes: spike_counts, counts per 1 ms bin (channel × time);
  • Behavior: force/position/EMG, etc. (this example uses force → 2D).

3. Directory and Files​

  • Example archive: Jango_ISO_2015.7z → extract to obtain multiple .mat files.
  • Key fields inside each .mat file (example xds group):
    • trial_result: trial status code (82 = success);
    • time_frame: timestamps in ms;
    • spike_counts: channel × time spike counts (1 ms bins);
    • force: 2 × time force signals (used as behavioral signal in this example);
    • trial_start_time / trial_end_time: trial start/end time;
    • trial_target_dir: trial target direction (degrees: -45).

Note: Behavioral variable names may differ across tasks/monkeys. Map them according to README (e.g., position, velocity, or FSR force).

4. Processing Overview​

  1. Enumerate .mat files → read xds/* fields using h5py;
  2. Filter successful trials (trial_result == 82);
  3. Slice spike counts and behavior by trial timestamps;
  4. Time-aligned downsampling (binning):
    • Spikes: bin_spikes_for_regression(spikes, bin_size) → sum spikes per channel in each bin;
    • Behavior: bin_behavior_for_regression(behavior, bin_size) → average value per bin;
  5. Normalize direction labels: map {0,45,90,135,180,-135,-90,-45} → {0..7};
  6. Sliding window sample construction: create_windowed_dataset(data, label, trial_labels, window, stride):
    • window (in bins) and stride control window length and step;
    • Output: features [N, window, C], labels [N, window, F], trial_id/dir;
  7. Save: Each session/file → filename.npz containing features and labels.

5. Core Functions​

5.1 bin_spikes_for_regression(spikes, bin_size)​

  • Input: spikes [channels, time_steps], bin_size in ms (e.g., 20);
  • Process: Divide time into expect_num_bins = time_steps // bin_size bins; discard remaining tail; sum spikes within each bin;
  • Output: [expect_num_bins, channels] (time first).

If input is [time, channel], transpose before calling.

5.2 bin_behavior_for_regression(behavior, bin_size)​

  • Input: behavior [time_steps, features];
  • Process: Average within each bin;
  • Output: [expect_num_bins, features].

5.3 create_windowed_dataset(data, label, trial_labels, window_size, stride)​

  • Input:
    • data: list of trials, each [T, C];
    • label: list of trials, each [T, F];
    • trial_labels: discrete label for each trial (e.g., one of 8 directions);
    • window_size: length in bins; stride: step in bins.
  • Output:
    • features [N, window, C]
    • labels [N, window, F]
    • trial_meta [N] (trial label per window)
  • Note: Discard trials shorter than window_size; use inclusive sliding windows (start=0..T-window step=stride).

5.4 nprmsce(data_root, bin_size=20, window=30, step=6)​

  • Purpose: Batch-process .mat files, slice trials, bin, create windows, and save;
  • Default params:
    • bin_size=20 ms (~50 Hz);
    • window=30 bins (~600 ms);
    • step=6 bins (~120 ms) → 80% overlap;
  • Output: .npz with same name as input, containing features and labels.

6. Output Data Format​

  • features: np.ndarray, shape [N, W, C]
    • N: number of sliding windows; W: time steps per window (bins); C: number of channels;
  • labels: np.ndarray, shape [N, W, F]
    • F: number of behavioral features (example: 2 → force_x, force_y).

For classification (e.g., 8 directions), derive window-level labels from trial_meta / trial_target_dir (e.g., center or majority vote). Current script saves regression labels (per-time-step behavior).

7. Usage Example​

# 1) Extract Jango_ISO_2015.7z to ./Jango_ISO_2015
# 2) Run script (Python 3.9+)
python main.py  # internally calls nprmsce
# 3) Generates <session>.npz

8. Recommended Hyperparameters​

  • bin_size: 10–50 ms (match behavior bandwidth; 20 ms common);
  • window: 300–1000 ms (task-dependent; CO/ISO ~500–800 ms);
  • stride: 10–50% of window (controls sample correlation & dataset size);
  • Behavior selection:
    • ISO: force (2D);
    • CO/RT: position/velocity (2D);
    • PG: FSR force (1D or 2D difference/sum).

9. Common Pitfalls & Solutions​

  • Mismatched shapes: spike_counts [C,T]; behavior should be [T,F] before binning;
  • Tail discard: Binning discards remaining time < bin size → pad or use variable windows if needed;
  • Trial indexing: Use indices[0]:indices[-1]+1 when slicing via time_frame;
  • Task differences: Behavioral keys differ across tasks → handle in reading layer;
  • Label mapping: Direction angles must map to {0..7}; skip or remap invalid angles;
  • Data type: Use int16/float32 for large arrays to avoid overflow (int8 may overflow depending on spike counts).

10. Extensions & Downstream Usage​

  • Regression: Decode 2D force/velocity/position (R², RMSE metrics);
  • Classification: 8-direction discrete intention (accuracy, F1);
  • Cross-day alignment: Apply CycleGAN/ADAN/Procrustes for temporal robustness;
  • Deep model input: Convert [N,W,C] to model-required format (e.g., [N,C,W] for CNN/RNN/Transformer).

11. Dependencies​

  • numpy, h5py, scipy, matplotlib, pickle (optional)

12. References & Acknowledgments​

  • Dataset: Ma et al., 2023, Dryad. Using adversarial networks to extend brain computer interface decoding accuracy over time.
  • Hardware/Preprocessing: Blackrock Neurotech; threshold set at -4.75x ~ -6.25x RMS; 50 ms non-overlapping bins + 100 ms Gaussian smoothing (see README).
