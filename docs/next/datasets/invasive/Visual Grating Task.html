1. Project Overviewâ€‹

This project provides a complete preprocessing pipeline for Visual Grating Task ECoG recordings from neurotycho.org. The goal is to extract event-related segments from raw .mat files, perform preprocessing (e.g., common average referencing (CAR), downsampling), and save structured datasets suitable for neural decoding experiments.

Reference: Neural Decoding Using Kernel-Based Functional Representation of ECoG Recordings

2. Key Featuresâ€‹

  • Load .mat ECoG recordings from multiple sessions in visual_grating_task/data/
  • Detect and extract specific events: blank and raster
  • Apply CAR denoising to remove global noise
  • Downsample high-frequency ECoG segments to a fixed length
  • Save processed samples and metadata in .npz format
  • Generate a processing report with detailed statistics

3. Core Functionsâ€‹

get_event_indices(mat_file_path)â€‹

  • Purpose: Extract start and end indices for each labeled event segment (blank, raster) from the event label channel.

  • Output:

    • Dictionary containing event type, segment indices, and label values.

process_mat_file(mat_file_path)â€‹

  • Purpose: Process a single .mat file, extract valid event segments, perform CAR denoising, and downsample.

  • Outputs:

    • Structured dictionary containing:

      • samples: (num_samples, 300, 128)
      • labels: (num_samples,)
      • label_names: ['blank', 'raster']
      • file_info: (num_samples,), source file names
      • sample_label_names: event names per sample

6. Preprocessing Stepsâ€‹

  1. Load Data

    • Read .mat files with scipy.io.loadmat

    • Access X matrix:

      • X[:128, :] â†’ 128-channel raw ECoG signal
      • X[128, :] â†’ event labels per time step

  2. Event Detection

    • blank: label value in [300, 400]
    • raster: label value > 100 and not in [300, 400]
    • Extract continuous segments for each event type

  3. CAR Denoising

    car_signal = signal_data - np.mean(signal_data, axis=0, keepdims=True)

  4. Segment Selection

    • Keep only segments with length â‰¥ 1920 samples
    • Extract [start+20 : start+20+1900] to avoid state transition artifacts

  5. Downsampling

    • Interpolate 1900 samples â†’ 300 samples per channel

  6. Sample Assembly

    • Shape each event to (300, 128)
    • Store label, class name, and source file info

9. Troubleshootingâ€‹

  • MAT file load errors: Check file integrity and MATLAB version compatibility
  • No valid events found: Ensure event label channel exists and matches expected codes
  • Unexpected shapes: Verify that all .mat files have the correct X matrix format
