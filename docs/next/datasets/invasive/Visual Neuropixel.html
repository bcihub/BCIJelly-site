9_Visual Neuropixel

Neuropixels Data Processing Workflow

Allen Brain Observatory — Visual Coding Neuropixels dataset (NWB, downloaded via AllenSDK). This dataset contains 58 experiments, each with up to 6 Neuropixels probes covering cortical, thalamic, and hippocampal regions, and is designed for systematic study of the mouse visual system under passive stimulation. This workflow uses drifting gratings (8 orientations) as an example, showing the complete process of session selection, unit filtering, spike matrix construction, time binning, and data saving. The official documentation describes the 8 orientations and temporal/spatial frequency settings in detail.

Repository Link​

https://portal.brain-map.org/circuits-behavior/visual-coding-neuropixels

Data Processing Pipeline​

1. Environment Requirements​

  • Python ≥ 3.8
  • AllenSDK
  • NumPy、tqdm、 pickle
pip install allensdk numpy tqdm

2. Directory Structure & Caching​

  • By default, this workflow uses a local cache directory ./allen_vc_npx, where AllenSDK creates and maintains manifest.json and downloaded session/unit data.
  • You can set a custom cache_dir in the parameters section.

3. Objective and Output Format​

Objective: From the Visual Coding – Neuropixels project, select sessions containing the drifting_gratings stimulus, filter units (neurons) within a specified brain region (e.g., VISp) that meet the SNR threshold, build raw 0/1 spike-time matrices for each trial, and perform time binning (standardizing to a fixed number of time steps). Finally, save the processed data as .npy:

  • spike: integer spike count matrix with shape (num_trials, time_bins, num_units)
  • label: integer vector (num_trials,), containing the class labels for the 8 orientations (0–315°, in 45° steps)

4. Key Parameters​

ParameterExample ValueDescription
cache_dir./allen_vc_npxAllenSDK cache directory; stores downloaded metadata and NWB files.
sampling_rate30,000 HzRaw sampling frequency of the Neuropixels probe.
snr_thresh2.0Signal-to-noise ratio (SNR) threshold for unit filtering; higher values ensure cleaner signals.
target_area"VISp"Target brain region for analysis (e.g., Primary Visual Cortex, VISp).
expect_num_bins50Time-dimension alignment parameter; uniformly segments each trial into 50 time steps.

5. Processing Pipeline (Step-by-Step)​

Initialize Project Cache and Session Table​

  1. Create/verify the cache directory and manifest.json.
  2. Load the project cache using EcephysProjectCache.from_warehouse(manifest=...).
  3. Read the session table with cache.get_session_table() and iterate over session IDs.

AllenSDK’s EcephysProjectCache automatically manages downloads and local storage and is the recommended entry point for this dataset.

Select Sessions Containing drifting_gratings​

  • Copy session.stimulus_presentations and clean the orientation field (set "null" to NaN).
  • Keep only rows where stimulus_name == "drifting_gratings", removing empty orientations.
  • Keep the 8 valid orientations {0, 45, 90, 135, 180, 225, 270, 315}, mapping each to class labels 0..7.

According to official documentation, drifting gratings use 8 orientations spaced 45° apart. Entries with null orientation represent blank screens and should be removed.

Filter Units by Brain Region and Quality Metrics​

  • Select units from session.units where structure_acronym == target_area and snr >= snr_thresh.

Construct Raw Binary Spike Matrices for Each Trial​

  • For each drifting_gratings trial:
    • Compute trial duration from start_time to stop_time and convert to sample count (rounding up at 30 kHz).
    • Initialize a (num_units_selected, num_samples) 0/1 matrix.
    • For each unit, get spike_times[unit_id] within [start_time, stop_time), convert to relative sample indices, and set those positions to 1.

Perform Time Binning to Standardize Trial Length​

  • The function bin_spikes_for_classify(data, expect_num_bins=50):
    1. Accepts a matrix shaped [channels, time_steps].
    2. Computes bin_size = ceil(time_steps / expect_num_bins).
    3. Pads with zeros if necessary to match bin_size * expect_num_bins.
    4. Reshapes to [channels, expect_num_bins, bin_size] and sums over the last axis.
    5. Returns [expect_num_bins, channels] .
  • In the main loop, call this function for each trial and store results in X_list.

Package & Save​

  • Stack all binned trials into X_array of shape (num_trials, time_bins, num_units) and convert label list to y_array.
  • Save as .npy:
data_dict = {"spike": X_array, "label": y_array}
np.save("./allen_vc_npx/session_{ID}_dg8_{AREA}.npy", data_dict)

6. How to Run​

  1. Place check_data.py in any directory.

  2. Modify the parameter section (cache dir, brain area, SNR threshold, bin count, etc.).

  3. Run:

    python data_process.py

  4. The script will iterate through sessions, process only those with drifting_gratings, and save .npy files in the cache directory.

7. Output Data Format​

  • session_{session_id}_dg8_{target_area}.npy:
    • spike: integer (np.int8 or np.int32), shape (N, T, C)
    • label: integer, values 0–7 for 8 orientations

IV.Usage Instructions​

  • Missing drifting_gratings: Not all sessions contain this stimulus; script skips missing ones. Consider supporting static_gratings or natural movie stimuli.
  • Time steps & bin size: Larger expect_num_bins means higher temporal resolution but larger input tensors; adjust for model and memory constraints.
  • Unit quality control: Current filter is SNR ≥ 2; you can add criteria such as firing_rate and ISI violations from AllenSDK.
