13_Emotion

Repository Link：https://neurotycho.org/emotional-movie-task​

I.Experimental Design and Data Acquisition​

Subjects​

  • Species: Macaca mulatta (Rhesus monkey)
  • Number of subjects: Multiple

Recording Specifications​

  • Recording Modalities: Simultaneous acquisition of Electrocorticography (ECoG) and eye-tracking data during task performance. No motion tracking was conducted.

Task Paradigm​

  • Task Type: Passive Viewing Task.

  • Stimuli: Six distinct movie clips.

  • Subject State: The subject (macaque) remained seated in a head-fixed configuration to minimize motion artifacts and ensure the stability of visual stimulation.

  • Feedback & Reward Mechanism: No food reward was provided. The subject was not required to perform any active behavioral interactions, thereby eliminating potential confounding effects from operant conditioning.

  • Synchronized Data Acquisition:

  • Neural Recording: Cortical ECoG signals were recorded at a sampling rate of 1 kHz.

  • Behavioral Recording: Eye-tracking data were acquired at a sampling rate of 120 Hz.

  • Synchronization & Constraints: Neural and behavioral signals were strictly synchronized via hardware/software triggers. The experimental design explicitly excluded limb motion tracking, focusing the analytical scope on the mapping between visual input, oculomotor behavior, and cortical neural dynamics.

Sampling Rate​

  • ECoG (Neural): 1 kHz
  • Eye-tracking: 120 Hz

Channel configuration​

  • This workflow processes 128-channel ECoG LFP data and converts it into a binned spike representation suitable for classification and pattern recognition tasks.

II.Data Processing​

Data Format​

  • 128-channel LFP .mat files (one file per channel)
  • Task_info.mat file containing trial labels and stimulus sequences
  • Movie_start_time.mat file containing stimulus onset times

Data Processing Pipeline​

This workflow aims to perform band-pass filtering, trial segmentation, and adaptive spike extraction on Local Field Potential (LFP) signals. After time binning, these signals are converted into a sparse tensor dataset (.npz format) suitable for emotion classification tasks.

1. Label Mapping & Trial Alignment

  • The system first reads the stimulus sequence (MovieSequence) from Task_info.mat and strictly maps the 6 types of physical stimulus IDs into 3 categories of numerical emotion labels (0=Frighten, 1=Threaten, 2=Neutral). Simultaneously, it reads Movie_start_time.mat to obtain the start timestamps (ms) for each trial. The labels and timestamps are then truncated to their minimum common length to ensure absolute alignment in the number of trials and labels.

2. Signal Cropping & Filtering

  • The system traverses the raw LFP_ch{n}.mat files for all channels and crops fixed-length (trial_duration_sec) signal segments based on the extracted trial start times. Subsequently, a 3rd-order Butterworth band-pass filter (20–250 Hz) is employed to eliminate low-frequency baseline drift and high-frequency ambient noise. Using the np.interp linear interpolation method, the signals are uniformly downsampled to the target timesteps (target_timesteps), completing the reconstruction from raw one-dimensional signals into a (channels, trials, target_timesteps) three-dimensional matrix.

3. Adaptive Spike Detection

  • Based on the filtered signals, the system independently calculates the mean (μ\muμ) and standard deviation (σ\sigmaσ) of the signal for each channel in every trial. Adaptive bidirectional detection thresholds are set accordingly (Positive Threshold = μ+2σ\mu + 2\sigmaμ+2σ, Negative Threshold = μ−2σ\mu - 2\sigmaμ−2σ). The system traverses all timesteps; once the signal exceeds the boundaries, it is marked as a Spike (value of 1). Upon triggering, it immediately enters a set refractory period (refractory_ms) to prevent redundant detection. This step transforms continuous analog signals into 0/1 binarized features with a shape of (trials, target_timesteps, channels).

4. Time Binning & Aggregation

  • To further compress the temporal dimension, the system aggregates the extracted 0/1 spike sequences into bins according to a set bin_size, counting the total number of spikes within each time window (Bin). For example, with a configuration of target_timesteps=50 and bin_size=10, the system reconstructs them into 5 Bins, finally outputting a dense count matrix with a shape of (trials, binned_time, channels).

5. Visualization & Export

  • At the end of the pipeline, the system automatically generates Filtered LFP signal curves, label distribution histograms, Spike Raster Plots, and Binned Spike Heatmaps for data quality spot-checking. The processed final data will be unified and packaged, then saved to the processed/data_YYYYMMDD.npz file.

IV.Usage Instructions​

  • Filter cutoff frequencies: low cutoff > 0 Hz, high cutoff < Nyquist frequency
  • Trial truncation: trials shorter than trial_duration_sec are discarded
  • bin_size must evenly divide target_timesteps or the remainder is discarded
  • Detection sensitivity: 2σ can be adjusted to 1.5σ or 3σ
  • Refractory period: refractory_ms is measured in timesteps after downsampling, ensure proper conversion
