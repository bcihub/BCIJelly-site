13_Emotion

Data Preprocessing Documentation

Dataset Link​

https://neurotycho.org/emotional-movie-task

1. Data Source and Goal​

This workflow processes 128-channel ECoG LFP data and converts it into a binned spike representation suitable for classification and pattern recognition tasks.

The dataset contains:

  • 128-channel LFP .mat files (one file per channel)
  • Task_info.mat file containing trial labels and stimulus sequences
  • Movie_start_time.mat file containing stimulus onset times

Processing goals:

  • Band-pass filter LFP (20–250 Hz)
  • Segment into trials with uniform sample count
  • Downsample to specified number of timesteps (target_timesteps)
  • Extract spikes using an adaptive threshold
  • Perform temporal binning to obtain a sparse spike array of shape (trials, binned_time, channels)
  • Save as .npz file containing both spike and label fields

3. Processing Steps​

3.1 Aligning Labels and Trial Times​

  • Load MovieSequence from Task_info.mat
  • Build stimulus ID → emotion category mapping:{1: Fear, 2: Fear, 3: Threat, 4: Threat, 5: Neutral, 6: Neutral}Numerical labels: 0 = Frighten, 1 = Threaten, 2 = Neutral
  • Load trial start times (ms) from Movie_start_time.mat
  • Use the smaller length of the two arrays to ensure alignment

3.2 LFP Segmentation and Filtering​

For each channel:

  1. Load LFP_ch{n}.mat
  2. Slice segments of trial_duration_sec starting from each trial’s start time
  3. Apply 3rd-order Butterworth band-pass filter (20–250 Hz)
  4. Downsample to target_timesteps using np.interp linear interpolation

Shape transformation: Original: (samples,) After segmentation & processing: (channels, trials, target_timesteps)

3.3 Spike Detection (Adaptive Threshold)​

For each trial and each channel:

  • Compute mean μ and standard deviation σ
  • Set detection thresholds:Positive threshold = μ + 2σNegative threshold = μ − 2σ
  • Iterate over timesteps: If signal > positive threshold or < negative threshold → mark spike Enforce refractory_ms period where detection is skipped

Output: spike_data shape: (trials, target_timesteps, channels) Values: 0/1

3.4 Temporal Binning​

  • Group spikes into bins of size bin_size
  • Count spikes within each bin

Output shape: (trials, binned_time, channels)

Example: If target_timesteps = 50, bin_size = 10 → binned_time = 50 // 10 = 5 Final shape: (n_trials, 5, 128)

3.5 Visualization​

  • Filtered LFP waveform plots
  • Label distribution bar chart
  • Spike raster plot
  • Binned spike heatmap

3.6 Saving Output​

  • Save to: processed/data_YYYYMMDD.npz
  • Contents:
    • spike: binned spike data (trials, binned_time, channels), dtype uint8
    • label: emotion labels (trials, 1), dtype int

4. Example Output​

Fields: ['spike', 'label']

5. Notes​

  • Filter cutoff frequencies: low cutoff > 0 Hz, high cutoff < Nyquist frequency
  • Trial truncation: trials shorter than trial_duration_sec are discarded
  • bin_size must evenly divide target_timesteps or the remainder is discarded
  • Detection sensitivity: 2σ can be adjusted to 1.5σ or 3σ
  • Refractory period: refractory_ms is measured in timesteps after downsampling, ensure proper conversion
