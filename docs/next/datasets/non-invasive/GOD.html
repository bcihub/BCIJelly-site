2_GOD

Based on the requirements of the classification task, we only use the perceptionTraining subset (150 categories × 8 images = 1,200 natural images). This subset has undergone standard fMRI preprocessing in the original paper, including head motion correction, spatial registration, and global normalization, making it suitable for subsequent training-test splitting.

Repository Link：Generic Object Decoding (fMRI on ImageNet) - OpenNeuro​

DOI：10.1038/ncomms15037​

I.Experimental Design and Data Acquisition​

Subjects​

  • Species: Human
  • Number of subjects:

Recording Specifications​

  • fMRI (Functional Magnetic Resonance Imaging): Each scanning session includes:
    • Functional images (EPI): Covering the entire brain.
    • Anatomical images (in-plane T2 weighted): Acquired using the same slices as the EPI.
    • High-resolution anatomical reference images (T1 weighted): Scanned only once for each participant, used as structural references.

Task Paradigm​

The experiment is divided into two main types of tasks: Image Presentation (Perception) and Imagery (Representation) tasks.

  • Image Presentation Task (Perception Task): Participants view object images from ImageNet and perform a 1-back image repetition task.
    • Training Set (perceptionTraining): Contains 150 object categories, with 8 images per category, totaling 1,200 natural images. Each image is presented only once (24 runs in total, with 3-5 sessions).
    • Test Set (perceptionTest): Contains 50 object categories, with 1 image per category, totaling 50 images. Each image is repeated 35 times (35 runs in total, with 4-6 sessions).
  • Imagery Task (imageryTest): Participants are asked to mentally visualize one of the 50 categories from the test set.
    • There are 20 runs in total (3-5 scanning sessions), with each run containing 25 categories, and 10 samples per category.
    • Participants are also required to self-rate the vividness of their mental imagery.

II.Data Processing​

Data Format​

  • MRI Imaging Data: All DICOM files were converted to NIfTI-1 (.nii.gz) format using FreeSurfer’s mri_convert. The T1-weighted images have been processed with pydeface to remove facial features.
  • Task Event Files: In .tsv format, containing detailed stimulus timestamps and behavioral records.
  • ROI Masks: Binary mask images in .nii.gz format.

Dataset Structure​

  • The dataset is organized at the participant level and contains the following main directories/session types:
    • ses-perceptionTraining: fMRI data for the perception training set.
    • ses-perceptionTest: fMRI data for the perception test set.
    • ses-imageryTest: fMRI data for the imagery test task.
    • ses-anatomy: T1-weighted anatomical reference images.
    • sourcedata: Manually defined Regions of Interest (ROI) mask files.
    • derivatives/preproc-spm: fMRI data preprocessed using SPM.

Data Processing Pipeline​

This pipeline is designed to convert preprocessed 4D fMRI time-series data into a reduced-dimensional feature tensor format suitable for model training, covering the complete workflow from data loading, extraction, alignment, feature engineering, to the final construction of the data structure.

1. Data Loading & Indexing

  • Preprocessed 4D fMRI data (.nii.gz), predefined ROI mask files (.nii.gz), and event annotation files (.tsv) are obtained from the derivatives folder. The data is strictly organized by participant (subject), session, and run levels. A data index dictionary is built using regular expression matching to map participants to their corresponding data files.

2. Signal Extraction & Temporal Alignment

  • Each participant’s ROI mask is loaded and binarized. The 2D voxel signal matrices for each ROI are extracted in a fixed order. Based on the fMRI sampling period (TR = 3.0 s) and the peak latency of the Hemodynamic Response Function (HRF), the stimulus onset is shifted by one TR for temporal alignment. Voxel signals for each trial corresponding to the aligned TRs are extracted and concatenated to form trial data with a shape of [total_voxels, TRs_per_trial]. The trial labels are standardized to integer encoding starting from 0.

3. Feature Engineering & Dimensionality Reduction

  • The data for each trial at each time point (TR) is standardized using the StandardScaler, such that the data has a mean of 0 and a standard deviation of 1. Subsequently, Principal Component Analysis (PCA) is applied to reduce the dimensionality for each time point, and the results are integrated across all time points. The final feature matrix is constructed while preserving the principal component matrix and the proportion of explained variance.

4. Label Generation & Data Storage

  • The processed data is stored by participant in .npz format. After loading the data with np.load, the feature matrix can be extracted using the fmri keyword with a shape of (trials, number_of_principal_components, TRs). Labels ranging from 0 to 149 (150 possible values) can be extracted using the label keyword. Additionally, specific label names, ROI information, and voxel count statistics can be accessed through label_names, rois, and roi_voxel_counts.

IV.Usage Instructions​

  • The n_components parameter can be adjusted according to actual needs to balance the dimensionality reduction effect and computational resource consumption. The data uploaded in this study has been reduced to 1000 dimensions, which is the setting that achieved the optimal classification performance in the current experiments.
  • For data augmentation, category-specific processing was attempted, but experimental results showed that the adopted data augmentation methods did not improve the classification performance, so they were not included in the final workflow.
