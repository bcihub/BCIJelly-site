1_LSTM

I. Model Overview​

Long Short-Term Memory (LSTM) is a specialized Recurrent Neural Network (RNN) architecture, originally proposed by Hochreiter and Schmidhuber in 1997. Its primary objective is to resolve the vanishing and exploding gradient problems encountered by traditional RNNs when processing long sequential data. By introducing three gating mechanisms (the input gate, forget gate, and output gate), LSTM allows the network to selectively retain or discard information during the learning process, thereby demonstrating superior performance in tasks involving long-term dependencies.

From the perspectives of code implementation and network topology, this LSTM architecture comprises the following core modules:

  1. Sequential Feature Encoder

    The raw input typically consists of high-dimensional, multi-channel neural or physiological signals. The foundational layer of the model relies on a highly encapsulated RNNEncoder component for sequence ingestion. By configuring the number of network layers and the bidirectional flag, the model constructs a deep, bidirectional receptive field, simultaneously capturing historical context and future evolutionary trends at the current time step.

  2. Gating Mechanism & State Transition

    Unlike the Transformer, which relies on explicit positional encoding, the LSTM intrinsically aligns with tasks exhibiting strong temporal dependencies due to its recursive topology. Through the precise coordination of the forget gate, input gate, and output gate, the network can autonomously determine the "retention" or "forgetting" of neural potential features over extended time-step propagation. This core gating mechanism effectively overcomes the gradient vanishing bottleneck inherent in traditional RNNs.

  3. Dual-Track Task Head

    To seamlessly adapt to multi-scenario paradigms such as BCI, the model incorporates a highly flexible branched decoding logic at the output end:

  • Pooled Classification Decoding:

    For discrete motor intention recognition or qualitative analysis of entire signal segments, the model provides flexible pooling strategies. Whether applying global average pooling across the temporal dimension or precisely extracting the state at the final step of the sequence, the features subsequently flow into cascaded fully connected layers equipped with Layer Normalization and Dropout, outputting the final classification logits.

  • Time-Distributed Regression Decoding:

    When addressing Sequence-to-Sequence (Seq2Seq) regression requirements, such as continuous kinematic coordinate resolution, the model employs a dedicated TimeDistributedLinear module. Bypassing temporal pooling, this module applies a linear projection with Layer Normalization directly across each time step of the sequence length independently, utilizing shared weights. This facilitates frame-by-frame, high-fidelity tracking of high-frequency dynamic systems.

II. Data Preprocessing & Feature Engineering​

To ensure the training stability of the model on complex sequential data (such as EEG signals and time-series sensor signals), the framework establishes strict standardized protocols for data engineering:

  • Tensor Dimensionality Constraints: All input feature matrices must be strictly cast into standard three-dimensional array formats, representing the number of trials/samples, time steps, and channels/features, respectively.

  • Target Label Alignment:

    • Classification Tasks: The labels are required to be one-dimensional arrays. The framework incorporates comprehensive methods to automatically and accurately map labels—whether in the form of text, bytes, or non-contiguous discrete numerical values—into contiguous integer indices starting from 0.
    • Regression Tasks: It is strictly mandated that the label matrix assumes a three-dimensional shape, and the time steps of the features must be perfectly aligned with the time steps of the labels.

  • Feature Normalization and Strict Data Isolation (Z-score Normalization & Data Isolation):

    To eliminate the adverse effects of feature dimensional discrepancies on gradient updates, the framework adopts rigorous engineering practices to eradicate data leakage. The system independently fits the statistical metrics exclusively on the training set, and unidirectionally, statically maps the generated normalization statistics to the validation and test sets. This ensures absolute fairness in model performance evaluation and reflects its true generalization capability.

  • Feature Selection and Engineering:

    For complex time-series data, the performance of the LSTM model can be further enhanced through feature selection, PCA dimensionality reduction, or feature engineering in both the temporal and frequency domains.

IV. Demo​


import numpy as np
import bcijelly

def test_single_day() -> None:
    '''
        Demonstration of an intra-session (single-day) decoding task.
        The standardized workflow comprises the following stages:
        1. Importing the specific model module (e.g., bcijelly.model.lstm.classifier). The selection of the module determines the underlying architecture and the task paradigm (classification or regression; note: regression is currently not implemented).
        2. Invoking 'bcijelly.load_data' to retrieve the 'loaded_data' object.
        3. Executing the 'train' function of the model module to generate 'train_result'.
        4. Executing the 'test' function of the model module to generate 'test_result'.
        5. Invoking 'bcijelly.summary' for the synthesis and presentation of performance metrics.
    '''

    loaded_data = bcijelly.load_data(
        dataset_name="1_FALCONM1-A",            # Select the dataset
        protocol="single_day",                  # "single_day" | "cross_day"; demonstrates single-day decoding here
        subject_index=0,                        # Range: [0, n_subjects - 1]
        day_index=0,                            # Range: [0, n_days - 1]
        seed=42,                                # int; random seed for splitting test and validation sets in single-day decoding
        single_day_test_size=0.2,               # Range: (0, 1); the proportion of the test set within the single-day data
        single_day_val_ratio=0.125,             # Range: [0, 1); only valid when val_dataset is not None; represents the ratio of val within the train+val split
        # auto_download_missing=True,           # Automatically download from Hugging Face if local data is missing; defaults to True
        verbose=False,
    )

    train_result = module.train(
        train_dataset=loaded_data.train_dataset,
        val_dataset=loaded_data.val_dataset,    # loaded_data.val_dataset or None
        seed=42,  # int
        device="auto",
        run_name="demo_single_day_lstm",        # Used for naming the results directory
        epochs=1,
        plot=False,
        verbose=False,
    )

    test_result = module.test(
        test_dataset=loaded_data.test_dataset,
        verbose=False,
    )

    bcijelly.summary(
        loaded_data=loaded_data,                # The return value of bj.load_data
        Module=module,
        train_result=train_result,              # The return value of module.train
        test_result=test_result,                # The return value of module.test
        mode="full",                            # "compact" | "full"; controls the level of detail displayed in the summary
    )

def test_cross_day() -> None:
    '''
        Cross-day task example.
        In the testing code section, we demonstrate two ways to calculate test results:
        1. Combine all testing day data for testing to get an overall test result.
        2. Test each testing day's data individually to get the test result for each day, then calculate the average of these results as the overall test result.
        Users choose one of these methods according to their needs.
    '''

    import bcijelly.model.lstm.classifier as module      # Select the model; set classification/regression task (regression tasks are not yet supported)

    loaded_data = bcijelly.load_data(
        dataset_name="1_FALCONM1-A",
        protocol="cross_day",                   # "single_day" | "cross_day"; demonstrates cross-day decoding here
        subject_index=0,                        # Range: [0, n_subjects - 1]
        # auto_download_missing=True,           # Automatically download from Hugging Face if local data is missing; defaults to True
        verbose=False,
    )

    train_result = module.train(
        train_dataset=loaded_data.train_dataset,
        val_dataset=loaded_data.val_dataset,    # loaded_data.val_dataset or None
        seed=42,  # int
        device="auto",
        run_name="demo_cross_day_lstm",         # Used for naming the results directory
        epochs=1,
        plot=False,
        verbose=False,
    )

    # Testing all test days together
    test_result = module.test(
         test_dataset=loaded_data.test_dataset,
         verbose=False,
    )

    # Testing each test day individually
    day_acc_list: list[float] = []
    for one_day_test_dataset in loaded_data.test_day_datasets:
        one_day_result = module.test(
            test_dataset=one_day_test_dataset,
            verbose=False,
        )
        day_acc_list.append(float(one_day_result["acc"]))

    test_result = {
        "acc": float(np.mean(day_acc_list)),
        "per_day_acc": day_acc_list,
    }

    bcijelly.summary(
        loaded_data=loaded_data,                # The return value of bj.load_data
        Module=module,
        train_result=train_result,              # The return value of module.train
        test_result=test_result,                # The return value of module.test
        mode="full",                            # "compact" | "full"
    )

if __name__ == "__main__":
    test_single_day()
    test_cross_day()

V. Model Debugging & Evaluation​

Debugging and evaluating the LSTM model typically involves the following aspects:

  • Evaluation Metrics:
    • Classification Tasks: The framework not only computes the overall accuracy in real-time but also automatically generates a comprehensive confusion matrix, aiding in the intuitive diagnosis of class imbalances or misclassification biases.
    • Regression Tasks: Within the original physical dimensional space (following automatic inverse normalization), the framework strictly calculates the mean squared error, mean absolute error, and the coefficient of determination. The coefficient of determination provides a more precise measurement of the goodness of fit in high-fidelity trajectory tracking tasks.
  • Debugging Techniques:
    • Mandatory Gradient Clipping: To fundamentally circumvent the gradient explosion problem inherent to traditional RNN/LSTM architectures during long-sequence training, the underlying function mandates gradient clipping prior to every optimizer step. Serving as an intrinsic protective safeguard of the framework, this significantly enhances the training robustness of the model on datasets with low signal-to-noise ratios.
    • Dynamic Early Stopping & Best Checkpoint: To effectively combat model overfitting, the framework deeply integrates a dynamic early stopping mechanism tied to validation set performance during the training process. The model automatically monitors and saves the weight state that achieves optimal validation performance. If no improvement is observed within the tolerance period, training is terminated prematurely, thereby preventing overfitting.
    • Learning Rate Adjustment: An excessively large learning rate can lead to training instability, while one that is too small may result in unacceptably slow training. The training process can be optimized by progressively decaying the learning rate or by employing adaptive learning rate optimizers.

VI. Application Scenarios​

The design of this LSTM implementation framework exhibits exceptional scalability, enabling it to flexibly address core pain points across diverse domains:

  • Motor Intention Decoding (Motor Imagery, MI): Motor imagery tasks require subjects to mentally simulate specific actions without generating muscular electromyographic activity. During this period, scalp electroencephalography (EEG) exhibits non-stationary Event-Related Desynchronization/Synchronization (ERD/ERS) phenomena. The LSTM can sensitively capture such faint, temporally diffuse dynamic phase-transition features, thereby achieving highly accurate intention classification and providing a reliable control source for neural prosthetics and rehabilitation exoskeletons.
  • Emotional State Recognition: Neurophysiological signals provide the most authentic underlying emotional mapping, stripped of subjective cognitive disguise. By extracting multi-band feature sequences, bidirectional LSTMs can precisely track the dynamic temporal evolution of human emotions across the time dimension, subsequently outputting high-resolution emotional state evaluations.
  • Silent Speech Interfaces: For patients who have lost their vocal articulation capabilities, the decoding of "inner speech" based on brain-computer interfaces has emerged as a cutting-edge challenge. When patients silently rehearse command vocabularies in their minds, cortical networks generate neural spike sequences with robust temporal architectures. This represents the optimal theoretical focal point for LSTMs to demonstrate their capabilities in long-term semantic memory and temporal dependency mining.
  • Potential for Ultra-low Latency Online Closed-loop Control: In the clinical deployment of BCI, computational latency is the critical lifeline determining the success rate of prosthetic control or cursor clicks. When configured as unidirectional (by passing --no_bidirectional), the LSTM architecture employs strict causal time-step progression. This eliminates the need to wait for future neural data windows during inference, thereby offering ultra-low latency characteristics that are ideal for seamless deployment in real-time closed-loop systems. Conversely, for offline high-precision decoding tasks, the default bidirectional mode can be maintained to capture the complete global context.
