4_Stabilization

I. Model Overview​

Stabilization_MLP is a pipeline model specifically designed to address the non-stationarity of neural signals across days or different time periods in Brain-Computer Interfaces (BCI). In implanted microelectrode array recordings, the distribution of neural population signals drifts over time. This model hypothesizes that, despite the drift in neural firing at the observational level, the underlying low-dimensional neural dynamical manifold remains relatively stable.

From the perspective of underlying theory and engineering implementation, the core architecture of this model consists of the following five key components:

  1. Gaussian Factor Analysis Base Stabilizer

    During the training phase, the model extracts a specific number of trials at the end of the training data and uses the Expectation-Maximization algorithm to fit a standard Gaussian Factor Analysis model with diagonal private variances. Serving as the base stabilizer, this module is responsible for extracting pure initial latent variables from high-dimensional and noisy neural channels.

  2. Dynamic Cross-Day Calibration

    When faced with drifted data on a brand-new test day, the model refuses to retrain from scratch. Instead, it utilizes only a very small number of calibration trials at the very beginning of the test set (controlled by system-preset proportions or thresholds) to calculate and update the stabilizer parameters for that day. This few-shot calibration mechanism significantly reduces the daily maintenance cost of BCI systems.

  3. Orthogonal Alignment of Loading Matrices

    To eliminate the rotational uncertainty inherent in factor analysis across different days, the model automatically identifies stable loading rows by comparing the base loading matrix with the updated loading matrix, based on preset tolerance thresholds and channel counts. Subsequently, the system solves for the optimal orthogonal transformation matrix (i.e., the Procrustes alignment algorithm), forcibly and seamlessly rotating and mapping the latent variables of the test day back to the base training space.

  4. Task-Decoupled MLP Decoding

    After acquiring aligned and highly standardized latent features, the model connects to a downstream Multi-Layer Perceptron (MLP) for intention decoding. Tailored to different paradigms, the model enforces strict code decoupling at the underlying level:

    • Discrete Intention Classification: Invokes a standard classification network to delineate non-linear decision boundaries.
    • Continuous Kinematic Regression: To achieve high-fidelity trajectory tracking, the regression task specifically refactors the regression network, employing the Scaled Conjugate Gradient method for highly efficient weight optimization.

  5. Target-Driven Fine-Tuning

    Particularly in continuous regression tasks, to further squeeze decoding performance, the model natively supports weight inheritance and fine-tuning mechanisms. Upon completing orthogonal alignment, the model concatenates the original training data with the current calibration data and performs secondary optimization on the decoder's weights for a specified number of epochs, achieving final compensation for minor non-linear shifts.

II. Data Preprocessing & Feature Engineering​

The Stabilization_MLP architecture maintains highly stringent engineering control over input features to ensure the absolute safety of matrix operations and the effectiveness of alignment:

  • Trial Pooling & Flattening:

    Since traditional factor analysis lacks temporal memory capabilities, the model incorporates a built-in feature pooling method. This forcibly flattens the three-dimensional temporal tensor—containing trials, time steps, and channels—along the time axis, converting it into a two-dimensional matrix to feed the Expectation-Maximization algorithm for covariance estimation.

  • Strict Statistical Isolation:

    The aligned latent features must be standardized before being fed into the decoder. The model adheres to strict data isolation: it calculates the mean and standard deviation solely using the training set's latent variables, freezes these statistics, and unidirectionally applies them to the feature scaling of the test set, completely eradicating cross-day data leakage.

  • Chronological Calibration Split:

    In real-world BCI scenarios, calibration data can only be drawn from the beginning of the test day. Through slicing logic, the model absolutely guarantees that the samples used to update the stabilizer chronologically precede the validation samples used for performance evaluation.

IV. Demo​


import numpy as np
import bcijelly

def test_single_day() -> None:
    '''
        Demonstration of an intra-session (single-day) decoding task.
        The standardized workflow comprises the following stages:
        1. Importing the specific model module (e.g., bcijelly.model.stabilization.classifier). The selection of the module determines the underlying architecture and the task paradigm (classification or regression; note: regression is currently not implemented).
        2. Invoking 'bcijelly.load_data' to retrieve the 'loaded_data' object.
        3. Executing the 'train' function of the model module to generate 'train_result'.
        4. Executing the 'test' function of the model module to generate 'test_result'.
        5. Invoking 'bcijelly.summary' for the synthesis and presentation of performance metrics.
    '''

   loaded_data = bcijelly.load_data(
        dataset_name="1_FALCONM1-A",            # Select the dataset
        protocol="single_day",                  # "single_day" | "cross_day"; demonstrates single-day decoding here
        subject_index=0,                        # Range: [0, n_subjects - 1]
        day_index=0,                            # Range: [0, n_days - 1]
        seed=42,                                # int; random seed for splitting test and validation sets in single-day decoding
        single_day_test_size=0.2,               # Range: (0, 1); the proportion of the test set within the single-day data
        single_day_val_ratio=0.125,             # Range: [0, 1); only valid when val_dataset is not None; represents the ratio of val within the train+val split
        # auto_download_missing=True,           # Automatically download from Hugging Face if local data is missing; defaults to True
        verbose=False,
    )

    train_result = module.train(
        train_dataset=loaded_data.train_dataset,
        val_dataset=loaded_data.val_dataset,    # loaded_data.val_dataset or None
        seed=42,  # int
        device="auto",
        run_name="demo_single_day_stabilization",        # Used for naming the results directory
        epochs=1,
        plot=False,
        verbose=False,
    )

    test_result = module.test(
        test_dataset=loaded_data.test_dataset,
        verbose=False,
    )

    bcijelly.summary(
        loaded_data=loaded_data,                # The return value of bj.load_data
        Module=module,
        train_result=train_result,              # The return value of module.train
        test_result=test_result,                # The return value of module.test
        mode="full",                            # "compact" | "full"; controls the level of detail displayed in the summary
    )

def test_cross_day() -> None:
    '''
        Cross-day task example.
        In the testing code section, we demonstrate two ways to calculate test results:
        1. Combine all testing day data for testing to get an overall test result.
        2. Test each testing day's data individually to get the test result for each day, then calculate the average of these results as the overall test result.
        Users choose one of these methods according to their needs.
    '''

    import bcijelly.model.stabilization.classifier as module      # Select the model; set classification/regression task (regression tasks are not yet supported)

    loaded_data = bcijelly.load_data(
        dataset_name="1_FALCONM1-A",
        protocol="cross_day",                   # "single_day" | "cross_day"; demonstrates cross-day decoding here
        subject_index=0,                        # Range: [0, n_subjects - 1]
        # auto_download_missing=True,           # Automatically download from Hugging Face if local data is missing; defaults to True
        verbose=False,
    )

    train_result = module.train(
        train_dataset=loaded_data.train_dataset,
        val_dataset=loaded_data.val_dataset,    # loaded_data.val_dataset or None
        seed=42,  # int
        device="auto",
        run_name="demo_cross_day_stabilization",         # Used for naming the results directory
        epochs=1,
        plot=False,
        verbose=False,
    )

    # Testing all test days together
    test_result = module.test(
         test_dataset=loaded_data.test_dataset,
         verbose=False,
    )

    # Testing each test day individually
    day_acc_list: list[float] = []
    for one_day_test_dataset in loaded_data.test_day_datasets:
        one_day_result = module.test(
            test_dataset=one_day_test_dataset,
            verbose=False,
        )
        day_acc_list.append(float(one_day_result["acc"]))

    test_result = {
        "acc": float(np.mean(day_acc_list)),
        "per_day_acc": day_acc_list,
    }

    bcijelly.summary(
        loaded_data=loaded_data,                # The return value of bj.load_data
        Module=module,
        train_result=train_result,              # The return value of module.train
        test_result=test_result,                # The return value of module.test
        mode="full",                            # "compact" | "full"
    )

if __name__ == "__main__":
    test_single_day()
    test_cross_day()

V. Model Debugging & Evaluation​

For the performance evaluation of non-stationary systems, the model features built-in, highly automated diagnostic metrics and visualization outputs:

  • Multi-dimensional Regression Metrics:

    In continuous kinematic (e.g., spatial arm coordinates) reconstruction, the model not only outputs the globally averaged coefficient of determination but also forcibly decouples and outputs the goodness of fit for each independent dimension. This greatly assists in troubleshooting the physical causes of decoding failures in specific directions within the control system.

  • Optimization Loss Curve:

    During simulation training, the system automatically captures and plots the loss curve, allowing researchers to intuitively determine whether oscillations or premature convergence occur during the conjugate gradient descent process.

  • Chronological Robustness Plot:

    After evaluating multiple test days, the system sorts them in natural chronological order and outputs a cross-day trend chart of accuracy or the coefficient of determination, complete with a mean baseline. This visually demonstrates the anti-degradation ability of the manifold alignment algorithm against the passage of time.

VI. Application Scenarios​

The theoretical mechanisms of Stabilization_MLP directly address the core pain points of long-term implanted BCIs, demonstrating exceptionally high scientific research and clinical application value:

  • Stabilizing BCIs against Signal Drift

    Over weeks or months of recording, micromotion between the microelectrodes and brain tissue occurs, causing changes in the neurons recorded by the same channel. By extracting the intrinsic manifold and utilizing few-shot learning for orthogonal rotation alignment, the model effectively compensates for this deviation. This deceives the downstream decoder into perceiving the input data as still being in its initial state, thereby maintaining high-level, long-term stable decoding.

  • Ultra-low Cost Daily Calibration

    Traditional BCIs require patients to spend significant amounts of time every day performing repetitive actions to retrain the entire decoding model. Relying on the preset mechanisms of this architecture, the system only requires an extremely small amount of data for baseline warm-up at startup each day. By updating the relatively small loading matrix, it can instantly revert to a usable state, vastly improving the user experience for clinical patients.

  • Calibration-driven Online Real-time Decoding

    This manifold stabilization architecture achieves an exceptionally elegant engineering separation at the boundary between offline and online operations. Before initiating the daily system, it requires only a minimal amount of time during the offline or preparatory phase to extract few-shot data for updating and aligning the stabilization matrix. Once the orthogonal rotation matrix is locked in, the downstream decoupled network can unleash ultimate inference speeds. This endows the model with remarkably strong online real-time closed-loop control capabilities, ensuring that clinical patients can enjoy ultra-low latency prosthetic manipulation experiences.
