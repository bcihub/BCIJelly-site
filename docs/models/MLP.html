2_MLP

I. Model Overview​

The Multi-Layer Perceptron (MLP) is the most classical feedforward neural network architecture, whose backpropagation training algorithm was systematically proposed and popularized by Rumelhart, Hinton, and Williams in 1986. In the domain of brain-computer interfaces (BCI), the MLP is widely deployed as a performance benchmark for neural signal decoding. Its core hypothesis is that high-dimensional neural population activity can be mapped to behavioral labels or kinematic states through multiple layers of nonlinear transformations. Although the model itself contains no temporal modeling structure whatsoever, its minimalist topology and extremely high training efficiency allow it to remain surprisingly competitive across a broad range of decoding scenarios.

From the perspective of code implementation and network topology, this MLP architecture comprises the following core modules:

  1. Spatiotemporal Flattening Input Layer

The model performs no temporal modeling of raw neural signals whatsoever. The input tensor (batch, time, channels) is directly flattened into a one-dimensional vector flatten(time × channels). This design completely discards the sequential order among time steps, treating all spatiotemporal information as an unordered feature set. Statistical associations among features are implicitly captured through the global connectivity weights of the subsequent fully connected layers.

  1. Cascaded Fully Connected Hidden Layers

The flattened feature vector passes sequentially through three fully connected hidden layers, with dimensions compressed stepwise as [512, 256, 128]. Each linear transformation is followed by a nonlinear activation function (e.g., ReLU or GELU) and interleaved with Dropout regularization (0.3 for classification tasks, 0.35 for regression tasks). This "wide-then-narrow" pyramidal structure preserves sufficient representational capacity while effectively suppressing the tendency to overfit on small-sample neural datasets.

  1. Dual-Task Decoding Heads

To accommodate the diverse decoding paradigms of BCI, the model is equipped with flexible branching decode logic at the output:

  • Classification decoding: The final hidden state is fed into a linear classification layer, outputting class logits. A Softmax function yields the probability distribution over discrete behavioral intentions.

  • Regression decoding: The final hidden state is fed into a linear regression layer, outputting two-dimensional kinematic coordinates sample by sample, achieving point estimation of continuous movement trajectories.

  1. Z-Score Input Normalization

The model enforces a Z-score normalization strategy using the training-set mean and standard deviation, bringing all neural channels to a common scale of zero mean and unit variance before forward propagation. This preprocessing step is critical for eliminating cross-channel scale disparities and accelerating the convergence of the fully connected layers.

II. Demo​

classification​

import bcijelly as bj

loaded_data = bj.load_data(
    dataset_name="5_Jango",
    protocol="cross_day", task="classification",
    subject_index=0, split_ratio="7:1:2",
)
model = bcijelly.model.mlp(task="classification")
train_result = model.train(loaded_data.train_dataset, loaded_data.val_dataset)
test_result = model.test(loaded_data.test_dataset)

bj.summary(loaded_data, model, train_result, test_result)

IV. Model Debugging & Evaluation​

Evaluation Metrics

  • Classification tasks: Overall accuracy is computed, and a confusion matrix is generated to diagnose class imbalance or misclassification bias.
  • Regression tasks: Mean squared error (MSE), mean absolute error (MAE), and the coefficient of determination (R²) are computed in the original physical space after automatic inverse normalization.

Key Debugging Techniques

  • Dropout tuning: dropout is the primary regularization mechanism for the MLP. The default is 0.3 for classification and 0.35 for regression. For datasets with few channels and small sample sizes, increasing dropout can further suppress overfitting; conversely, it can be reduced for larger datasets.
  • Hidden layer dimensionality: The three-layer [512, 256, 128] configuration is the default for typical neural datasets (hundreds of channels, thousands of time steps). When the number of input features is far smaller than 512, the hidden dimensions should be proportionally scaled down to avoid severe over-parameterization.
  • Learning rate and early stopping: The Adam optimizer is paired with an early-stopping patience of 15. The MLP typically converges faster than deeper architectures such as Transformers, reaching optimal performance within 80 epochs.
  • Z-Score normalization: Input normalization is critical for fully connected networks. If data is fed into the MLP without normalization, large-scale channels will dominate gradient updates, rendering the model effectively deaf to other channels.

V. Application Notes​

This MLP implementation framework trades topological minimalism for maximum generality:

(i) Performance Baseline Benchmark: As a purely feedforward model devoid of any temporal inductive bias (no convolution, no recurrence, no attention), the MLP defines the lower bound of complexity among all BCI decoders. If a complex model (e.g., Transformer or LFADS) fails to substantially outperform the MLP on a given dataset, this signals that the information in that dataset is primarily encoded in cross-channel correlations rather than in temporal structure—providing a critical diagnostic signal for downstream model selection.

(ii) High-Throughput Batch Inference: Owing to the extreme parallel efficiency of fully connected layers on GPUs, the MLP achieves the highest inference throughput among all decoders (a single forward pass involves only three matrix multiplications), making it naturally suited for scenarios requiring rapid labeling or offline pre-screening of massive neural datasets.
