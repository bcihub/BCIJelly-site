6_FENet

I. Model Overview​

FENet is a standalone regression wrapper specifically designed for time-series signals. Its core concept lies in addressing feature extraction and dimensionality reduction in multi-channel time series through a network architecture inspired by the wavelet transform. It can efficiently separate and extract key frequency band/scale features from time-series data while preserving global context.

From the code implementation and network topology, the present FENet architecture consists of the following core modules:

  1. Wavelet Feature Split Block (WaveletBlock)

    As the underlying basic component, this module performs a single-channel feature/pass split block inspired by FENet. Input data is padded and fed into parallel 1D convolutional layers (feat_conv and pass_conv), respectively. The feature extraction branch is compressed via LeakyReLU activation and adaptive average pooling, while the pass branch applies Dropout and continues to forward the signal to subsequent network layers, thereby achieving layer-by-layer information decoupling and extraction in deep networks.

  2. Channel-Shared Backbone Network (FENetBackbone)

    When processing multi-channel physiological data of shape (B, T, C), the framework adopts a special isolation topology: it applies an identical single-channel convolutional stack independently and repeatedly to every individual neural signal channel (apply same single-channel stack to every neural channel). After each channel undergoes multi-stage WaveletBlock filtering, the extracted multi-scale features are concatenated in the final stage and standardized globally through LayerNorm.

  3. Multi-Mode Sequence Decoder Head (Dual-Track / Sequence Decoder Head)

    To accommodate various time-series tasks, FENetRegressorModel is equipped with an extremely flexible decoding route, which can be freely switched among three modes via the sequence_decoder parameter:

  • Global decoding (Global): Flattens the backbone-extracted features and sends them into a multilayer perceptron (MLP) head composed of fully connected layers (Linear), GELU activation, and Dropout, directly outputting global prediction results after dimensionality reduction.
  • Temporal/hybrid decoding (Temporal/Hybrid): Specially equipped with a temporal decoding head (temporal_head), which replaces fully connected layers with cascaded 1D convolutions (Conv1d) to maintain temporal resolution unchanged. In Hybrid mode, global static feature predictions can also be directly added to temporal dynamic predictions to meet the complex requirements of time-distributed outputs.

IV. Model Debugging & Evaluation​

The debugging and evaluation of the FENet model are deeply integrated within its dedicated training and validation loop (regressor.py):

Evaluation Metrics:

Regression task: The framework is highly focused on continuous variable regression tracking. During the model validation phase, in addition to computing gradients via SmoothL1Loss, the system also rigorously calculates mean squared error (MSE), mean absolute error (MAE), and the coefficient of determination (R2 Score). Among them, the validation set R2 score (best_val_r2) is established by the framework as the absolute core metric to evaluate whether the model's current state reaches the "best" threshold.

Debugging Techniques:

Mandatory Gradient Clipping: To ensure the training stability of deep time-series networks under complex data, the underlying _train_one_epoch function incorporates a mandatory clip_grad_norm_ operation before each optimizer step, strictly limiting the maximum gradient norm of parameters to 1.0.

Dynamic Early Stopping & Best Checkpoint: A training tolerance (patience) monitoring mechanism is built in. When the validation set R2 performance fails to surpass the historical best for a specified number of epochs, the system decisively triggers early stopping to terminate training. At the same time, the system clones and serializes the optimal model weights (best_state_dict) in real time during training, ensuring automatic rollback to the performance peak state (restore_best) before the end of training.

Cosine Annealing Learning Rate (Cosine Annealing LR): Combined with a robust AdamW optimizer, the learning rate scheduler adopts CosineAnnealingLR, allowing the learning rate to smoothly decay to eta_min (set to 10% of the initial learning rate) over the given epoch cycle, effectively preventing the model from oscillating near the optimum point in the later stages of training.

Test-time Calibration: In addition to the standard "train-test" split, the framework innovatively supports online calibration on the test set. In the test phase, by setting parameters such as calib_epochs and calib_ratio, the model can utilize a very small number of calibration trial data to perform fine-tuning updates. To prevent catastrophic forgetting, during debugging one can use calib_scope to finely control the unfreezing of gradients only for specific parts of the network (e.g., updating only the decoder head or the entire network all).

V. Application Description​

Combined with the built-in data validation assertions and input tensor processing logic in the source code, the architectural design of FENet is primarily anchored in the following cutting-edge application scenarios:

  • High-Fidelity 2D Kinematic Trajectory Regression (2D Kinematic Trajectory Decoding):There exist strict checkpoints in the data loading logic: the ground truth target must have a shape of (N, T, 2) (e.g., triggering the alarm Regression y must have shape (N, T, 2)), and the default recognition label name of the input data is "neural_activity" or "spike". This indicates that the model perfectly fits into the brain–computer interface (BCI) field, enabling the use of a subject’s neural firing or feature time series to continuously predict two-dimensional kinematic coordinates frame by frame (such as on-screen cursor control, 2D velocity vector mapping, etc.).

  • Weakly Coupled Multi-channel Sensor Feature Extraction (Uncoupled Multi-channel Sensor Mining): Considering that the FENetBackbone abandons dense spatial mixing between channels and instead rigorously applies a “shared-parameter single-channel filter bank” to every input channel, this is highly suitable for processing sensor arrays that are physically highly uncorrelated or independently arranged. For example, extracting time-frequency features without bias from widely distributed microelectrode arrays or high-density surface electromyography (HD-sEMG).

  • Cross-day/Non-stationary Physiological Online Adaptive Control (Cross-day Non-stationary Adaptive Control): Since neural physiological signals easily undergo baseline drift and strong non-stationarity across different natural days. The model presets targeted preprocessing normalization strategies such as day_mean_std. Combined with the aforementioned test-time calibration functionality, it can rapidly align and fine-tune the specific head of the network using only a few seconds to minutes of new trial data on the current day after model deployment, significantly enhancing the usability and online robustness of BCI systems.
