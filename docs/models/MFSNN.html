16_MFSNN

I. Model Overview​

MFSNN (Multi-Frequency Spiking Neural Network) is a Spiking Neural Network (SNN) architecture specifically designed for neural signal classification tasks. Unlike conventional artificial neural networks, MFSNN draws inspiration from the spiking mechanisms of biological neurons, achieving end-to-end gradient training while preserving the biological plausibility of SNNs through differentiable Leaky Integrate-and-Fire (LIF) firing rate approximation. Its core hypothesis is that the combination of multi-frequency temporal channel grouping and channel attention mechanisms can effectively capture feature patterns of neural signals across different temporal scales.

From the perspective of implementation and network topology, the MFSNN architecture comprises the following core modules:

  1. LIF Firing Rate Approximation Layer

The model relies on a differentiable LIF neuron model at its foundation. When the SpikingJelly library is installed, the model invokes its native LIFNode implementation with the ATan surrogate function for spike simulation; in environments lacking this dependency, it automatically falls back to a Sigmoid soft-activation approximation, ensuring code portability. This layer performs temporal firing rate encoding of the input through multi-step simulation (default: 20 steps), converting continuous values into spike frequency representations.

  1. Multi-Channel Grouped Sub-Networks (SubNet)

Input channels are uniformly partitioned into several groups (default: 16), with each group independently constructing a sub-network (SubNet). Each SubNet comprises:

  • Spatial attention branch: Through GroupNorm → LIF activation → Channel Attention module, it adaptively assigns weights to each channel. The channel attention employs a two-layer fully connected structure following global average pooling, with an embedded LIF spiking layer to preserve biological spiking characteristics.
  • Temporal convolution branch: Dilated causal convolution (dilation=2, kernel_size=3) is employed to extract multi-scale temporal features, followed by average pooling for downsampling.
  • Learnable temporal mapping matrix: A trainable parameter matrix directly maps the original temporal dimension to the downsampled temporal dimension, which is added to the spatiotemporal fused output, providing additional residual temporal modeling capability.
  1. Spatiotemporal Fusion and Classification Head

Outputs from each SubNet are concatenated along the feature dimension, then processed through LIF activation, LayerNorm normalization, and Dropout regularization before being fed into a fully connected classification layer to produce the final classification logits. This design enables the model to capture neural activity patterns across different frequency bands in parallel and perform cross-group information fusion at the decision layer.

III. Demo​

classification​


import bcijelly as bj

loaded_data = bj.load_data(
    dataset_name="5_Jango",
    protocol="cross_day", task="classification",
    subject_index=0, split_ratio="7:1:2",
)
model = bcijelly.model.mfsnn(task="classification")
train_result = model.train(loaded_data.train_dataset, loaded_data.val_dataset)
test_result = model.test(loaded_data.test_dataset)

bj.summary(loaded_data, model, train_result, test_result)

IV. Model Debugging & Evaluation​

Evaluation Metrics

  • Classification tasks: Compute overall accuracy and generate a confusion matrix to diagnose class imbalance or misclassification bias.

Key Debugging Strategies

  • LIF time constant adjustment: The tau parameter controls the leaky time constant of the LIF neuron membrane potential; larger values make the neuron more sensitive to long-duration inputs, while smaller values focus on transient responses.
  • Simulation steps: The steps parameter determines the number of simulation iterations for the LIF layer; more steps provide more precise firing rate estimates but increase computational cost.
  • Channel grouping: The groups parameter must evenly divide the input channel dimension; a larger number of groups results in fewer channels per sub-network, facilitating the learning of local inter-channel correlations.
  • Spiking vs. soft activation switching: The use_spikingjelly toggle enables switching between genuine spike simulation and Sigmoid approximation; the former is more biologically realistic but requires additional dependencies.
  • Loss function selection: Both MSE (one-hot target) and cross-entropy loss types are supported; MSE loss typically provides smoother gradient signals in spiking networks.

V. Application Notes​

This MFSNN implementation fully leverages the inherent advantages of spiking neural networks:

(i) Low-Power Event-Driven Computing:The spike sparsity of SNNs confers inherent low-power advantages on neuromorphic hardware (e.g., Intel Loihi, IBM TrueNorth). The multi-frequency grouped architecture of MFSNN further enhances spike sparsity, making it highly suitable for real-time deployment on edge BCI devices.

(ii) Multi-Scale Temporal Feature Capture:Through the synergistic design of channel grouping and dilated convolution, the model simultaneously extracts rapid transient responses and slow modulation trends of neural signals at different temporal resolutions, demonstrating broad applicability across diverse BCI tasks including motor intention decoding and emotional state classification.
