19_UniBCI

I. Model Overview​

UniBCI (Universal Brain-Computer Interface) is a general-purpose invasive neural signal decoding model based on a hybrid attention mechanism, with its backbone network derived from the large-scale neural population activity pretraining architecture of POYO-1 (Azabou et al., 2024). In the field of brain-computer interfaces (BCI), traditional single-task decoders face significant bottlenecks in generalization across brain regions, species, and behavioral paradigms. The core hypothesis of UniBCI is that by uniformly transforming spike sequences of arbitrary shapes into fixed-length token representations and combining intra-patch and inter-patch dual-stage attention mechanisms, the model can simultaneously capture local temporal dynamics and global cross-channel interactions within a unified representation space, thereby sharing a single pretrained backbone across both classification (discrete intent recognition) and regression (continuous movement decoding) tasks.

From the perspective of implementation and network topology, the UniBCI architecture comprises the following core modules:

  1. Spike Tokenizer

The model does not perform manual feature extraction on raw spike sequences. Instead, it transforms neural data of arbitrary shapes into fixed-dimensional sequence tensors through a unified tokenization pipeline:

  • Channel Grouping: C neural channels are evenly distributed across S spatial tokens (default S=8), with each token accommodating N=group_size neurons (default 32), for a total capacity of S × N = 256 channels. Shortfalls are zero-padded, and excess channels are truncated.
  • Temporal Chunking: Through uniform temporal binning (mean-pooling aggregation by default), the original T time steps are compressed or padded to a fixed length of P×W (P=patch_num=10, W=patch_size=10, totaling 100 time steps).
  • Positional Encoding Generation: Temporal position indices (time_pos) and spatial position indices (space_pos) are automatically generated, along with a padding_mask for under-capacity channels, enabling selective masking in subsequent attention modules.
  1. Input Projection and Spatiotemporal Positional Encoding

The grouped spike tensor (B, P, W, S, C) passes through two linear projections (C → embed_dim → embed_dim, with ReLU activation) to map binary spikes into dense real-valued representations. The model then injects two independent learnable position embedding tables:

  • Time Embedding: nn.Embedding(max_t, embed_dim), providing absolute temporal information for each time step.
  • Space Embedding: nn.Embedding(max_s, embed_dim), providing channel group identity information for each spatial token.
  • Both embeddings are added directly to the input features and regularized via Dropout, enabling the model to perceive the "when" and "where" of spike events.
  1. Optional Text-Conditional Encoding

The model supports conditional modulation of representations through textual metadata (species, subject, brain region, task type, experimental session). Specifically, a frozen SentenceTransformer (MiniLM) encodes natural language descriptions into 384-dimensional text vectors, which are then mapped to embed_dim through a frozen two-layer projection network (Linear → LayerNorm → ReLU) and injected additively into the token sequence. This design enables the model to perceive contextual differences in data sources during multi-dataset joint pretraining.

  1. Intra-Inter Hybrid Attention Encoder

The core backbone is composed of multiple stacked IntraInterBlock layers (default 4), with each block sequentially executing two complementary attention operations:

  • Intra-patch Linear Attention: Linear attention is performed along the temporal dimension W within each patch, using the elu(x)+1 feature map kernel, reducing the standard dot-product attention complexity from O(n²) to O(n). This enables the model to efficiently handle fine-grained temporal dependencies within patches.
  • Inter-patch Sliding Window Attention: Causal-masked sliding window attention is performed along the patch dimension P, with a default window size of 10. Standard scaled dot-product softmax attention is used, allowing each patch to attend only to its neighboring preceding patches, balancing global receptive field with computational efficiency.
  • Feature Fusion: Inter-patch features are broadcast-expanded to the temporal dimension, added to intra-patch outputs, and processed through residual connections and an FFN (two-layer fully connected, expansion ratio 4.0, ReLU activation), achieving deep fusion of cross-scale information.
  1. Dual-Track Task Readout Heads

To accommodate diverse BCI decoding paradigms, the model configures flexible branching readout logic atop the pretrained backbone via UniBCIWrapper:

  • Classification Readout: The backbone's four-dimensional output features (B, P×S, W, embed_dim) are flattened into a one-dimensional vector and sequentially fed through Linear(dim, 4×embed_dim) → ReLU → Linear(4×embed_dim, num_classes), producing class logits.

  • Regression Readout: The same flattened features pass through Linear(dim, 4×embed_dim) → ReLU → Linear(4×embed_dim, num_outputs × P × W) and are reshaped to (B, P×W, num_outputs), realizing time-distributed sequence regression, precisely adapted for frame-by-frame prediction of continuous kinematic coordinates.

II. Demo​

classification​


import bcijelly as bj

loaded_data = bj.load_data(
    dataset_name="5_Jango",
    protocol="cross_day", task="classification",
    subject_index=0, split_ratio="7:1:2",
)
model = bcijelly.model.unibci(task="classification")
train_result = model.train(loaded_data.train_dataset, loaded_data.val_dataset)
test_result = model.test(loaded_data.test_dataset)

bj.summary(loaded_data, model, train_result, test_result)

regression​


import bcijelly as bj

loaded_data = bj.load_data(
    dataset_name="5_Jango",
    protocol="cross_day", task="regression",
    subject_index=0, split_ratio="7:1:2",
)
model = bcijelly.model.unibci(task="regression")
train_result = model.train(loaded_data.train_dataset, loaded_data.val_dataset)
test_result = model.test(loaded_data.test_dataset)

bj.summary(loaded_data, model, train_result, test_result)

By default, pretrained weights are not loaded. To load pretrained weights, use:

train_result = model.train(
    loaded_data.train_dataset, loaded_data.val_dataset,
    pretrained_path="auto",
)

Notes​

  • UniBCI's classification and regression tasks share the exact same default hyperparameter configuration. The core differences lie in the optimizer and loss function: classification uses Adam + CrossEntropyLoss, while regression uses AdamW + CosineAnnealingLR + SmoothL1Loss.
  • In the regression task, although the default value of weight_decay is 0, the effective value is enforced via max(weight_decay, 1e-4), meaning even passing 0 results in weight decay of 1e-4.
  • val_ratio only takes effect when no external validation set (val_dataset) is provided, and is used to split a validation set from the training data.
  • pretrained_path supports "auto" to automatically download POYO-1 pretrained weights; set to None for random initialization.

IV. Model Debugging & Evaluation​

Evaluation Metrics

  • Classification: Accuracy + confusion matrix.
  • Regression: MSE, MAE, R2 computed in the original space after inverse normalization.

Optimization Strategies

  • Classification: Adam + CrossEntropyLoss.
  • Regression: AdamW + CosineAnnealingLR + SmoothL1Loss (β=1.0), with gradient clipping (max_norm=1.0) to prevent gradient explosion.

Hyperparameter Tuning

  • Channel capacity: max_group_num × group_size (default 256) sets the max channels. For datasets exceeding this, select the highest-variance channel subset; for fewer channels, zero-padding applies automatically — reduce max_group_num to cut useless computation.
  • Temporal receptive field: patch_num × patch_size (default 100). Increase time_size for high-sampling-rate data (>1 kHz) to cover longer behavioral windows; scale down for low-sampling-rate data.
  • Cross-patch attention: attn_window (default 10, i.e., all-to-all). Larger values capture longer-range coupling but linearly increase compute.
  • Linear attention: Intra-patch uses linear attention for O(n) complexity; precision loss is negligible at W=10. Monitor approximation error when patch_size > 50.
  • Text encoding: Only meaningful for multi-dataset joint pretraining. No benefit for single-dataset fine-tuning, only adds inference overhead.

V. Application Notes​

UniBCI trades a unified tokenized representation and hybrid attention mechanism for cross-task, cross-dataset generalization:

(1) Universal neural signal decoding: UniBCI is the only model in the framework sharing a classification and regression pretraining backbone. Its unified spike tokenization allows seamless transfer across motor decoding, sensory decoding, and cognitive state recognition without task-specific input layer redesign.

(2) Large-scale pretraining and fine-tuning: POYO-1 pretrained weights enable rapid few-shot convergence on small datasets. The text-conditional encoding mechanism further supports joint pretraining on heterogeneous multi-source data, providing a foundation for cross-species, cross-region universal neural decoding models.
